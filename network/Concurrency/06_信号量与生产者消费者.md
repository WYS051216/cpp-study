# 线程同步笔记：信号量与生产者-消费者模型

## 17. 信号量

### 17.1 信号量的作用
信号量用于多线程、多任务同步。  
一个线程完成某个动作后，可以通过信号量通知其他线程继续执行。

信号量本质上强调“**资源计数**”：
- 灯亮：表示资源可用
- 灯灭：表示资源不可用，线程需要等待

---

### 17.2 信号量与互斥锁、条件变量的区别
- **互斥锁**：保护临界区，保证同一时刻只有一个线程访问共享资源
- **条件变量**：用于等待某个条件成立
- **信号量**：用于表示资源数量，控制线程能否继续执行

注意：
- 信号量可以完成同步
- 但它不能完全代替互斥锁
- 多线程访问共享数据时，通常仍要配合互斥锁使用

---

### 17.3 头文件与类型
```c
#include <semaphore.h>
sem_t
```

---

### 17.4 常用函数

#### 17.4.1 初始化
```c
sem_init(sem_t *sem, int pshared, unsigned int value);
```

参数说明：
- `sem`：信号量地址
- `pshared`
  - `0`：线程间同步
  - 非 `0`：进程间同步
- `value`：初始资源数

#### 17.4.2 销毁
```c
sem_destroy(sem_t *sem);
```

#### 17.4.3 申请资源
```c
sem_wait(sem_t *sem);
```

作用：
- 资源数大于 `0` 时，获取资源并减 `1`
- 资源数等于 `0` 时，线程阻塞

#### 17.4.4 尝试申请资源
```c
sem_trywait(sem_t *sem);
```

作用：
- 不阻塞
- 如果资源不可用，立即返回错误

#### 17.4.5 释放资源
```c
sem_post(sem_t *sem);
```

作用：
- 资源数加 `1`
- 若有线程阻塞，则可唤醒其中一个

#### 17.4.6 查看资源数
```c
sem_getvalue(sem_t *sem, int *sval);
```

作用：
- 获取当前信号量资源个数

## 18. 单资源情况下的生产者-消费者模型（信号量实现）

### 18.1 实现思路
定义两个信号量：
- `semp`：生产者信号量
- `semc`：消费者信号量

初始值：
- `semp = 1`
- `semc = 0`

表示：
- 一开始允许生产
- 一开始不允许消费

> 说明：这种写法适合单资源同步场景，但访问共享链表时严格来说仍建议加锁。

### 18.2 示例代码
```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <semaphore.h>

// 生产者信号量
sem_t semp;
// 消费者信号量
sem_t semc;

pthread_mutex_t mutex;

// 链表结点
typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

// 生产者
void* producer(void* arg)
{
    while (1)
    {
        sem_wait(&semp);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %ld, number: %d\n", pthread_self(), newNode->number);

        sem_post(&semc);
        sleep(rand() % 3);
    }
    return NULL;
}

// 消费者
void* consumer(void* arg)
{
    while (1)
    {
        sem_wait(&semc);

        Node* node = head;
        printf("消费者, id: %ld, number: %d\n", pthread_self(), node->number);
        head = head->next;
        free(node);

        sem_post(&semp);
        sleep(rand() % 3);
    }
    return NULL;
}

int main()
{
    sem_init(&semp, 0, 1);
    sem_init(&semc, 0, 0);
    pthread_mutex_init(&mutex, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);
    sem_destroy(&semc);
    sem_destroy(&semp);

    return 0;
}
```

### 18.3 代码说明
这段代码表达的逻辑是：
- 生产者生产一个资源后，通知消费者
- 消费者消费一个资源后，再通知生产者继续生产

但问题是：
- 对共享链表 `head` 的操作没有加锁保护
- 多线程环境下可能出现线程安全问题

因此更推荐下面的“**信号量 + 互斥锁**”写法。

## 19. 多线程生产者-消费者模型（信号量 + 互斥锁）

### 19.1 实现思路
在多生产者、多消费者模型中：
- 用**信号量**控制是否可以生产 / 消费
- 用**互斥锁**保护共享链表

初始值设置：
- `semp = 5`：最多允许生产 5 个资源
- `semc = 0`：开始时没有资源可消费

### 19.2 示例代码
```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <semaphore.h>

// 生产者信号量
sem_t semp;
// 消费者信号量
sem_t semc;

pthread_mutex_t mutex;

// 链表结点
typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

// 生产者
void* producer(void* arg)
{
    while (1)
    {
        sem_wait(&semp);
        pthread_mutex_lock(&mutex);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %ld, number: %d\n", pthread_self(), newNode->number);

        pthread_mutex_unlock(&mutex);
        sem_post(&semc);
    }
    return NULL;
}

// 消费者
void* consumer(void* arg)
{
    while (1)
    {
        sem_wait(&semc);
        pthread_mutex_lock(&mutex);

        Node* node = head;
        printf("消费者, id: %ld, number: %d\n", pthread_self(), node->number);
        head = head->next;
        free(node);

        pthread_mutex_unlock(&mutex);
        sem_post(&semp);

        sleep(2);
    }
    return NULL;
}

int main()
{
    sem_init(&semp, 0, 5);
    sem_init(&semc, 0, 0);
    pthread_mutex_init(&mutex, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);
    sem_destroy(&semc);
    sem_destroy(&semp);

    return 0;
}
```

### 19.3 代码分析
- `sem_wait(&semp)`：生产前先申请资源名额
- `sem_post(&semc)`：生产完成后通知消费者可消费
- `sem_wait(&semc)`：消费前先检查是否有资源
- `sem_post(&semp)`：消费完成后释放生产名额
- `pthread_mutex_lock()` / `pthread_mutex_unlock()`：保护共享链表

## 20. 条件变量与信号量对比总结

### 20.1 相同点
- 都能用于线程同步
- 都能实现生产者-消费者模型
- 都能让线程在条件不满足时阻塞

### 20.2 不同点

| 对比项 | 条件变量 | 信号量 |
|---|---|---|
| 本质 | 等待某个条件成立 | 表示资源数量 |
| 是否需要互斥锁 | 必须配合互斥锁 | 一般建议配合互斥锁 |
| 典型用途 | 等待事件发生 | 控制资源访问数量 |
| 是否有计数功能 | 没有 | 有 |
| 线程唤醒方式 | `signal / broadcast` | `post` |

---

## 21. 使用建议

### 21.1 什么时候用条件变量
适用于：
- 线程等待某个条件成立
- 一个线程通知另一个线程“条件已经满足”
- 例如：队列为空时消费者等待，生产后唤醒消费者

### 21.2 什么时候用信号量
适用于：
- 控制多个线程对有限资源的访问
- 需要记录资源数量
- 例如：连接池、缓冲区、停车位模型

### 21.3 什么时候要配合互斥锁
只要有**共享数据结构**（链表、队列、数组、缓冲区等），就建议加互斥锁保护，避免线程安全问题。

---

## 22. 注意事项

1. `pthread_cond_wait()` 一定要放在 `while` 循环中判断条件  
2. 条件变量本身不保护共享资源，仍然需要互斥锁  
3. 信号量能做同步，但不一定能保证共享数据操作安全  
4. 多线程访问链表、队列、缓冲区时，应加互斥锁  
5. `sem_init()` 中的初始值要根据实际资源数量合理设置  
6. 写生产者-消费者模型时，要先明确：
   - 谁负责生产
   - 谁负责消费
   - 共享资源是什么
   - 需要几个信号量
   - 是否需要互斥锁保护临界区

---

## 23. 总结
- **条件变量**更适合“等待条件满足”的场景，本质是线程之间的通知机制
- **信号量**更适合“资源数量控制”的场景，本质是资源计数器
- 在生产者-消费者模型中：
  - 条件变量方案更强调“有无数据”
  - 信号量方案更强调“资源个数”
- 如果多个线程要同时操作共享数据，最好搭配**互斥锁**，确保线程安全
