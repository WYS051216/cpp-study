# 一、线程基础概念

## 1.1 线程概述

线程是**轻量级进程**，英文简称 LWP，Light Weight Process。

在 Linux 环境下，线程本质上是通过轻量级进程实现的。

核心结论：

- 进程是**资源分配的最小单位**
- 线程是**操作系统调度执行的最小单位**
- 一个进程可以包含多个线程
- 同一个进程中的多个线程共享进程资源

---

## 1.2 线程与进程的区别

### 地址空间与资源

进程：

- 拥有独立的虚拟地址空间
- 资源分配相互独立
- 一个进程崩溃，一般不会直接影响其他进程

线程：

- 多个线程共享同一个进程的地址空间
- 多个线程共享代码段、堆区、全局数据区、打开的文件等资源
- 每个线程有自己独立的栈区和寄存器上下文

线程共享的资源：

- 代码段
- 堆区
- 全局数据区
- 打开的文件描述符
- 当前工作目录
- 信号处理方式等进程级资源

线程独有的资源：

- 线程 ID
- 栈空间
- 寄存器上下文
- 调度状态

### 执行与调度

进程：

- 是资源分配单位
- 一个进程对应一个虚拟地址空间

线程：

- 是程序执行单位
- 一个进程中可以有多个线程
- 每个线程都可以被操作系统调度执行

### 线程的优势

线程相比进程有以下优势：

- 更节省系统资源
- 创建和退出速度更快
- 上下文切换开销更小
- 线程间通信更加方便，因为多个线程共享同一进程地址空间

### 对比表

| 对比项 | 进程 | 线程 |
|---|---|---|
| 地址空间 | 独立地址空间 | 共享所属进程的地址空间 |
| 资源 | 独立资源 | 共享进程资源 |
| 栈空间 | 每个进程独立 | 每个线程独立 |
| 调度 | 可以被调度 | 可以被调度 |
| 创建开销 | 较大 | 较小 |
| 通信成本 | 较高 | 较低 |
| 稳定性 | 进程之间影响较小 | 一个线程出错可能影响整个进程 |

## 1.3 线程 ID

线程 ID 的类型是：

```c
pthread_t
```

作用：

- 用于唯一标识一个线程
- 创建线程、回收线程、分离线程、取消线程时都需要用到线程 ID

获取当前线程 ID：

```c
pthread_t pthread_self(void);
```

示例：

```c
#include <stdio.h>
#include <pthread.h>

int main()
{
    pthread_t tid = pthread_self();

    printf("当前线程 ID: %lu\n", (unsigned long)tid);

    return 0;
}
```

注意：

`pthread_t` 的底层实现不一定是普通整数，所以打印时通常可以强制转换为 `unsigned long`：

```c
printf("%lu\n", (unsigned long)pthread_self());
```

所属类别：

> 线程基础概念类

## 1.4 线程 ID 比较

线程 ID 类型是 `pthread_t`，它的底层实现不一定是整数，因此不推荐直接用 `==` 比较。

推荐使用：

```c
int pthread_equal(pthread_t t1, pthread_t t2);
```

返回值：

| 返回值 | 含义 |
|---|---|
| 非 0 | 两个线程 ID 相等 |
| 0 | 两个线程 ID 不相等 |

示例：

```c
#include <stdio.h>
#include <pthread.h>

int main()
{
    pthread_t tid1 = pthread_self();
    pthread_t tid2 = pthread_self();

    if (pthread_equal(tid1, tid2))
    {
        printf("两个线程ID相等\n");
    }
    else
    {
        printf("两个线程ID不相等\n");
    }

    return 0;
}
```

## 1.5 编译线程程序

Linux 下编译 pthread 程序时，需要链接线程库。

推荐写法：

```bash
gcc test.c -o test -pthread
```

也可以写：

```bash
gcc test.c -o test -lpthread
```

更推荐使用 `-pthread`。

原因：

- `-pthread` 不仅会链接 pthread 库
- 还会设置线程相关的编译选项
- 可移植性和语义更好
