# 一、线程基础概念

## 1.1 线程概述

线程是**轻量级进程**，英文简称 LWP，Light Weight Process。

在 Linux 环境下，线程本质上是通过轻量级进程实现的。

核心结论：

- 进程是**资源分配的最小单位**
- 线程是**操作系统调度执行的最小单位**
- 一个进程可以包含多个线程
- 同一个进程中的多个线程共享进程资源

---

## 1.2 线程与进程的区别

### 地址空间与资源

进程：

- 拥有独立的虚拟地址空间
- 资源分配相互独立
- 一个进程崩溃，一般不会直接影响其他进程

线程：

- 多个线程共享同一个进程的地址空间
- 多个线程共享代码段、堆区、全局数据区、打开的文件等资源
- 每个线程有自己独立的栈区和寄存器上下文

线程共享的资源：

- 代码段
- 堆区
- 全局数据区
- 打开的文件描述符
- 当前工作目录
- 信号处理方式等进程级资源

线程独有的资源：

- 线程 ID
- 栈空间
- 寄存器上下文
- 调度状态

### 执行与调度

进程：

- 是资源分配单位
- 一个进程对应一个虚拟地址空间

线程：

- 是程序执行单位
- 一个进程中可以有多个线程
- 每个线程都可以被操作系统调度执行

### 线程的优势

线程相比进程有以下优势：

- 更节省系统资源
- 创建和退出速度更快
- 上下文切换开销更小
- 线程间通信更加方便，因为多个线程共享同一进程地址空间

### 对比表

| 对比项 | 进程 | 线程 |
|---|---|---|
| 地址空间 | 独立地址空间 | 共享所属进程的地址空间 |
| 资源 | 独立资源 | 共享进程资源 |
| 栈空间 | 每个进程独立 | 每个线程独立 |
| 调度 | 可以被调度 | 可以被调度 |
| 创建开销 | 较大 | 较小 |
| 通信成本 | 较高 | 较低 |
| 稳定性 | 进程之间影响较小 | 一个线程出错可能影响整个进程 |

---

## 1.3 线程 ID

线程 ID 的类型是：

```c
pthread_t
```

作用：

- 用于唯一标识一个线程
- 创建线程、回收线程、分离线程、取消线程时都需要用到线程 ID

获取当前线程 ID：

```c
pthread_t pthread_self(void);
```

示例：

```c
#include <stdio.h>
#include <pthread.h>

int main()
{
    pthread_t tid = pthread_self();

    printf("当前线程 ID: %lu\n", (unsigned long)tid);

    return 0;
}
```

注意：

`pthread_t` 的底层实现不一定是普通整数，所以打印时通常可以强制转换为 `unsigned long`：

```c
printf("%lu\n", (unsigned long)pthread_self());
```

所属类别：

> 线程基础概念类

---

## 1.4 线程 ID 比较

线程 ID 类型是 `pthread_t`，它的底层实现不一定是整数，因此不推荐直接用 `==` 比较。

推荐使用：

```c
int pthread_equal(pthread_t t1, pthread_t t2);
```

返回值：

| 返回值 | 含义 |
|---|---|
| 非 0 | 两个线程 ID 相等 |
| 0 | 两个线程 ID 不相等 |

示例：

```c
#include <stdio.h>
#include <pthread.h>

int main()
{
    pthread_t tid1 = pthread_self();
    pthread_t tid2 = pthread_self();

    if (pthread_equal(tid1, tid2))
    {
        printf("两个线程ID相等\n");
    }
    else
    {
        printf("两个线程ID不相等\n");
    }

    return 0;
}
```

---

## 1.5 编译线程程序

Linux 下编译 pthread 程序时，需要链接线程库。

推荐写法：

```bash
gcc test.c -o test -pthread
```

也可以写：

```bash
gcc test.c -o test -lpthread
```

更推荐使用 `-pthread`。

原因：

- `-pthread` 不仅会链接 pthread 库
- 还会设置线程相关的编译选项
- 可移植性和语义更好

# 二、线程生命周期管理

线程生命周期管理主要包括：

- 线程创建
- 线程退出
- 线程回收
- 线程分离
- 线程取消

---

## 2.1 创建线程 pthread_create

函数原型：

```c
int pthread_create(
    pthread_t *thread,
    const pthread_attr_t *attr,
    void *(*start_routine)(void *),
    void *arg
);
```

参数说明：

| 参数 | 含义 |
|---|---|
| `thread` | 传出参数，用于保存新线程 ID |
| `attr` | 线程属性，一般写 `NULL` |
| `start_routine` | 线程执行函数 |
| `arg` | 传递给线程函数的参数 |

返回值：

| 返回值 | 含义 |
|---|---|
| 0 | 创建成功 |
| 非 0 | 创建失败，返回错误码 |

线程函数格式必须是：

```c
void* func(void* arg);
```

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

void* callback(void* arg)
{
    for (int i = 0; i < 5; i++)
    {
        printf("子线程: i = %d\n", i);
    }

    printf("子线程 ID: %lu\n", (unsigned long)pthread_self());

    return NULL;
}

int main()
{
    pthread_t tid;

    pthread_create(&tid, NULL, callback, NULL);

    for (int i = 0; i < 5; i++)
    {
        printf("主线程: i = %d\n", i);
    }

    printf("主线程 ID: %lu\n", (unsigned long)pthread_self());

    pthread_exit(NULL);

    return 0;
}
```

### 注意事项

1. 线程函数必须是 `void* func(void*)` 格式。
2. `pthread_create()` 创建线程成功后，新线程会和主线程并发执行。
3. 主线程如果直接 `return 0;`，整个进程结束，子线程也会被迫结束。
4. 如果希望主线程退出但进程不结束，可以在主线程中调用：

```c
pthread_exit(NULL);
```

---

## 2.2 线程退出 pthread_exit

函数原型：

```c
void pthread_exit(void *retval);
```

作用：

- 终止当前线程
- 不影响同一进程中的其他线程
- 可以携带一个返回值，该返回值可以被 `pthread_join()` 获取

参数说明：

| 参数 | 含义 |
|---|---|
| `retval` | 线程退出时携带的数据 |
| `NULL` | 不需要返回数据时使用 |

示例：

```c
pthread_exit(NULL);
```

或者：

```c
pthread_exit(&data);
```

### 主线程中使用 pthread_exit 的意义

如果主线程中执行：

```c
return 0;
```

那么整个进程会结束，进程中的所有线程都会结束。

如果主线程中执行：

```c
pthread_exit(NULL);
```

那么只是主线程退出，其他子线程仍然可以继续执行。

---

## 2.3 线程回收 pthread_join

函数原型：

```c
int pthread_join(pthread_t thread, void **retval);
```

作用：

- 阻塞等待指定线程结束
- 回收线程资源
- 获取线程退出时返回的数据

参数说明：

| 参数 | 含义 |
|---|---|
| `thread` | 要等待的目标线程 ID |
| `retval` | 接收线程退出时返回的数据；如果不关心，可设为 `NULL` |

返回值：

| 返回值 | 含义 |
|---|---|
| 0 | 成功 |
| 非 0 | 失败，返回错误码 |

注意：

- 一个线程只能被 `pthread_join()` 回收一次
- 多个线程需要循环 `join`
- 分离线程不能被 `pthread_join()` 回收
- `pthread_join()` 会阻塞当前线程，直到目标线程结束

示例：

```c
pthread_t tid;

pthread_create(&tid, NULL, callback, NULL);

pthread_join(tid, NULL);
```

---

## 2.4 线程分离 pthread_detach

线程默认是**可连接状态**，即 joinable。

可连接线程退出后，其资源不会自动释放，必须由其他线程调用 `pthread_join()` 回收。

如果希望线程结束后由系统自动回收资源，可以将其设置为**分离线程**。

函数原型：

```c
#include <pthread.h>

int pthread_detach(pthread_t thread);
```

参数说明：

| 参数 | 含义 |
|---|---|
| `thread` | 要设置为分离状态的线程 ID |

返回值：

| 返回值 | 含义 |
|---|---|
| 0 | 成功 |
| 非 0 | 失败，返回错误码 |

### 分离线程的特点

- 分离线程结束后，系统会自动回收资源
- 分离线程不能再被 `pthread_join()` 回收
- 线程不能重复分离
- 适用于“不关心返回值，只希望执行任务”的线程

### 注意事项

1. `pthread_detach()` 只是设置线程为分离状态，不会终止线程。
2. 主线程如果直接 `return 0;`，整个进程会结束，所有线程也会结束。
3. 如果希望子线程继续执行，主线程通常应调用：

```c
pthread_exit(NULL);
```

4. 不要返回局部变量地址，局部变量在线程结束后会失效。
5. 分离线程无法通过 `pthread_join()` 获取返回值。

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

struct Test
{
    int num;
    int age;
};

void* callback(void* arg)
{
    struct Test* t = (struct Test*)arg;

    for (int i = 0; i < 5; i++)
    {
        printf("子线程: i = %d\n", i);
    }

    printf("子线程ID: %lu\n", (unsigned long)pthread_self());

    t->num = 100;
    t->age = 6;

    pthread_exit(NULL);
}

int main()
{
    struct Test t;
    pthread_t tid;

    pthread_create(&tid, NULL, callback, &t);

    printf("主线程ID: %lu\n", (unsigned long)pthread_self());

    pthread_detach(tid);

    pthread_exit(NULL);
}
```

## 2.5 线程取消 pthread_cancel

线程取消是指：

> 一个线程向另一个线程发送“取消请求”。

函数原型：

```c
#include <pthread.h>

int pthread_cancel(pthread_t thread);
```

参数说明：

| 参数 | 含义 |
|---|---|
| `thread` | 目标线程 ID |

返回值：

| 返回值 | 含义 |
|---|---|
| 0 | 成功 |
| 非 0 | 失败，返回错误码 |

### 线程取消的本质

`pthread_cancel()` 发出的不是“立即杀死线程”的命令，而是发送一个**取消请求**。

默认情况下线程属性通常为：

- 取消状态：`PTHREAD_CANCEL_ENABLE`
- 取消类型：`PTHREAD_CANCEL_DEFERRED`

这表示：

- 线程允许被取消
- 但只有运行到**取消点**时，才会真正响应取消请求

### 常见取消点

常见取消点包括：

- `read()`
- `write()`
- `sleep()`
- `usleep()`
- `pthread_join()`
- `pthread_cond_wait()`
- `accept()`
- `recv()`

### 设置取消状态

函数原型：

```c
int pthread_setcancelstate(int state, int *oldstate);
```

可选值：

```c
PTHREAD_CANCEL_ENABLE
PTHREAD_CANCEL_DISABLE
```

含义：

| 取消状态 | 含义 |
|---|---|
| `PTHREAD_CANCEL_ENABLE` | 允许线程被取消 |
| `PTHREAD_CANCEL_DISABLE` | 禁止线程被取消 |

### 设置取消类型

函数原型：

```c
int pthread_setcanceltype(int type, int *oldtype);
```

可选值：

```c
PTHREAD_CANCEL_DEFERRED
PTHREAD_CANCEL_ASYNCHRONOUS
```

含义：

| 取消类型 | 含义 |
|---|---|
| `PTHREAD_CANCEL_DEFERRED` | 延迟取消，运行到取消点才取消 |
| `PTHREAD_CANCEL_ASYNCHRONOUS` | 异步取消，可能立即取消 |

一般更推荐使用默认的延迟取消。

### 手动设置取消点

函数原型：

```c
void pthread_testcancel(void);
```

适用于计算密集型线程。

如果线程长时间没有调用 `sleep()`、`read()`、`pthread_cond_wait()` 等取消点函数，可以主动调用：

```c
pthread_testcancel();
```

让线程检查是否有取消请求。

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

void* working(void* arg)
{
    printf("我是子线程, 线程ID: %lu\n", (unsigned long)pthread_self());

    for (int i = 0; i < 15; i++)
    {
        printf("child i: %d\n", i);
        usleep(100000);
    }

    return NULL;
}

int main()
{
    pthread_t tid;

    pthread_create(&tid, NULL, working, NULL);

    printf("子线程创建成功, 线程ID: %lu\n", (unsigned long)tid);
    printf("我是主线程, 线程ID: %lu\n", (unsigned long)pthread_self());

    for (int i = 0; i < 3; i++)
    {
        printf("main i = %d\n", i);
    }

    pthread_cancel(tid);

    pthread_join(tid, NULL);

    return 0;
}
```

### pthread_detach 与 pthread_cancel 的关系

- 分离线程后，不能再 `pthread_join()`
- 如果你需要确认线程是否真正结束，一般不要先 `detach`
- 如果只是想发送取消请求，可以对线程调用 `pthread_cancel()`
- 但分离线程被取消后，无法通过 `join` 获取其结束状态

一般原则：

| 需求 | 推荐方式 |
|---|---|
| 需要获取返回值 | 使用 `pthread_join()` |
| 需要确认线程结束 | 使用 `pthread_join()` |
| 不关心返回值 | 可以使用 `pthread_detach()` |
| 只希望线程后台执行 | 可以使用 `pthread_detach()` |

# 三、线程间数据传递

线程间数据传递主要包括：

- 创建线程时向线程函数传递参数
- 线程退出时向其他线程返回数据
- 主线程通过 `pthread_join()` 获取返回值

---

## 3.1 通过参数向线程传值

`pthread_create()` 的第四个参数 `arg` 可以用来向线程函数传参。

示例：

```c
pthread_create(&tid, NULL, callback, &data);
```

线程函数中接收：

```c
void* callback(void* arg)
{
    struct Test* t = (struct Test*)arg;

    return NULL;
}
```

注意：

- `arg` 是 `void*` 类型
- 传入时通常传地址
- 在线程函数中需要强制类型转换

---

## 3.2 通过 pthread_exit 返回值

线程可以通过 `pthread_exit()` 返回数据：

```c
pthread_exit(ptr);
```

其他线程可以通过 `pthread_join()` 接收：

```c
void *ptr = NULL;

pthread_join(tid, &ptr);
```

注意：

`pthread_exit()` 返回的是一个指针，因此必须保证该指针指向的内存在被访问时仍然有效。

---

## 3.3 使用全局变量传值

这种方式中，子线程修改全局变量，然后通过 `pthread_exit()` 返回全局变量地址。

示例：

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

struct Test
{
    int num;
    int age;
};

struct Test t;

void* callback(void* arg)
{
    t.num = 100;
    t.age = 6;

    pthread_exit(&t);
}

int main()
{
    pthread_t tid;

    pthread_create(&tid, NULL, callback, NULL);

    void *ptr = NULL;

    pthread_join(tid, &ptr);

    struct Test* pt = (struct Test*)ptr;

    printf("num = %d, age = %d\n", pt->num, pt->age);

    return 0;
}
```

特点：

- 全局变量生命周期长
- 线程结束后，全局变量仍然有效
- 可以安全返回全局变量地址

注意：

如果多个线程同时访问这个全局变量，需要考虑线程同步问题。

---

## 3.4 使用主线程栈空间传值

主线程中定义变量，然后把变量地址传给子线程。

示例：

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

struct Test
{
    int num;
    int age;
};

void* callback(void* arg)
{
    struct Test* t = (struct Test*)arg;

    t->num = 100;
    t->age = 6;

    pthread_exit(t);
}

int main()
{
    struct Test t;
    pthread_t tid;

    pthread_create(&tid, NULL, callback, &t);

    void *ptr = NULL;

    pthread_join(tid, &ptr);

    printf("num = %d, age = %d\n", t.num, t.age);

    return 0;
}
```

特点：

- 主线程栈空间中的变量在主线程函数结束前有效
- 子线程可以通过参数修改主线程中的变量
- 主线程通过 `pthread_join()` 等待子线程结束后再访问数据

注意：

主线程中的变量必须在子线程使用期间保持有效。

---

## 3.5 不要返回子线程局部变量地址

错误示例：

```c
void* callback(void* arg)
{
    struct Test t;

    t.num = 100;
    t.age = 6;

    pthread_exit(&t);
}
```

原因：

- `t` 是子线程函数内部的局部变量
- 它存储在线程自己的栈空间中
- 子线程结束后，该栈空间会失效
- 主线程再通过该地址访问数据会产生未定义行为

正确做法：

1. 返回全局变量地址
2. 返回堆区内存地址
3. 返回仍然有效的主线程栈变量地址
4. 在线程结束前通过同步机制拷贝数据


---

# 四、线程同步基础

## 4.1 为什么需要线程同步

多个线程共享同一份数据时，如果同时访问或修改共享资源，可能出现数据不一致。

例如：

```c
int number = 0;
```

多个线程同时执行：

```c
number++;
```

这并不是一个真正的原子操作，它可能包含：

1. 从内存读取 `number`
2. 对值加 1
3. 写回内存

多个线程交叉执行时，就可能造成数据丢失。

常见问题：

- 数据覆盖
- 结果不符合预期
- 链表断裂
- 指针访问非法
- 程序崩溃
- 死锁
- 资源泄露

---

## 4.2 共享资源与临界区

共享资源：

> 多个线程都可能访问的数据或资源。

例如：

- 全局变量
- 堆区数据
- 链表
- 队列
- 数组
- 文件描述符
- 数据库连接
- 网络连接
- 缓冲区

临界区：

> 访问共享资源的代码区域。

示例：

```c
pthread_mutex_lock(&mutex);

/* 临界区：访问共享资源 */

pthread_mutex_unlock(&mutex);
```

线程同步的目的：

> 保证多个线程访问共享资源时，执行顺序是安全、可控的。

---

## 4.3 常见线程同步工具

| 同步工具 | 作用 | 典型场景 |
|---|---|---|
| 互斥锁 mutex | 保护临界区 | 多线程修改共享变量 |
| 读写锁 rwlock | 读共享、写独占 | 读多写少 |
| 条件变量 cond | 等待某个条件成立 | 生产者消费者 |
| 信号量 sem | 控制资源数量 | 缓冲区、连接池 |

---

# 五、互斥锁 mutex

## 5.1 互斥锁作用

互斥锁用于保护临界区资源，避免多个线程同时访问同一份共享数据。

核心特点：

- 同一时刻只允许一个线程持有锁
- 其他线程如果加锁失败，会阻塞等待
- 持有锁的线程访问完共享资源后，需要解锁

---

## 5.2 互斥锁相关 API

### pthread_mutex_init

函数原型：

```c
int pthread_mutex_init(pthread_mutex_t *mutex,
                       const pthread_mutexattr_t *attr);
```

作用：

- 初始化互斥锁

示例：

```c
pthread_mutex_t lock;

pthread_mutex_init(&lock, NULL);
```

---

### pthread_mutex_destroy

函数原型：

```c
int pthread_mutex_destroy(pthread_mutex_t *mutex);
```

作用：

- 销毁互斥锁
- 释放互斥锁相关资源

注意：

销毁互斥锁前，应确保没有线程正在持有或等待该锁。

---

### pthread_mutex_lock

函数原型：

```c
int pthread_mutex_lock(pthread_mutex_t *mutex);
```

作用：

- 加锁
- 如果锁已被占用，则阻塞等待

---

### pthread_mutex_trylock

函数原型：

```c
int pthread_mutex_trylock(pthread_mutex_t *mutex);
```

作用：

- 尝试加锁
- 如果锁可用，则加锁成功
- 如果锁不可用，不阻塞，立即返回错误码

---

### pthread_mutex_unlock

函数原型：

```c
int pthread_mutex_unlock(pthread_mutex_t *mutex);
```

作用：

- 解锁
- 释放当前线程持有的锁

注意：

一般应由持有锁的线程解锁。

---

## 5.3 互斥锁基本流程

```c
pthread_mutex_t lock;

pthread_mutex_init(&lock, NULL);

pthread_mutex_lock(&lock);

/* 访问共享资源 */

pthread_mutex_unlock(&lock);

pthread_mutex_destroy(&lock);
```

核心逻辑：

1. 初始化锁
2. 访问共享资源前加锁
3. 访问共享资源
4. 访问完成后解锁
5. 不再使用时销毁锁


---

## 5.4 互斥锁适用场景

适合：

- 多线程修改同一份数据
- 多线程操作链表、队列、数组
- 需要严格保证同一时刻只有一个线程访问临界区
- 共享资源读写逻辑比较简单

不适合：

- 读操作非常多，写操作很少，并且希望提高读并发性能的场景

这种情况下可以考虑读写锁。


---

# 六、读写锁 rwlock

## 6.1 读写锁作用

读写锁适用于**读多写少**场景。

读写锁允许：

- 多个线程同时读
- 写操作必须独占

也就是说：

- 读读可以并发
- 读写不能并发
- 写写不能并发

---

## 6.2 读写锁特点

读写锁类型：

```c
pthread_rwlock_t
```

特点：

- **读锁**：共享锁，多个线程可以同时持有
- **写锁**：独占锁，同一时刻只允许一个线程持有
- 有线程持有写锁时，其他线程不能加读锁或写锁
- 有线程持有读锁时，其他线程可以继续读，但不能写

适合场景：

- 配置读取
- 缓存读取
- 查询远多于更新的数据结构
- 读操作频繁，写操作较少

不适合场景：

- 写操作频繁
- 临界区非常短，锁开销明显
- 读写比例差不多

---

## 6.3 读写锁相关 API

### pthread_rwlock_init

函数原型：

```c
int pthread_rwlock_init(pthread_rwlock_t *rwlock,
                        const pthread_rwlockattr_t *attr);
```

作用：

- 初始化读写锁

---

### pthread_rwlock_rdlock

函数原型：

```c
int pthread_rwlock_rdlock(pthread_rwlock_t *rwlock);
```

作用：

- 加读锁
- 多个线程可以同时持有读锁

---

### pthread_rwlock_wrlock

函数原型：

```c
int pthread_rwlock_wrlock(pthread_rwlock_t *rwlock);
```

作用：

- 加写锁
- 写锁是独占锁

---

### pthread_rwlock_unlock

函数原型：

```c
int pthread_rwlock_unlock(pthread_rwlock_t *rwlock);
```

作用：

- 解锁
- 无论是读锁还是写锁，都使用这个函数解锁

---

### pthread_rwlock_destroy

函数原型：

```c
int pthread_rwlock_destroy(pthread_rwlock_t *rwlock);
```

作用：

- 销毁读写锁

---

## 6.4 读写锁优先级说明

“写锁优先级一定高于读锁”这种说法并不严谨。

更准确地说：

- POSIX 没有统一规定读写锁一定是读优先还是写优先
- 等待中的读线程和写线程谁先获得锁，取决于具体实现策略

可以确定的是：

- 如果当前有线程持有写锁，其他线程都要等待
- 如果当前有线程持有读锁，其他读线程可以继续读
- 如果当前有线程持有读锁，写线程必须等待

---

## 6.5 读写锁示例代码

### 示例 1：基础读写锁示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

int number = 0;
pthread_rwlock_t rwlock;

void* read_task(void* arg)
{
    for (int i = 0; i < 5; i++)
    {
        pthread_rwlock_rdlock(&rwlock);

        printf("读线程 %lu, number = %d\n",
               (unsigned long)pthread_self(), number);

        pthread_rwlock_unlock(&rwlock);

        usleep(100000);
    }

    return NULL;
}

void* write_task(void* arg)
{
    for (int i = 0; i < 5; i++)
    {
        pthread_rwlock_wrlock(&rwlock);

        number++;

        printf("写线程 %lu, number = %d\n",
               (unsigned long)pthread_self(), number);

        pthread_rwlock_unlock(&rwlock);

        usleep(150000);
    }

    return NULL;
}

int main()
{
    pthread_t r1, r2, w1;

    pthread_rwlock_init(&rwlock, NULL);

    pthread_create(&r1, NULL, read_task, NULL);
    pthread_create(&r2, NULL, read_task, NULL);
    pthread_create(&w1, NULL, write_task, NULL);

    pthread_join(r1, NULL);
    pthread_join(r2, NULL);
    pthread_join(w1, NULL);

    pthread_rwlock_destroy(&rwlock);

    return 0;
}
```

### 示例 2：多个读线程和多个写线程

```c
#define _XOPEN_SOURCE 500

#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <time.h>

#define MAX 50

int number;

pthread_rwlock_t rwlock;

void* read_num(void* arg)
{
    for (int i = 0; i < MAX; i++)
    {
        pthread_rwlock_rdlock(&rwlock);

        printf("Thread read, id = %lu, number = %d\n",
               (unsigned long)pthread_self(), number);

        pthread_rwlock_unlock(&rwlock);

        usleep(rand() % 200);
    }

    return NULL;
}

void* write_num(void* arg)
{
    for (int i = 0; i < MAX; i++)
    {
        pthread_rwlock_wrlock(&rwlock);

        int cur = number;
        cur++;
        number = cur;

        printf("Thread write, id = %lu, number = %d\n",
               (unsigned long)pthread_self(), number);

        pthread_rwlock_unlock(&rwlock);

        usleep(rand() % 10);
    }

    return NULL;
}

int main()
{
    srand(time(NULL));

    pthread_t p1[5], p2[3];

    pthread_rwlock_init(&rwlock, NULL);

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&p1[i], NULL, read_num, NULL);
    }

    for (int i = 0; i < 3; i++)
    {
        pthread_create(&p2[i], NULL, write_num, NULL);
    }

    for (int i = 0; i < 3; i++)
    {
        pthread_join(p2[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(p1[i], NULL);
    }

    pthread_rwlock_destroy(&rwlock);

    printf("Final number = %d\n", number);

    return 0;
}
```

# 七、条件变量 cond

## 7.1 条件变量作用

条件变量通常和**互斥锁**一起使用，用于解决：

- 当某个条件不满足时，线程阻塞等待
- 当条件满足后，再唤醒线程继续执行

在生产者消费者模型中：

- 当缓冲区或链表为空时，消费者阻塞等待
- 当生产者生产数据后，唤醒消费者继续消费

条件变量本质上是：

> 等待条件成立的机制。

条件变量类型：

```c
pthread_cond_t
```

---

## 7.2 条件变量相关 API

### pthread_cond_init

函数原型：

```c
int pthread_cond_init(pthread_cond_t *cond,
                      const pthread_condattr_t *attr);
```

作用：

- 初始化条件变量

---

### pthread_cond_destroy

函数原型：

```c
int pthread_cond_destroy(pthread_cond_t *cond);
```

作用：

- 销毁条件变量

---

### pthread_cond_wait

函数原型：

```c
int pthread_cond_wait(pthread_cond_t *cond,
                      pthread_mutex_t *mutex);
```

作用：

- 阻塞等待条件变量
- 等待期间会释放互斥锁
- 被唤醒后会重新加锁

---

### pthread_cond_timedwait

函数原型：

```c
int pthread_cond_timedwait(pthread_cond_t *cond,
                           pthread_mutex_t *mutex,
                           const struct timespec *abstime);
```

作用：

- 带超时时间的条件变量等待

---

### pthread_cond_signal

函数原型：

```c
int pthread_cond_signal(pthread_cond_t *cond);
```

作用：

- 唤醒至少一个等待在该条件变量上的线程

---

### pthread_cond_broadcast

函数原型：

```c
int pthread_cond_broadcast(pthread_cond_t *cond);
```

作用：

- 唤醒所有等待在该条件变量上的线程

---

## 7.3 pthread_cond_wait 的特点

`pthread_cond_wait()` 会自动完成三个动作：

1. 释放互斥锁
2. 阻塞等待条件满足
3. 被唤醒后重新加锁

典型结构：

```c
pthread_mutex_lock(&mutex);

while (条件不满足)
{
    pthread_cond_wait(&cond, &mutex);
}

/* 条件满足，访问共享资源 */

pthread_mutex_unlock(&mutex);
```

注意：

条件变量本身不保护共享资源，仍然需要互斥锁。

---

## 7.4 为什么使用 while 而不是 if

推荐写法：

```c
while (head == NULL)
{
    pthread_cond_wait(&cond, &mutex);
}
```

不推荐写法：

```c
if (head == NULL)
{
    pthread_cond_wait(&cond, &mutex);
}
```

原因：

1. 可能出现虚假唤醒。
2. 多个消费者可能同时被唤醒。
3. 某个消费者拿到资源后，其他消费者醒来时资源可能已经没了。
4. 被唤醒不代表条件一定满足，必须重新检查。

---

## 7.5 条件变量版生产者消费者模型

### 实现思路

- 使用链表作为共享资源
- 生产者负责创建节点并插入链表
- 消费者负责从链表中取出节点
- 使用互斥锁保护链表
- 使用条件变量通知消费者有数据可消费

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

pthread_cond_t cond;
pthread_mutex_t mutex;

typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

void* producer(void* arg)
{
    while (1)
    {
        pthread_mutex_lock(&mutex);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), newNode->number);

        pthread_mutex_unlock(&mutex);

        pthread_cond_broadcast(&cond);

        sleep(rand() % 3);
    }

    return NULL;
}

void* consumer(void* arg)
{
    while (1)
    {
        pthread_mutex_lock(&mutex);

        while (head == NULL)
        {
            pthread_cond_wait(&cond, &mutex);
        }

        Node* node = head;
        head = head->next;

        printf("消费者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), node->number);

        free(node);

        pthread_mutex_unlock(&mutex);

        sleep(rand() % 3);
    }

    return NULL;
}

int main()
{
    pthread_mutex_init(&mutex, NULL);
    pthread_cond_init(&cond, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);
    pthread_cond_destroy(&cond);

    return 0;
}
```

### 使用条件变量时的注意事项

1. 条件变量本身不保护共享资源，仍需要互斥锁。
2. `pthread_cond_wait()` 一定放在 `while` 循环里判断条件。
3. 先加锁，再检查条件，再决定是否等待。
4. 修改共享条件后再发信号，逻辑更清晰。
5. `signal` 适合唤醒一个线程。
6. `broadcast` 适合唤醒多个线程。

# 八、信号量 sem

## 8.1 信号量作用

信号量用于多线程、多任务同步。

一个线程完成某个动作后，可以通过信号量通知其他线程继续执行。

信号量本质上强调：

> 资源计数。

可以理解为：

- 资源数量大于 0：线程可以继续执行
- 资源数量等于 0：线程需要阻塞等待

信号量头文件：

```c
#include <semaphore.h>
```

信号量类型：

```c
sem_t
```

---

## 8.2 信号量与互斥锁、条件变量的区别

互斥锁：

- 保护临界区
- 保证同一时刻只有一个线程访问共享资源

条件变量：

- 用于等待某个条件成立
- 通常配合互斥锁使用

信号量：

- 用于表示资源数量
- 控制线程能否继续执行

注意：

- 信号量可以完成同步
- 但它不能完全代替互斥锁
- 多线程访问共享数据时，通常仍要配合互斥锁使用

---

## 8.3 信号量相关 API

### sem_init

函数原型：

```c
int sem_init(sem_t *sem, int pshared, unsigned int value);
```

参数说明：

| 参数 | 含义 |
|---|---|
| `sem` | 信号量地址 |
| `pshared` | `0` 表示线程间同步，非 `0` 表示进程间同步 |
| `value` | 初始资源数 |

---

### sem_destroy

函数原型：

```c
int sem_destroy(sem_t *sem);
```

作用：

- 销毁信号量

---

### sem_wait

函数原型：

```c
int sem_wait(sem_t *sem);
```

作用：

- 申请资源
- 资源数大于 `0` 时，获取资源并减 `1`
- 资源数等于 `0` 时，线程阻塞

---

### sem_trywait

函数原型：

```c
int sem_trywait(sem_t *sem);
```

作用：

- 尝试申请资源
- 不阻塞
- 如果资源不可用，立即返回错误

---

### sem_post

函数原型：

```c
int sem_post(sem_t *sem);
```

作用：

- 释放资源
- 资源数加 `1`
- 如果有线程阻塞在该信号量上，可能唤醒其中一个线程

---

### sem_getvalue

函数原型：

```c
int sem_getvalue(sem_t *sem, int *sval);
```

作用：

- 获取当前信号量资源个数

## 8.4 单资源生产者消费者模型

### 实现思路

定义两个信号量：

```c
sem_t semp;
sem_t semc;
```

含义：

| 信号量 | 含义 |
|---|---|
| `semp` | 生产者信号量 |
| `semc` | 消费者信号量 |

初始值：

```c
semp = 1;
semc = 0;
```

表示：

- 一开始允许生产
- 一开始不允许消费
- 生产者生产一个资源后，通知消费者
- 消费者消费一个资源后，再通知生产者继续生产

注意：

这种写法适合单资源同步场景，但如果访问共享链表，严格来说仍建议加锁。

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <semaphore.h>

sem_t semp;
sem_t semc;

pthread_mutex_t mutex;

typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

void* producer(void* arg)
{
    while (1)
    {
        sem_wait(&semp);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), newNode->number);

        sem_post(&semc);

        sleep(rand() % 3);
    }

    return NULL;
}

void* consumer(void* arg)
{
    while (1)
    {
        sem_wait(&semc);

        Node* node = head;

        printf("消费者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), node->number);

        head = head->next;

        free(node);

        sem_post(&semp);

        sleep(rand() % 3);
    }

    return NULL;
}

int main()
{
    sem_init(&semp, 0, 1);
    sem_init(&semc, 0, 0);

    pthread_mutex_init(&mutex, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);

    sem_destroy(&semc);
    sem_destroy(&semp);

    return 0;
}
```

### 代码问题说明

这段代码表达的逻辑是：

- 生产者生产一个资源后，通知消费者
- 消费者消费一个资源后，再通知生产者继续生产

但问题是：

- 对共享链表 `head` 的操作没有加锁保护
- 多线程环境下可能出现线程安全问题

因此更推荐使用：

> 信号量 + 互斥锁

## 8.5 多线程生产者消费者模型：信号量 + 互斥锁

### 实现思路

在多生产者、多消费者模型中：

- 用**信号量**控制是否可以生产 / 消费
- 用**互斥锁**保护共享链表

初始值设置：

```c
sem_init(&semp, 0, 5);
sem_init(&semc, 0, 0);
```

含义：

| 信号量 | 初始值 | 含义 |
|---|---|---|
| `semp` | 5 | 最多允许生产 5 个资源，即空位数量 |
| `semc` | 0 | 开始时没有资源可消费，即产品数量 |

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <semaphore.h>

sem_t semp;
sem_t semc;

pthread_mutex_t mutex;

typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

void* producer(void* arg)
{
    while (1)
    {
        sem_wait(&semp);

        pthread_mutex_lock(&mutex);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), newNode->number);

        pthread_mutex_unlock(&mutex);

        sem_post(&semc);
    }

    return NULL;
}

void* consumer(void* arg)
{
    while (1)
    {
        sem_wait(&semc);

        pthread_mutex_lock(&mutex);

        Node* node = head;

        printf("消费者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), node->number);

        head = head->next;

        free(node);

        pthread_mutex_unlock(&mutex);

        sem_post(&semp);

        sleep(2);
    }

    return NULL;
}

int main()
{
    sem_init(&semp, 0, 5);
    sem_init(&semc, 0, 0);

    pthread_mutex_init(&mutex, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);

    sem_destroy(&semc);
    sem_destroy(&semp);

    return 0;
}
```

### 代码分析

生产者：

```c
sem_wait(&semp);
```

含义：

- 生产前先申请一个空位
- 如果没有空位，则阻塞等待

```c
pthread_mutex_lock(&mutex);
```

含义：

- 加锁保护共享链表

```c
sem_post(&semc);
```

含义：

- 生产完成后，通知消费者可以消费

消费者：

```c
sem_wait(&semc);
```

含义：

- 消费前先检查是否有产品
- 如果没有产品，则阻塞等待

```c
pthread_mutex_lock(&mutex);
```

含义：

- 加锁保护共享链表

```c
sem_post(&semp);
```

含义：

- 消费完成后，释放一个空位
- 通知生产者可以继续生产

# 九、生产者消费者模型总结

## 9.1 生产者消费者模型的组成

生产者消费者模型通常包含：

- 生产者线程
- 消费者线程
- 共享缓冲区
- 同步机制

共享缓冲区可以是：

- 链表
- 队列
- 数组
- 环形缓冲区

同步机制可以是：

- 互斥锁 + 条件变量
- 信号量 + 互斥锁

---

## 9.2 条件变量方案

条件变量方案更强调：

> 条件是否满足。

例如：

- 队列是否为空
- 是否有任务
- 是否可以继续执行

典型逻辑：

消费者：

```c
pthread_mutex_lock(&mutex);

while (队列为空)
{
    pthread_cond_wait(&cond, &mutex);
}

/* 消费数据 */

pthread_mutex_unlock(&mutex);
```

生产者：

```c
pthread_mutex_lock(&mutex);

/* 生产数据 */

pthread_mutex_unlock(&mutex);

pthread_cond_signal(&cond);
```

适合：

- 等待某个事件发生
- 等待某个状态改变
- 等待队列不为空

---

## 9.3 信号量方案

信号量方案更强调：

> 资源数量。

例如：

- 缓冲区有几个空位
- 当前有几个产品
- 连接池还有几个连接
- 停车场还有几个车位

典型逻辑：

生产者：

```c
sem_wait(&empty);

pthread_mutex_lock(&mutex);

/* 生产数据 */

pthread_mutex_unlock(&mutex);

sem_post(&full);
```

消费者：

```c
sem_wait(&full);

pthread_mutex_lock(&mutex);

/* 消费数据 */

pthread_mutex_unlock(&mutex);

sem_post(&empty);
```

适合：

- 控制有限资源数量
- 固定容量缓冲区
- 连接池
- 停车位模型

# 十、同步工具对比与选型

## 10.1 pthread_join 与 pthread_detach 对比

| 对比项 | pthread_join | pthread_detach |
|---|---|---|
| 是否等待线程结束 | 是 | 否 |
| 是否回收资源 | 是 | 线程结束后系统自动回收 |
| 是否能获取返回值 | 能 | 不能 |
| 是否阻塞调用线程 | 会阻塞 | 不阻塞 |
| 适合场景 | 需要结果、需要确认线程结束 | 不关心结果，只希望自动释放资源 |

记忆：

> 要结果，用 join；不要结果，用 detach。

---

## 10.2 pthread_detach 与 pthread_cancel 对比

| 对比项 | pthread_detach | pthread_cancel |
|---|---|---|
| 作用 | 设置线程为分离状态 | 向线程发送取消请求 |
| 是否终止线程 | 不会 | 可能会，但不是立即 |
| 是否回收资源 | 线程结束后自动回收 | 本身不负责回收 |
| 是否能 join | 分离后不能 join | 如果未分离，仍可 join |
| 典型用途 | 不关心线程返回值 | 请求线程提前结束 |

注意：

- `pthread_detach()` 不是结束线程
- `pthread_cancel()` 不是立即杀死线程
- 分离线程后无法通过 `pthread_join()` 确认线程是否结束

---

## 10.3 互斥锁与读写锁对比

| 锁类型 | 读操作 | 写操作 | 适用场景 |
|---|---|---|---|
| 互斥锁 mutex | 读也独占 | 写也独占 | 读写都频繁，逻辑简单 |
| 读写锁 rwlock | 可多个线程同时读 | 写时必须独占 | 读多写少 |

更详细对比：

| 对比项 | 互斥锁 mutex | 读写锁 rwlock |
|---|---|---|
| 并发读 | 不支持 | 支持 |
| 写操作 | 独占 | 独占 |
| 使用复杂度 | 较低 | 稍高 |
| 性能优势 | 简单稳定 | 读多写少时性能更好 |
| 适合场景 | 普通共享资源 | 查询多、修改少 |

记忆：

> 普通共享资源用 mutex，读多写少用 rwlock。

---

## 10.4 条件变量与信号量对比

### 相同点

- 都能用于线程同步
- 都能实现生产者消费者模型
- 都能让线程在条件不满足时阻塞

### 不同点

| 对比项 | 条件变量 | 信号量 |
|---|---|---|
| 本质 | 等待某个条件成立 | 表示资源数量 |
| 是否有计数功能 | 没有 | 有 |
| 是否需要互斥锁 | 必须配合互斥锁 | 一般建议配合互斥锁 |
| 典型用途 | 等待事件发生 | 控制资源访问数量 |
| 线程唤醒方式 | `signal` / `broadcast` | `post` |
| 典型场景 | 队列为空时等待 | 缓冲区容量控制 |

记忆：

> 等条件，用条件变量；数资源，用信号量。

---

## 10.5 什么时候用条件变量

适用于：

- 线程等待某个条件成立
- 一个线程通知另一个线程“条件已经满足”
- 队列为空时消费者等待
- 任务队列中没有任务时工作线程等待
- 某个状态没达到时阻塞等待

例子：

- 生产者消费者模型
- 线程池任务队列
- 等待配置加载完成
- 等待某个标志位改变

---

## 10.6 什么时候用信号量

适用于：

- 控制多个线程对有限资源的访问
- 需要记录资源数量
- 资源数量本身很重要

例子：

- 连接池
- 缓冲区
- 停车位模型
- 固定数量任务槽
- 限制同时访问某资源的线程数量

---

## 10.7 什么时候需要配合互斥锁

只要有共享数据结构，就建议加互斥锁保护。

例如：

- 链表
- 队列
- 数组
- 缓冲区
- 全局结构体
- 共享计数器

原因：

- 信号量可以控制资源数量
- 条件变量可以控制等待和唤醒
- 但它们本身不一定能保护共享数据结构的内部一致性
