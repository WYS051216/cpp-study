# 八、信号量 sem

## 8.1 信号量作用

信号量用于多线程、多任务同步。

一个线程完成某个动作后，可以通过信号量通知其他线程继续执行。

信号量本质上强调：

> 资源计数。

可以理解为：

- 资源数量大于 0：线程可以继续执行
- 资源数量等于 0：线程需要阻塞等待

信号量头文件：

```c
#include <semaphore.h>
```

信号量类型：

```c
sem_t
```

---

## 8.2 信号量与互斥锁、条件变量的区别

互斥锁：

- 保护临界区
- 保证同一时刻只有一个线程访问共享资源

条件变量：

- 用于等待某个条件成立
- 通常配合互斥锁使用

信号量：

- 用于表示资源数量
- 控制线程能否继续执行

注意：

- 信号量可以完成同步
- 但它不能完全代替互斥锁
- 多线程访问共享数据时，通常仍要配合互斥锁使用

---

## 8.3 信号量相关 API

### sem_init

函数原型：

```c
int sem_init(sem_t *sem, int pshared, unsigned int value);
```

参数说明：

| 参数 | 含义 |
|---|---|
| `sem` | 信号量地址 |
| `pshared` | `0` 表示线程间同步，非 `0` 表示进程间同步 |
| `value` | 初始资源数 |

---

### sem_destroy

函数原型：

```c
int sem_destroy(sem_t *sem);
```

作用：

- 销毁信号量

---

### sem_wait

函数原型：

```c
int sem_wait(sem_t *sem);
```

作用：

- 申请资源
- 资源数大于 `0` 时，获取资源并减 `1`
- 资源数等于 `0` 时，线程阻塞

---

### sem_trywait

函数原型：

```c
int sem_trywait(sem_t *sem);
```

作用：

- 尝试申请资源
- 不阻塞
- 如果资源不可用，立即返回错误

---

### sem_post

函数原型：

```c
int sem_post(sem_t *sem);
```

作用：

- 释放资源
- 资源数加 `1`
- 如果有线程阻塞在该信号量上，可能唤醒其中一个线程

---

### sem_getvalue

函数原型：

```c
int sem_getvalue(sem_t *sem, int *sval);
```

作用：

- 获取当前信号量资源个数

---

## 8.4 单资源生产者消费者模型

### 实现思路

定义两个信号量：

```c
sem_t semp;
sem_t semc;
```

含义：

| 信号量 | 含义 |
|---|---|
| `semp` | 生产者信号量 |
| `semc` | 消费者信号量 |

初始值：

```c
semp = 1;
semc = 0;
```

表示：

- 一开始允许生产
- 一开始不允许消费
- 生产者生产一个资源后，通知消费者
- 消费者消费一个资源后，再通知生产者继续生产

注意：

这种写法适合单资源同步场景，但如果访问共享链表，严格来说仍建议加锁。

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <semaphore.h>

sem_t semp;
sem_t semc;

pthread_mutex_t mutex;

typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

void* producer(void* arg)
{
    while (1)
    {
        sem_wait(&semp);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), newNode->number);

        sem_post(&semc);

        sleep(rand() % 3);
    }

    return NULL;
}

void* consumer(void* arg)
{
    while (1)
    {
        sem_wait(&semc);

        Node* node = head;

        printf("消费者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), node->number);

        head = head->next;

        free(node);

        sem_post(&semp);

        sleep(rand() % 3);
    }

    return NULL;
}

int main()
{
    sem_init(&semp, 0, 1);
    sem_init(&semc, 0, 0);

    pthread_mutex_init(&mutex, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);

    sem_destroy(&semc);
    sem_destroy(&semp);

    return 0;
}
```

### 代码问题说明

这段代码表达的逻辑是：

- 生产者生产一个资源后，通知消费者
- 消费者消费一个资源后，再通知生产者继续生产

但问题是：

- 对共享链表 `head` 的操作没有加锁保护
- 多线程环境下可能出现线程安全问题

因此更推荐使用：

> 信号量 + 互斥锁

## 8.5 多线程生产者消费者模型：信号量 + 互斥锁

### 实现思路

在多生产者、多消费者模型中：

- 用**信号量**控制是否可以生产 / 消费
- 用**互斥锁**保护共享链表

初始值设置：

```c
sem_init(&semp, 0, 5);
sem_init(&semc, 0, 0);
```

含义：

| 信号量 | 初始值 | 含义 |
|---|---|---|
| `semp` | 5 | 最多允许生产 5 个资源，即空位数量 |
| `semc` | 0 | 开始时没有资源可消费，即产品数量 |

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>
#include <semaphore.h>

sem_t semp;
sem_t semc;

pthread_mutex_t mutex;

typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

void* producer(void* arg)
{
    while (1)
    {
        sem_wait(&semp);

        pthread_mutex_lock(&mutex);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), newNode->number);

        pthread_mutex_unlock(&mutex);

        sem_post(&semc);
    }

    return NULL;
}

void* consumer(void* arg)
{
    while (1)
    {
        sem_wait(&semc);

        pthread_mutex_lock(&mutex);

        Node* node = head;

        printf("消费者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), node->number);

        head = head->next;

        free(node);

        pthread_mutex_unlock(&mutex);

        sem_post(&semp);

        sleep(2);
    }

    return NULL;
}

int main()
{
    sem_init(&semp, 0, 5);
    sem_init(&semc, 0, 0);

    pthread_mutex_init(&mutex, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);

    sem_destroy(&semc);
    sem_destroy(&semp);

    return 0;
}
```

### 代码分析

生产者：

```c
sem_wait(&semp);
```

含义：

- 生产前先申请一个空位
- 如果没有空位，则阻塞等待

```c
pthread_mutex_lock(&mutex);
```

含义：

- 加锁保护共享链表

```c
sem_post(&semc);
```

含义：

- 生产完成后，通知消费者可以消费

消费者：

```c
sem_wait(&semc);
```

含义：

- 消费前先检查是否有产品
- 如果没有产品，则阻塞等待

```c
pthread_mutex_lock(&mutex);
```

含义：

- 加锁保护共享链表

```c
sem_post(&semp);
```

含义：

- 消费完成后，释放一个空位
- 通知生产者可以继续生产
