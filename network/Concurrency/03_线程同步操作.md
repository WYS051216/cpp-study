# Linux 线程控制笔记

## 6. 线程的分离

线程默认是**可连接（joinable）**的。  
可连接线程退出后，其资源不会自动释放，必须由其他线程调用 `pthread_join()` 回收。

如果希望线程结束后由系统自动回收资源，可以将其设置为**分离线程（detached thread）**。

### 6.1 `pthread_detach()` 函数
```c
#include <pthread.h>
int pthread_detach(pthread_t thread);
```

参数说明：
- `thread`：要设置为分离状态的线程 ID

返回值：
- 成功返回 `0`
- 失败返回错误码

### 6.2 分离线程的特点
- 分离线程结束后，系统会自动回收资源
- 分离线程不能再被 `pthread_join()` 回收
- 线程不能重复分离
- 适用于“不关心返回值，只希望执行任务”的线程

### 6.3 注意事项
1. `pthread_detach()` 只是设置线程为分离状态，不会终止线程  
2. 主线程如果直接 `return 0;`，整个进程会结束，所有线程也会结束  
3. 如果希望子线程继续执行，主线程通常应调用 `pthread_exit(NULL);`  
4. 不要返回局部变量地址，局部变量在线程结束后会失效  

### 6.4 示例代码
```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

struct Test
{
    int num;
    int age;
};

void* callback(void* arg)
{
    struct Test* t = (struct Test*)arg;

    for (int i = 0; i < 5; i++)
    {
        printf("子线程: i = %d\n", i);
    }

    printf("子线程ID: %lu\n", (unsigned long)pthread_self());

    t->num = 100;
    t->age = 6;

    pthread_exit(NULL);
}

int main()
{
    struct Test t;
    pthread_t tid;

    pthread_create(&tid, NULL, callback, &t);
    printf("主线程ID: %lu\n", (unsigned long)pthread_self());

    pthread_detach(tid);

    pthread_exit(NULL);
}
```

## 7. 线程取消

线程取消是指：一个线程向另一个线程发送“取消请求”。

### 7.1 `pthread_cancel()` 函数
```c
#include <pthread.h>
int pthread_cancel(pthread_t thread);
```

参数说明：
- `thread`：目标线程 ID

返回值：
- 成功返回 `0`
- 失败返回错误码

### 7.2 线程取消的本质
`pthread_cancel()` 发出的不是“立即杀死线程”的命令，而是发送一个**取消请求**。

默认情况下线程属性通常为：
- 取消状态：`PTHREAD_CANCEL_ENABLE`
- 取消类型：`PTHREAD_CANCEL_DEFERRED`

这表示：
- 线程允许被取消
- 但只有运行到**取消点（cancellation point）**时，才会真正响应取消请求

常见取消点：
- `read()`
- `write()`
- `sleep()`
- `usleep()`
- `pthread_join()`
- `pthread_cond_wait()`
- `accept()`
- `recv()`

### 7.3 相关函数补充

#### 7.3.1 设置取消状态
```c
int pthread_setcancelstate(int state, int *oldstate);
```

可选值：
- `PTHREAD_CANCEL_ENABLE`
- `PTHREAD_CANCEL_DISABLE`

#### 7.3.2 设置取消类型
```c
int pthread_setcanceltype(int type, int *oldtype);
```

可选值：
- `PTHREAD_CANCEL_DEFERRED`
- `PTHREAD_CANCEL_ASYNCHRONOUS`

#### 7.3.3 手动设置取消点
```c
void pthread_testcancel(void);
```

适用于计算密集型线程，在没有明显取消点时主动检查取消请求。

### 7.4 示例代码
```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

void* working(void* arg)
{
    printf("我是子线程, 线程ID: %lu\n", (unsigned long)pthread_self());

    for (int i = 0; i < 15; i++)
    {
        printf("child i: %d\n", i);
        usleep(100000);
    }

    return NULL;
}

int main()
{
    pthread_t tid;
    pthread_create(&tid, NULL, working, NULL);

    printf("子线程创建成功, 线程ID: %lu\n", (unsigned long)tid);
    printf("我是主线程, 线程ID: %lu\n", (unsigned long)pthread_self());

    for (int i = 0; i < 3; i++)
    {
        printf("main i = %d\n", i);
    }

    pthread_cancel(tid);
    pthread_join(tid, NULL);

    return 0;
}
```

### 7.5 `pthread_detach()` 与 `pthread_cancel()` 的关系
- 分离线程后，不能再 `pthread_join()`
- 如果你需要确认线程是否真正结束，一般不要先 `detach`
- 一般原则：
  - 需要获取返回值 / 确认结束：用 `pthread_join()`
  - 不关心返回值 / 自动回收：用 `pthread_detach()`
