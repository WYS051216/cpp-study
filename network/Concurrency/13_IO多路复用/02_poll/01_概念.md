# IO多路复用：poll函数学习笔记

## 一、 poll与select的区别与联系

1. **底层机制相同**：内核对于 `poll` 和 `select` 对应文件描述符的检测，都是以**线性轮询**的方式进行，根据描述符的状态进行相应的处理（时间复杂度为 O(n)）。
2. **拷贝开销大**：`poll` 和 `select` 监控的文件描述符集合，都会在检测过程中**频繁地进行用户区和内核区之间的数据拷贝**。随着文件描述符数量的增加，其性能开销也会线性增加，从而导致在海量并发场景下效率越来越低。
3. **连接数上限不同**：`select` 检测的文件描述符个数存在硬性上限（默认通常是 1024）；而 `poll` 使用结构体数组，**没有最大文件描述符数量的硬性限制**（仅受限于系统物理内存和单个进程能打开的最大文件数限制）。
4. **跨平台支持**：`select` 拥有良好的移植性，可以跨平台使用（Windows, Linux, macOS等均支持）；而 `poll` 主要是 POSIX 标准，基本**只能在 Linux/类UNIX 平台上使用**。

## 二、 poll函数核心结构体与原型解析

### 1. pollfd 结构体定义

`poll` 将原本 `select` 中的文件描述符和事件分离开来，结构更清晰，且避免了每次调用都需要重置集合的麻烦。

```c
struct pollfd {
    int   fd;       // 委托内核检测的文件描述符
    short events;   // 委托内核检测文件描述符的什么事件 (例如：POLLIN 读事件, POLLOUT 写事件)
    short revents;  // 文件描述符实际发生的事件 --> 由内核在返回时修改并传出
};

// 使用示例：创建一个能容纳 100 个描述符的监控名单
// struct pollfd myfd[100]; 
```

### 2. poll 函数原型及参数详解

```c
#include <poll.h>

int poll(struct pollfd *fds, nfds_t nfds, int timeout);
```

**参数描述补充：**

* **`fds`**（输入/输出参数）：
  * **描述**：一个指向 `struct pollfd` 结构体数组的指针。
  * **作用**：你需要在这个数组中填入所有你想要内核帮忙监控的文件描述符，以及你关心的事件（赋值给 `events`）。内核检测完毕后，会将实际发生的事件写在 `revents` 里供你读取。

* **`nfds`**（输入参数）：
  * **描述**：`nfds_t` 类型（通常是无符号整型），表示传入的 `fds` 数组中**实际有效元素的个数**。
  * **作用**：告诉内核只需要轮询数组的前 `nfds` 个元素即可，不需要遍历整个系统所有的描述符，从而提高扫描效率。

* **`timeout`**（输入参数）：
  * **描述**：指定 `poll` 函数的阻塞等待时间，**单位为毫秒（ms）**。
  * **作用**：控制 `poll` 的阻塞行为，具体取值如下：
    * **`timeout = -1`**：**一直阻塞**。程序会死等，直到 `fds` 数组中至少有一个文件描述符触发了关心的事件，才会解除阻塞往下执行。
    * **`timeout = 0`**：**非阻塞**。立刻进行一次检测并返回，无论是否有事件发生都不会停留。
    * **`timeout > 0`**：**限时阻塞**。例如设为 1000（1秒），如果在 1 秒内有事件发生则立即返回；如果等了 1 秒依然没有任何动静，则超时强制返回 0。

**返回值说明：**
* `> 0`：成功，返回数组中其 `revents` 字段被设置为非零值的文件描述符的总个数（即就绪的文件描述符数量）。
* `== 0`：超时，在设定的 `timeout` 时间内没有发生任何指定的事件。
* `-1`：调用失败，通常是因为收到了信号中断或参数错误，系统会设置全局变量 `errno`。
