# 四、线程同步基础

## 4.1 为什么需要线程同步

多个线程共享同一份数据时，如果同时访问或修改共享资源，可能出现数据不一致。

例如：

```c
int number = 0;
```

多个线程同时执行：

```c
number++;
```

这并不是一个真正的原子操作，它可能包含：

1. 从内存读取 `number`
2. 对值加 1
3. 写回内存

多个线程交叉执行时，就可能造成数据丢失。

常见问题：

- 数据覆盖
- 结果不符合预期
- 链表断裂
- 指针访问非法
- 程序崩溃
- 死锁
- 资源泄露

---

## 4.2 共享资源与临界区

共享资源：

> 多个线程都可能访问的数据或资源。

例如：

- 全局变量
- 堆区数据
- 链表
- 队列
- 数组
- 文件描述符
- 数据库连接
- 网络连接
- 缓冲区

临界区：

> 访问共享资源的代码区域。

示例：

```c
pthread_mutex_lock(&mutex);

/* 临界区：访问共享资源 */

pthread_mutex_unlock(&mutex);
```

线程同步的目的：

> 保证多个线程访问共享资源时，执行顺序是安全、可控的。

---

## 4.3 常见线程同步工具

| 同步工具 | 作用 | 典型场景 |
|---|---|---|
| 互斥锁 mutex | 保护临界区 | 多线程修改共享变量 |
| 读写锁 rwlock | 读共享、写独占 | 读多写少 |
| 条件变量 cond | 等待某个条件成立 | 生产者消费者 |
| 信号量 sem | 控制资源数量 | 缓冲区、连接池 |
