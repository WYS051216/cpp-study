# 三、线程间数据传递

线程间数据传递主要包括：

- 创建线程时向线程函数传递参数
- 线程退出时向其他线程返回数据
- 主线程通过 `pthread_join()` 获取返回值

---

## 3.1 通过参数向线程传值

`pthread_create()` 的第四个参数 `arg` 可以用来向线程函数传参。

示例：

```c
pthread_create(&tid, NULL, callback, &data);
```

线程函数中接收：

```c
void* callback(void* arg)
{
    struct Test* t = (struct Test*)arg;

    return NULL;
}
```

注意：

- `arg` 是 `void*` 类型
- 传入时通常传地址
- 在线程函数中需要强制类型转换

---

## 3.2 通过 pthread_exit 返回值

线程可以通过 `pthread_exit()` 返回数据：

```c
pthread_exit(ptr);
```

其他线程可以通过 `pthread_join()` 接收：

```c
void *ptr = NULL;

pthread_join(tid, &ptr);
```

注意：

`pthread_exit()` 返回的是一个指针，因此必须保证该指针指向的内存在被访问时仍然有效。

---

## 3.3 使用全局变量传值

这种方式中，子线程修改全局变量，然后通过 `pthread_exit()` 返回全局变量地址。

示例：

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

struct Test
{
    int num;
    int age;
};

struct Test t;

void* callback(void* arg)
{
    t.num = 100;
    t.age = 6;

    pthread_exit(&t);
}

int main()
{
    pthread_t tid;

    pthread_create(&tid, NULL, callback, NULL);

    void *ptr = NULL;

    pthread_join(tid, &ptr);

    struct Test* pt = (struct Test*)ptr;

    printf("num = %d, age = %d\n", pt->num, pt->age);

    return 0;
}
```

特点：

- 全局变量生命周期长
- 线程结束后，全局变量仍然有效
- 可以安全返回全局变量地址

注意：

如果多个线程同时访问这个全局变量，需要考虑线程同步问题。

---

## 3.4 使用主线程栈空间传值

主线程中定义变量，然后把变量地址传给子线程。

示例：

```c
#include <stdio.h>
#include <stdlib.h>
#include <pthread.h>

struct Test
{
    int num;
    int age;
};

void* callback(void* arg)
{
    struct Test* t = (struct Test*)arg;

    t->num = 100;
    t->age = 6;

    pthread_exit(t);
}

int main()
{
    struct Test t;
    pthread_t tid;

    pthread_create(&tid, NULL, callback, &t);

    void *ptr = NULL;

    pthread_join(tid, &ptr);

    printf("num = %d, age = %d\n", t.num, t.age);

    return 0;
}
```

特点：

- 主线程栈空间中的变量在主线程函数结束前有效
- 子线程可以通过参数修改主线程中的变量
- 主线程通过 `pthread_join()` 等待子线程结束后再访问数据

注意：

主线程中的变量必须在子线程使用期间保持有效。

---

## 3.5 不要返回子线程局部变量地址

错误示例：

```c
void* callback(void* arg)
{
    struct Test t;

    t.num = 100;
    t.age = 6;

    pthread_exit(&t);
}
```

原因：

- `t` 是子线程函数内部的局部变量
- 它存储在线程自己的栈空间中
- 子线程结束后，该栈空间会失效
- 主线程再通过该地址访问数据会产生未定义行为

正确做法：

1. 返回全局变量地址
2. 返回堆区内存地址
3. 返回仍然有效的主线程栈变量地址
4. 在线程结束前通过同步机制拷贝数据
