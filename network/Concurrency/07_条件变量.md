# 七、条件变量 cond

## 7.1 条件变量作用

条件变量通常和**互斥锁**一起使用，用于解决：

- 当某个条件不满足时，线程阻塞等待
- 当条件满足后，再唤醒线程继续执行

在生产者消费者模型中：

- 当缓冲区或链表为空时，消费者阻塞等待
- 当生产者生产数据后，唤醒消费者继续消费

条件变量本质上是：

> 等待条件成立的机制。

条件变量类型：

```c
pthread_cond_t
```

---

## 7.2 条件变量相关 API

### pthread_cond_init

函数原型：

```c
int pthread_cond_init(pthread_cond_t *cond,
                      const pthread_condattr_t *attr);
```

作用：

- 初始化条件变量

---

### pthread_cond_destroy

函数原型：

```c
int pthread_cond_destroy(pthread_cond_t *cond);
```

作用：

- 销毁条件变量

---

### pthread_cond_wait

函数原型：

```c
int pthread_cond_wait(pthread_cond_t *cond,
                      pthread_mutex_t *mutex);
```

作用：

- 阻塞等待条件变量
- 等待期间会释放互斥锁
- 被唤醒后会重新加锁

---

### pthread_cond_timedwait

函数原型：

```c
int pthread_cond_timedwait(pthread_cond_t *cond,
                           pthread_mutex_t *mutex,
                           const struct timespec *abstime);
```

作用：

- 带超时时间的条件变量等待

---

### pthread_cond_signal

函数原型：

```c
int pthread_cond_signal(pthread_cond_t *cond);
```

作用：

- 唤醒至少一个等待在该条件变量上的线程

---

### pthread_cond_broadcast

函数原型：

```c
int pthread_cond_broadcast(pthread_cond_t *cond);
```

作用：

- 唤醒所有等待在该条件变量上的线程

---

## 7.3 pthread_cond_wait 的特点

`pthread_cond_wait()` 会自动完成三个动作：

1. 释放互斥锁
2. 阻塞等待条件满足
3. 被唤醒后重新加锁

典型结构：

```c
pthread_mutex_lock(&mutex);

while (条件不满足)
{
    pthread_cond_wait(&cond, &mutex);
}

/* 条件满足，访问共享资源 */

pthread_mutex_unlock(&mutex);
```

注意：

条件变量本身不保护共享资源，仍然需要互斥锁。

---

## 7.4 为什么使用 while 而不是 if

推荐写法：

```c
while (head == NULL)
{
    pthread_cond_wait(&cond, &mutex);
}
```

不推荐写法：

```c
if (head == NULL)
{
    pthread_cond_wait(&cond, &mutex);
}
```

原因：

1. 可能出现虚假唤醒。
2. 多个消费者可能同时被唤醒。
3. 某个消费者拿到资源后，其他消费者醒来时资源可能已经没了。
4. 被唤醒不代表条件一定满足，必须重新检查。

## 7.5 条件变量版生产者消费者模型

### 实现思路

- 使用链表作为共享资源
- 生产者负责创建节点并插入链表
- 消费者负责从链表中取出节点
- 使用互斥锁保护链表
- 使用条件变量通知消费者有数据可消费

### 示例代码

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

pthread_cond_t cond;
pthread_mutex_t mutex;

typedef struct Node
{
    int number;
    struct Node* next;
} Node;

Node *head = NULL;

void* producer(void* arg)
{
    while (1)
    {
        pthread_mutex_lock(&mutex);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), newNode->number);

        pthread_mutex_unlock(&mutex);

        pthread_cond_broadcast(&cond);

        sleep(rand() % 3);
    }

    return NULL;
}

void* consumer(void* arg)
{
    while (1)
    {
        pthread_mutex_lock(&mutex);

        while (head == NULL)
        {
            pthread_cond_wait(&cond, &mutex);
        }

        Node* node = head;
        head = head->next;

        printf("消费者, id: %lu, number: %d\n",
               (unsigned long)pthread_self(), node->number);

        free(node);

        pthread_mutex_unlock(&mutex);

        sleep(rand() % 3);
    }

    return NULL;
}

int main()
{
    pthread_mutex_init(&mutex, NULL);
    pthread_cond_init(&cond, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);
    pthread_cond_destroy(&cond);

    return 0;
}
```

### 使用条件变量时的注意事项

1. 条件变量本身不保护共享资源，仍需要互斥锁。
2. `pthread_cond_wait()` 一定放在 `while` 循环里判断条件。
3. 先加锁，再检查条件，再决定是否等待。
4. 修改共享条件后再发信号，逻辑更清晰。
5. `signal` 适合唤醒一个线程。
6. `broadcast` 适合唤醒多个线程。
