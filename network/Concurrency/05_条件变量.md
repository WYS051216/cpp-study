# 线程同步笔记：条件变量

## 16. 条件变量

### 16.1 条件变量的作用
条件变量通常和**互斥锁**一起使用，用于解决：

- 当某个条件不满足时，线程阻塞等待
- 当条件满足后，再唤醒线程继续执行

在生产者-消费者模型中：
- 当缓冲区或链表为空时，消费者阻塞等待
- 当生产者生产数据后，唤醒消费者继续消费

---

### 16.2 条件变量类型
```c
pthread_cond_t
```

---

### 16.3 常用函数

#### 16.3.1 初始化
```c
pthread_cond_init()
```

#### 16.3.2 销毁
```c
pthread_cond_destroy()
```

#### 16.3.3 阻塞等待
```c
pthread_cond_wait()
pthread_cond_timedwait()
```

#### 16.3.4 唤醒阻塞线程
```c
pthread_cond_signal()      // 唤醒至少一个等待线程
pthread_cond_broadcast()   // 唤醒所有等待线程
```

---

### 16.4 使用特点
- 条件变量**必须配合互斥锁使用**
- `pthread_cond_wait()` 会自动完成三个动作：
  1. 释放互斥锁
  2. 阻塞等待条件满足
  3. 被唤醒后重新加锁
- 推荐使用 `while` 检查条件，而不是 `if`
- 原因：线程被唤醒后，应再次确认条件是否真的满足

---

### 16.5 条件变量版生产者-消费者模型示例

```c
#include <stdio.h>
#include <stdlib.h>
#include <unistd.h>
#include <pthread.h>

pthread_cond_t cond;
pthread_mutex_t mutex;

// 链表结点
typedef struct Node
{
    int number;
    struct Node* next;
} Node;

// 头结点
Node *head = NULL;

// 生产者
void* producer(void* arg)
{
    while (1)
    {
        pthread_mutex_lock(&mutex);

        Node* newNode = (Node*)malloc(sizeof(Node));
        newNode->number = rand() % 1000;
        newNode->next = head;
        head = newNode;

        printf("生产者, id: %ld, number: %d\n", pthread_self(), newNode->number);

        pthread_mutex_unlock(&mutex);

        // 通知消费者有数据可消费
        pthread_cond_broadcast(&cond);

        sleep(rand() % 3);
    }
    return NULL;
}

// 消费者
void* consumer(void* arg)
{
    while (1)
    {
        pthread_mutex_lock(&mutex);

        while (head == NULL)
        {
            pthread_cond_wait(&cond, &mutex);
        }

        Node* node = head;
        head = head->next;

        printf("消费者, id: %ld, number: %d\n", pthread_self(), node->number);

        free(node);

        pthread_mutex_unlock(&mutex);

        sleep(rand() % 3);
    }
    return NULL;
}

int main()
{
    pthread_mutex_init(&mutex, NULL);
    pthread_cond_init(&cond, NULL);

    pthread_t t1[5], t2[5];

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t1[i], NULL, producer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_create(&t2[i], NULL, consumer, NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t1[i], NULL);
    }

    for (int i = 0; i < 5; i++)
    {
        pthread_join(t2[i], NULL);
    }

    pthread_mutex_destroy(&mutex);
    pthread_cond_destroy(&cond);

    return 0;
}
```

---

### 16.6 使用条件变量时的注意事项
1. 条件变量本身不保护共享资源，仍需要互斥锁  
2. `pthread_cond_wait()` 一定放在 `while` 循环里判断条件  
3. 先加锁，再检查条件，再决定是否等待  
4. 修改共享条件后再发信号，逻辑更清晰  
5. `signal` 适合唤醒一个线程，`broadcast` 适合唤醒多个线程  

---

### 16.7 一句话总结
- 条件变量本质上是**等待条件成立的机制**
- 常和互斥锁一起使用
- 非常适合生产者-消费者模型中“等待 / 唤醒”的场景
