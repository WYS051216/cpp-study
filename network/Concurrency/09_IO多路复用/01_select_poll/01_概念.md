# IO 多路复用学习笔记

## 1. IO 多路复用简介
IO 多路复用就是用来处理并发的。它通过一种机制，能够监视多个文件描述符（FD），一旦某个描述符就绪（读就绪或写就绪），就能通知程序进行相应的处理。

### 解决方案对比
* **select**: 跨平台 (Windows, Mac, Linux)，底层线性表，检测上限 1024。
* **poll**: 主要 Linux，底层线性表，无检测上限。
* **epoll**: 仅 Linux，底层红黑树（效率高），无检测上限。

## 2. select 函数详解

### 函数原型
```c
int select(
    int nfds, 
    fd_set *readfds, 
    fd_set *writefds, 
    fd_set *exceptfds, 
    struct timeval *timeout
);
```
### 返回值说明
`select` 函数的返回值是一个整数（`int`），用于表示检测的结果：
* **`> 0`**：成功，返回所有集合（读、写、异常）中状态发生变化的**就绪文件描述符的总数**。
* **`== 0`**：超时，在设定的 `timeout` 时间内，没有任何文件描述符的状态发生变化。
* **`-1`**：失败/错误，发生异常（例如被系统信号中断、传入了无效的描述符等），此时系统会设置全局变量 `errno` 以指示具体错误。

### 函数参数补充
1. **nfds**: 委托内核检测的文件描述符范围，通常是所有加入集合的 FD 中的 **最大值 + 1**。
2. **readfds**: **读集合**。内核检测该集合中的 FD 是否有数据可读。
3. **writefds**: **写集合**。内核检测该集合中的 FD 是否可以写入数据。
4. **exceptfds**: **异常集合**。内核检测该集合中的 FD 是否发生异常。
5. **timeout**: 超时时间。
    - `NULL`: 永久阻塞，直到有 FD 就绪。
    - `0`: 立即返回（非阻塞）。
    - `>0`: 等待指定时长，超时返回 0。

### timeval 结构体原型
```c
struct timeval {
    time_t      tv_sec;   // 秒 (s)
    suseconds_t tv_usec;  // 微秒 (μs) -> 注意：1s = 1,000,000μs
};
```

## 3. fd_set 集合操作函数
初始化 `fd_set` 类型的参数需要使用以下一系列宏：

* **FD_ZERO**: `void FD_ZERO(fd_set *set);`
  - 将 set 集合中，所有文件描述符对应的标志位设置为 0，集合中没有添加任何文件描述符。
* **FD_SET**: `void FD_SET(int fd, fd_set *set);`
  - 将文件描述符 fd 添加到 set 集合中，即**将 fd 对应的标志位设置为 1**。
* **FD_CLR**: `void FD_CLR(int fd, fd_set *set);`
  - 将文件描述符 fd 从 set 集合中删除，即**将 fd 对应的标志位设置为 0**。
* **FD_ISSET**: `int FD_ISSET(int fd, fd_set *set);`
  - 判断文件描述符 fd 是否在 set 集合中，即读一下 fd 对应的标志位到底是 0 还是 1。

## 4. 底层原理：文件描述符表 (FD Table)

### 虚拟地址空间逻辑

1. **进程启动**  
   启动一个进程后，操作系统会为其分配一个独立的**虚拟地址空间**。  
   这个虚拟地址空间通常分为两大部分：

   - **用户空间**
   - **内核空间**

   #### 用户空间
   用户空间是进程可以直接访问和使用的内存区域，主要用于存放进程运行时需要的代码和数据。通常包括：

   - **代码段**：存放程序的可执行指令
   - **数据段**：存放已初始化的全局变量和静态变量
   - **BSS 段**：存放未初始化的全局变量和静态变量
   - **堆（Heap）**：用于动态内存分配，例如 `malloc`、`new`
   - **栈（Stack）**：用于函数调用，保存局部变量、参数、返回地址等
   - **内存映射区**：用于动态库加载、文件映射等

   用户空间的特点：
   - 每个进程的用户空间彼此独立
   - 一个进程不能直接访问另一个进程的用户空间
   - 用户程序大多数时间都运行在用户空间中

   #### 内核空间
   内核空间是操作系统内核使用的内存区域，普通用户程序不能直接访问。它主要用于：

   - 存放操作系统内核代码和内核数据
   - 管理进程、内存、文件系统、设备驱动等资源
   - 提供系统调用接口，响应用户程序的请求

   内核空间的特点：
   - 由操作系统统一管理
   - 对用户程序不可直接访问
   - 当程序执行系统调用、发生中断或异常时，会从用户态切换到内核态，在内核空间中执行相关操作

2. **PCB 管理**  
   在内核空间中，有专门负责进程管理的模块。  
   在 Linux 中，进程控制块（PCB）本质上是一个名为 `task_struct` 的结构体，其中保存了管理进程所需的各种信息，例如进程状态、进程 ID、内存信息、文件描述符信息等。

3. **文件描述符**  
   `task_struct` 中包含与文件管理相关的结构。  
   其中有一个指向**文件描述符表**的指针，而文件描述符表本质上是一个整型索引表，用于存储文件描述符（File Descriptor, FD）。

4. **FD 的独立性**  
   内核会为**每一个进程**维护一张独立的文件描述符表。  
   这张表中的索引值通常都是从 `0` 开始分配的，因此在不同进程中可能会看到相同编号的文件描述符。  
   但是，这些相同编号的文件描述符**不一定指向同一个磁盘文件**，因为它们分别属于不同进程，各自维护、各自管理。

### fd_set 检测原理
- `fd_set` 存储了待检测的文件描述符集合。
- **标志位为 0**：代表不检测这个文件描述符。
- **标志位为 1**：代表检测这个文件描述符。

## 5. 核心演示代码 (C++)

```cpp
fd_set reads, temp;
FD_ZERO(&reads);
FD_SET(lfd, &reads); // 监听套接字放入集合
int maxfd = lfd;

while(1) {
    temp = reads; // 备份，防止内核修改原始集合
    int ret = select(maxfd + 1, &temp, NULL, NULL, NULL);
    
    if (FD_ISSET(lfd, &temp)) {
        int cfd = accept(lfd, NULL, NULL);
        FD_SET(cfd, &reads); 
        maxfd = cfd > maxfd ? cfd : maxfd;
    }
}
```
