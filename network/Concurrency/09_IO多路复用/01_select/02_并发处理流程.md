# IO 多路复用：基于 select 的并发处理流程

## 核心流程步骤详解

基于 `select` 实现并发服务器，其本质是**将“等待 IO 就绪”的任务委托给内核**，从而实现单线程处理多个并发连接。

---

### 1. 准备阶段 (初始化监听)
* **① 创建监听套接字**：
  使用 `lfd = socket();` 创建用于监听连接请求的文件描述符。
* **② 绑定地址信息**：
  使用 `bind();` 将 `lfd` 与本地的 IP 和端口进行绑定。
* **③ 设置监听模式**：
  使用 `listen();` 开启监听，等待客户端连接。

### 2. 集合初始化
* **④ 创建并初始化文件描述符集合**：
  - 定义 `fd_set read_set;` 用于存储所有需要检测**读事件**的文件描述符。
  - 使用 `FD_ZERO(&read_set);` 进行初始化，确保位图清零。
  - 使用 `FD_SET(lfd, &read_set);` 将监听套接字放入集合中，因为 `accept()` 本质也是一种“读”操作（等待连接请求）。

### 3. 事件循环检测
* **⑤ 周期性检测**：
  在 `while(1)` 循环中调用 `select()`。内核会阻塞进程，直到集合中有 FD 就绪或超时。
  > **注意**：由于 `select` 是传出参数，每次调用前通常需要使用一个 `temp_set` 备份原始集合。

### 4. 结果处理 (核心逻辑)
* **⑥ 解除阻塞并处理事件**：
  `select()` 返回后，遍历检测范围内的文件描述符，通过 `FD_ISSET()` 判断标志位是否为 1。

    * **情况 A：就绪的是监听套接字 (`lfd`)**
        - 说明有新的客户端请求连接。
        - 调用 `cfd = accept();` 建立连接。
        - **关键步骤**：调用 `FD_SET(cfd, &read_set);` 将这个新的通信套接字加入到下一次检测集合中。

    * **情况 B：就绪的是通信套接字 (`cfd`)**
        - 说明客户端发送了数据过来。
        - 调用通信函数（如 `read()` / `recv()`）接收数据。
        - **连接断开处理**：如果读到数据长度为 0，说明客户端关闭了连接。此时必须：
            1. 调用 `close(cfd);` 关闭本地套接字。
            2. 使用 `FD_CLR(cfd, &read_set);` 将该文件描述符从检测集合中剔除。

### 5. 循环往复
* **⑦ 重复执行**：
  回到第 ⑥ 步继续循环，等待下一波 IO 事件。

## 实战核心代码片段 (C++)

```cpp
fd_set reads, temp;
FD_ZERO(&reads);
FD_SET(lfd, &reads); // 1. 放入监听 lfd
int maxfd = lfd;

while(1) {
    temp = reads; // 备份
    int ret = select(maxfd + 1, &temp, NULL, NULL, NULL);

    // 判断监听 lfd
    if (FD_ISSET(lfd, &temp)) {
        int cfd = accept(lfd, NULL, NULL);
        FD_SET(cfd, &reads); // 2. 动态加入通信 cfd
        maxfd = cfd > maxfd ? cfd : maxfd;
    }

    // 判断通信 cfd
    for (int i = lfd + 1; i <= maxfd; ++i) {
        if (FD_ISSET(i, &temp)) {
            char buf[1024];
            int len = read(i, buf, sizeof(buf));
            if (len == 0) {
                FD_CLR(i, &reads); // 3. 移除断开的 cfd
                close(i);
            } else if (len > 0) {
                // 处理业务逻辑...
            }
        }
    }
}
```
