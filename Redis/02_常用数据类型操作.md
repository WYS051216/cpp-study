# Redis 核心数据类型与常用指令精讲笔记

## 一、 通用 Key (键) 操作

* **查看当前数据库中的所有键：**
  `keys *` （注：生产环境慎用，会阻塞单线程，建议用 `scan`）
* **设置 Key 值（String 类型）：**
  `set k1 WYS`
* **查看指定 Key 的数据类型：**
  `type k1`
* **判断指定的 Key 是否存在：**
  `exists k1` （存在返回 1，不存在返回 0）
* **删除指定的 Key 值：**
  `del k1` （注：原笔记写为 `del key k1` 语法有误，已修正）
* **给 Key 设置过期时间（单位：秒）：**
  `expire k1 10`
* **查看指定 Key 的存活时间：**
  `ttl k1` （返回剩余秒数。返回 `-1` 表示永久有效；返回 `-2` 表示该 Key 已过期或不存在）
* **清空当前数据库：**
  `flushdb`
* **清空所有数据库：**
  `flushall`

---

## 二、 String (字符串) 操作

String 是 Redis 最基本的数据类型，最大可以包含 512MB 的数据，可以存储任何形式的字符串（包括图片、序列化对象等）。

### 1. 基础读写
* **设置 / 覆盖值：**
  `set k1 value`
* **获取值：**
  `get k1`
* **当 Key 不存在时才设置值（分布式锁基础）：**
  `setnx k1 v1`
* **追加内容到现有值的末尾：**
  `append k1 hus`
* **获取值的长度：**
  `strlen k1`

### 2. 批量与范围操作
* **批量设置 K-V：**
  `mset k1 v1 k2 v2`
* **批量获取值：**
  `mget k1 k2`
* **批量原子性设置（不存在才生效）：**
  `msetnx k1 v1 k2 v2` （注：这是一个原子操作，只要有任意一个 Key 已存在，**所有**的设置都会失败）
* **获取指定区间的值（索引从 0 开始）：**
  `getrange k1 0 6`
* **从指定索引开始替换值：**
  `setrange k1 0 kobe`
* **设置值并同时设置过期时间：**
  `setex k1 20 v1`
* **获取旧值并设置新值：**
  `getset k1 new_value`

### 3. 数字运算 (原子操作)
*注：这些命令执行期间不会被上下文切换打断，是绝对安全的。*
* **自增 / 自减 1：**
  `incr k1` / `decr k1`
* **按指定步长增减：**
  `incrby k1 10` / `decrby k1 5`

---

## 三、 List (列表) 操作

List 底层是一个双向链表，两端插入/弹出性能极高（O(1)），但中间索引操作较慢。常用于实现队列或栈。

### 1. 插入与弹出
* **从左/右侧压入元素：**
  `lpush k1 v1 v2 v3` / `rpush k1 v1 v2 v3`
* **从左/右侧弹出并返回元素：**
  `lpop k1` / `rpop k1`
* **右出左进（消息流转）：**
  `rpoplpush k1 k2` （将 k1 最右侧的值弹出，压入 k2 的最左侧）
* **【补充】阻塞弹出（常用于消息队列）：**
  `blpop k1 10` / `brpop k1 10` （如果列表为空则阻塞等待，直到有元素或 10 秒超时）

### 2. 查询与修改
* **查询列表中所有元素：**
  `lrange k1 0 -1`
* **获取指定索引的值：**
  `lindex k1 2`
* **获取列表长度：**
  `llen k1`
* **在指定元素的前/后插入新值：**
  `linsert k1 before v1 new_v` / `linsert k1 after v1 new_v`
* **修改指定索引的元素：**
  `lset k1 index new_v`
* **删除指定数量的特定元素：**
  `lrem k1 数量 元素值`

---

## 四、 Set (无序集合) 操作

Set 底层基于哈希表实现，天然去重，增删查的时间复杂度都是 O(1)。特别适合用来做去重、判断元素是否存在以及集合运算。

### 1. 基础操作
* **向集合添加元素（自动去重）：**
  `sadd s1 v1 v2 v3`
* **查询集合中所有元素：**
  `smembers s1` （注：原笔记写为 `smember`，少了个 s）
* **判断集合是否存在某元素：**
  `sismember s1 v1` （存在返回 1，否则返回 0）
* **返回集合中的元素个数：**
  `scard s1`

### 2. 删除与弹出
* **删除集合中指定的元素：**
  `srem s1 v1 v2`
* **随机弹出一个元素并删除：**
  `spop s1` （可加数量参数）
* **随机获取一个元素但不删除：**
  `srandmember s1` （可加数量参数）
* **移动特定元素到另一集合：**
  `smove s1 s2 v1`

### 3. 集合数学运算
* **交集（共有部分）：**
  `sinter s1 s2`
* **并集（全部元素去重）：**
  `sunion s1 s2`
* **差集（s1 有而 s2 没有的元素）：**
  `sdiff s1 s2` （注：这就是数学上的差集，以左侧集合为基准）

## 五、 Hash (哈希) 操作

Hash 类似于 Java 中的 `Map<String, String>`，是一个 field 和 value 的映射表。非常适合用来存储对象（例如用户信息）。

### 1. 增删改查
* **设置单个 field-value：**
  `hset user id 1001`
* **获取单个 field 的值：**
  `hget user id`
* **批量设置（更新迭代推荐直接用 hset）：**
  `hmset user username eric age 23`
* **批量获取：**
  `hmget user id username age`
* **【补充】获取当前 Key 的所有 field 和 value：**
  `hgetall user`
* **删除指定 field：**
  `hdel user age`

### 2. 状态查询与计算
* **判断某 field 是否存在：**
  `hexists user id`
* **获取包含的所有 field 列表：**
  `hkeys user`
* **获取包含的所有 value 列表：**
  `hvals user`
* **对指定 field 进行自增计算：**
  `hincrby user age 1`
* **当 field 不存在时才设置：**
  `hsetnx user email test@test.com`

## 六、 ZSet (有序集合) 操作

ZSet 在 Set 的基础上，为每个元素增加了一个双精度浮点数的分数（Score）。元素唯一，但分数可以重复。底层通常使用跳表（SkipList）实现，按分数自动排序。

### 1. 添加与更新
* **添加元素及对应分数：**
  `zadd user 99 kobe 88 durant 82 curry`
* **给指定元素的分数进行增减：**
  `zincrby user 12 kobe`

### 2. 查询排位
* **按分数从低到高显示所有元素：**
  `zrange user 0 -1`
* **显示时同时带上分数：**
  `zrange user 0 -1 withscores`
* **查询指定元素的排名（升序排位，从 0 开始）：**
  `zrank user durant`
* **【补充】查询指定元素的排名（降序排位）：**
  `zrevrank user durant`

### 3. 范围统计与删除
* **获取指定分数区间的元素（升序）：**
  `zrangebyscore user 80 100 (withscores)`
* **获取指定分数区间的元素（降序）：**
  `zrevrangebyscore user 100 80`
* **统计指定分数区间的元素个数：**
  `zcount user 80 100`
* **删除指定元素：**
  `zrem user kobe`
