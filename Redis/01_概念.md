# NoSQL 与 Redis 核心知识笔记

## 一、 NoSQL 数据库基础

**1. 核心概念**
* **NoSQL (Not Only SQL)：** 泛指非关系型数据库。
* **存储模式：** 以简单的 **Key-Value** 模式存储，不依赖业务逻辑方式存储，因此极大地增强了数据库的扩展能力。

**2. 核心特点**
* **无标准：** 不遵循 SQL 标准。
* **无事务：** 不支持关系型数据库中的 ACID 事务特性。
* **极高性能：** 存储数据不依赖复杂业务逻辑，且数据主要存放在**内存**中，读写性能远超传统 SQL 数据库。

**3. 场景适用性对比**

| 适用场景 (优势) | 不适用场景 (局限) |
| :--- | :--- |
| **高并发读写：** 应对海量请求。 | **事务支持：** 需要严格的 ACID 事务保障。 |
| **海量数据：** 海量数据的快速读写。 | **复杂查询：** 需要进行多表联查等复杂的结构化查询。 |
| **高扩展性：** 需要灵活横向扩展的数据架构。 | |

## 二、 常见 NoSQL 数据库对比

| 数据库 | 核心特性 | 典型应用 |
| :--- | :--- | :--- |
| **Memcache** | 数据全在内存，**不支持持久化**。仅支持单一 Key-Value。 | 简单的纯内存缓存。 |
| **Redis** | 数据在内存，**支持持久化**。支持丰富数据结构（List, Set, Hash, Zset等）。 | 高级缓存、辅助持久化数据库等。 |
| **MongoDB** | 开源文档型数据库，模式自由。内存+硬盘。提供丰富的 Value 查询功能。 | 灵活的文档存储、内容管理。 |

## 三、 Redis 核心剖析

### 1. Redis 的基本特征
* **多数据类型：** 基于 Key-Value，支持多种复杂数据结构。
* **原子性操作：** 所有数据类型的操作（如 push/pop, 交/并/差集计算）都是**原子性**的。
* **支持排序：** 在丰富数据结构基础上，支持多种方式的排序。
* **持久化机制：** 会周期性地将更新的数据**写入磁盘**，或将修改操作写入**追加记录文件**。
* **高可用保障：** 实现了主从同步（Master-Slave）机制。

### 2. 底层架构
* **单线程 + 多路 I/O 复用：** 使用一个线程来检查多个文件描述符（FD）的就绪状态。就绪后进行真正的操作，避免了多线程切换和锁竞争。
* **对比 Memcache：** Memcache 采用的是“串行 + 多线程 + 锁”的架构。

### 3. 默认数据库实例管理
* **数量与索引：** 默认 **16 个**库，类似数组下标从 `0` 开始。
* **常用指令：**
  * `select <dbid>`：切换数据库（如 `select 8`）。
  * `dbsize`：查看当前库 key 的数量。
  * `flushdb`：清空当前库所有数据。

## 四、 Redis 典型应用场景与数据结构

* **1. List (列表)：** 获取最新数据（按时间排序）；构建队列。
* **2. Zset (有序集合)：** 排行榜（利用分数自动排序）。
* **3. Expire (过期机制)：** 时效性数据（如手机验证码）。
* **4. Set (集合)：** 数据去重。
* **5. Pub/Sub (发布订阅)：** 消息广播系统。
* **6. INCR/DECR (原子自增减)：** 计数器、秒杀（保证高并发下的准确性）。
