# C++17 现代特性笔记：高级类型容器 variant 与 optional

---

## 一、 `std::variant` (类型安全的联合体)

### 1. 概念补充与痛点解决
在传统的 C/C++ 中，联合体（`union`）不支持存放复合数据类型（如 `std::string`），且**类型极不安全**。存入 `float` 却按 `int` 读取时，编译器不会报错，只会给出一个无意义的垃圾值。

C++17 引入了 `std::variant`（加强版联合体）：
* **支持复合类型**：可以存放任意复杂的 C++ 类和容器。
* **单实例存放**：同一时间只能代表其中一种类型的实例。
* **自带状态追踪**：它在底层清清楚楚地记着当前活跃的是哪个类型（通过索引 `index`）。
* **绝对的类型安全**：错误读取会在编译期被拒绝，或在运行期抛出 `std::bad_variant_access` 异常。

### 2. 核心操作与进阶用法
* **获取当前状态**：使用 `v.index()` 可以获取当前存放类型的**零基索引**（即声明时的顺序，0, 1, 2...）。
* **安全读取方式**：
  1. `std::holds_alternative<T>(v)`：返回 bool，判断当前是否是 T 类型。
  2. `std::get<T>(v)`：强制读取。如果类型不对，抛出异常。
  3. **`std::get_if<T>(&v)`（优雅且安全）**：传入指针。如果类型匹配，返回指向数据的指针；如果类型不匹配，返回 `nullptr`，**不抛异常**。
  4. **`std::visit`（终极杀手锏）**：配合 Lambda 表达式，可以一次性穷举处理它内部可能出现的所有类型。

### 3. 代码示例
```cpp
#include <iostream>
#include <variant>
#include <string>

int main() {
    // 声明一个变体，索引规则：int(0), double(1), string(2)
    std::variant<int, double, std::string> v;

    // --- 基础用法与 index() ---
    v = 3.14; 
    std::cout << "当前存的是 double，索引为: " << v.index() << std::endl; // 输出 1

    v = "Hello";
    std::cout << "当前存的是 string，索引为: " << v.index() << std::endl; // 输出 2

    // --- 进阶安全读取：std::get_if (不抛异常) ---
    // 注意：get_if 需要传入 variant 的地址，它返回的是指针
    if (auto ptr = std::get_if<std::string>(&v)) {
        std::cout << "通过 get_if 成功拿到字符串: " << *ptr << std::endl;
    } else {
        std::cout << "获取失败，当前不是字符串！" << std::endl;
    }

    // --- 终极杀手锏：std::visit 模式匹配 ---
    v = 42;
    std::cout << "使用 visit 处理变体: ";
    std::visit([](auto&& arg) {
        // 这里的 auto&& 会自动推导当前活跃的类型，极其强大！
        std::cout << arg << std::endl;
    }, v);

    return 0;
}
```

## 二、 `std::optional` (可选值容器)

### 1. 概念补充与痛点解决
在编程中处理“可能失败的查找”或“可能不存在的值”时，过去常使用魔术特异值（如 `-1`、`nullptr`）。这极易引发逻辑混乱与空指针崩溃。

C++17 引入了 `std::optional<T>`，它是一个**可能包含值、也可能不包含值的包装容器**：
* **空状态**：用常量 `std::nullopt` 表示里面没东西。
* **有值状态**：就像一个安全的包裹，把真实数据套在里面。

### 2. 核心操作
1. **状态检查**：直接把 `optional` 对象放到 `if` 判断中，或调用 `.has_value()`。
2. **指针式访问**：确定有值时，可用 `*` 或 `->` 直接操作（如果其实是空的，会导致未定义行为）。
3. **安全提取**：
   * `.value()`：如果有值则返回，如果为空则明确抛出 `std::bad_optional_access` 异常。
   * **`.value_or(默认值)`（🌟最推荐）**：如果有值则返回原值，**如果为空则返回你传入的默认值**。

### 3. 代码示例
```cpp
#include <iostream>
#include <optional>
#include <string>

// 一个可能找不到用户的模拟函数
std::optional<std::string> findNickname(int userId) {
    if (userId == 1) {
        return "吉姆克拉克"; // 返回有效值（自动包装）
    }
    return std::nullopt;     // 明确返回空状态
}

int main() {
    // 场景 A：查询存在的用户
    auto user1 = findNickname(1);
    if (user1) { // 隐式检查 has_value()
        std::cout << "用户1的昵称: " << *user1 << std::endl;
    }

    // 场景 B：查询不存在的用户
    auto user2 = findNickname(999);
    
    // 安全防御：如果为空，自动 fallback 到 "匿名用户"
    std::string finalName = user2.value_or("匿名用户");
    std::cout << "最终显示的名称: " << finalName << std::endl; // 输出: 匿名用户

    // 异常捕获演示
    try {
        std::cout << user2.value() << std::endl; // 强行扒开空包裹，抛出异常
    } catch (const std::bad_optional_access& e) {
        std::cout << "【安全防御】空可选值提取失败: " << e.what() << std::endl;
    }

    return 0;
}
```
