## 一、 折叠表达式 (Fold Expressions)

### 1. 概念补充
折叠表达式是 C++17 为了简化**可变参数模板**中参数包的展开而引入的特性。在 C++11 中，展开参数包通常需要写复杂的递归函数和出口函数；而折叠表达式允许我们用一行极其简洁的语法，直接对参数包中的所有元素进行某种运算符操作。

### 2. 四种实例化展开方式
折叠表达式根据省略号 `...` 的位置以及是否有初值，分为以下四种展开方式（以运算符 `+` 为例，设参数包为 `E1, E2, ..., En`，初值为 `I`）：

| 语法格式 | 称呼 | 编译期展开后的表达式形态 | 运算顺序 |
| :--- | :--- | :--- | :--- |
| `(... + pack)` | **一元左折叠** | `((E1 + E2) + ...) + En` | 从左往右算 |
| `(pack + ...)` | **一元右折叠** | `E1 + (... + (En-1 + En))` | 从右往左算 |
| `(init + ... + pack)` | **二元左折叠** | `(((I + E1) + E2) + ...) + En` | 带着初值从左往右算 |
| `(pack + ... + init)` | **二元右折叠** | `E1 + (... + (En + I))` | 带着初值从右往左算 |

*注：对于减法 `-` 或除法 `/` 等不满足结合律的运算符，左折叠和右折叠的计算结果会完全不同，开发中需格外注意。*

### 3. 代码示例
```cpp
#include <iostream>
#include <string>

// 1. 一元左折叠：计算所有参数的和
template <typename... Args>
auto sum(Args... args) {
    return (... + args); // 一元左折叠
}

// 2. 二元左折叠：打印所有参数，并以指定字符串分隔
template <typename... Args>
void printWithPrefix(std::string prefix, Args... args) {
    // 相当于: ((prefix << arg1) << arg2) << arg3...
    (std::cout << ... << args) << std::endl; 
}

int main() {
    // 展开为: ((1 + 2) + 3) + 4 = 10
    std::cout << "sum: " << sum(1, 2, 3, 4) << std::endl; 

    // 二元左折叠演示
    printWithPrefix("数据: ", "Joel", 100, 3.14); 
    return 0;
}
```

## 二、 类模板参数推导 (CTAD - Class Template Argument Deduction)

### 1. 概念补充
在 C++17 之前，**函数模板**支持自动类型推导，但**类模板**实例化时必须显式指定目标类型（例如必须写 `std::vector<int> v;`）。
C++17 引入了 CTAD，使得类模板在实例化时，也可以像普通变量一样让编译器通过构造函数传入的实参自动推导模板类型。

### 2. 推导指引 (Deduction Guides)
除了编译器自带的隐式推导外，C++17 还允许开发者自定义**推导指引**，明确告诉编译器当遇到某种特殊的构造参数时，应该推导为什么类型。

### 3. 代码示例
```cpp
#include <iostream>
#include <vector>
#include <utility>

// 自定义一个包装类模板
template <typename T>
class Wrapper {
public:
    T data;
    Wrapper(T val) : data(val) {}
};

// 自定义类模板
template <typename T>
class Point {
public:
    T x, y;
    Point(T a, T b) : x(a), y(b) {}
};
// 显式定义推导指引（非必须，演示语法）：当传入两个参数时，推导为 Point<T>
template<typename T> Point(T, T) -> Point<T>;

int main() {
    // 1. 标准库容器推导
    std::pair p(10, 3.14);      // 自动推导为 std::pair<int, double>
    std::vector v = {1, 2, 3};  // 自动推导为 std::vector<int>

    // 2. 自定义类模板推导 (无须写 <int>)
    Wrapper w(100);             // 自动推导为 Wrapper<int>
    Point pt(1.5, 2.5);         // 自动推导为 Point<double>

    std::cout << "pt.x: " << pt.x << std::endl;
    return 0;
}
```

## 三、 `auto` 占位的非类型模板形参

### 1. 概念补充
**非类型模板形参（NTTP）**是指模板参数列表中不是“类型”，而是一个“常量值”（例如经典的 `template <typename T, int Size> class Array;` 中的 `Size`）。
在 C++17 之前，这个常量的类型（如 `int`）必须死死地写在模板声明里。C++17 允许使用 `auto` 占位符来声明非类型模板形参，这意味着**同一个模板，既可以接收 `int` 型常量，也可以接收 `char` 型或 `long` 型常量**，编译器会自动推导常量的具体类型。

### 2. 代码示例
```cpp
#include <iostream>

// 使用 auto 占位非类型模板形参
template <auto N>
class ConstantProvider {
public:
    void print() {
        std::cout << "常量的类型是: " << typeid(N).name() 
                  << ", 值是: " << N << std::endl;
    }
};

int main() {
    // 传入整型常量
    ConstantProvider<100> c1;
    c1.print(); // N 的类型被推导为 int

    // 传入字符型常量
    ConstantProvider<'A'> c2;
    c2.print(); // N 的类型被推导为 char
    
    return 0;
}
```

## 四、 编译器 `if constexpr` 语句 (编译期条件分支)

### 1. 概念补充
传统的 `if-else` 语句是在**运行期**执行分支判断的，这意味着不管满足哪个条件，所有分支的代码都必须成功通过编译。
C++17 引入了 `if constexpr`（编译期 if），它让条件判断发生在**编译期**。编译器在编译时会计算条件表达式的值，**只保留满足条件的那一个分支的代码，不满足的分支代码会被直接丢弃（不进行编译和实例化）**。
这一特性完美替代了过去需要靠复杂的标签分发（Tag Dispatching）或模板特化才能实现的编译期逻辑控制。

### 2. 代码示例
```cpp
#include <iostream>
#include <type_traits>

// 这个函数可以接收任意类型：如果是指针，就打印指针指向的值；如果是普通值，直接打印
template <typename T>
void printValue(T x) {
    // 必须使用 constexpr，在编译期完成类型判断
    if constexpr (std::is_pointer_v<T>) {
        std::cout << "是指针，其指向的值为: " << *x << std::endl;
    } else {
        // 如果传入的是普通 int，上面指针解引用的代码 (*x) 在编译期就会被安全丢弃。
        // 如果用普通 if，传入 int 时编译 *x 会直接报错。
        std::cout << "是普通值，值为: " << x << std::endl;
    }
}

int main() {
    int a = 666;
    printValue(a);   // 触发 else 分支，编译期丢弃指针解引用代码
    printValue(&a);  // 触发 if 分支，正常解引用指针
    return 0;
}
```
