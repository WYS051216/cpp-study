# C/C++ 核心学习笔记：条件编译与 `#error` 指令

## 一、 条件编译的核心概念

**什么是条件编译？**
在正常情况下，源程序文件中的所有代码行都会被编译器编译。但是有些时候，我们希望**当满足某个条件时才编译某一组语句，条件不满足时则编译另一组语句**。
条件编译允许编译器根据我们设置的“开关（宏）”去有选择性地编译不同的代码片段，最终产生不同的目标代码文件。

**条件编译的四大核心作用：**
1. **屏蔽跨平台差异：** 针对 Windows、Linux、Mac 等不同操作系统，编译对应平台的底层接口代码。
2. **包含程序功能模块：** 根据不同的版本需求（如基础版、专业版），选择性地编译不同的功能模块。
3. **开关调试信息：** 在开发阶段开启调试日志（Debug 模式），在软件发布时一键关闭（Release 模式），提高程序运行效率。
4. **防止头文件重复包含：** 使用 `#ifndef` 防止同一个头文件被多次引入导致重复定义错误。

## 二、 条件编译的常用语法与示例

### 1. 判断宏是否定义：`#ifdef` / `#ifndef`
这是最简单的开关控制。
* `#ifdef` (if defined)：如果该宏**已定义**，则编译。
* `#ifndef` (if not defined)：如果该宏**未定义**，则编译。

**语法结构与示例：**
```cpp
#include <iostream>

#define DEBUG_MODE  // 定义了一个宏，相当于打开了开关

int main() {
    // 因为前面定义了 DEBUG_MODE，所以下面的代码会被编译并执行
#ifdef DEBUG_MODE
    std::cout << "[Debug] 正在连接本地测试数据库..." << std::endl;
#endif

    // 如果没有定义 RELEASE_MODE，则编译下面的代码
#ifndef RELEASE_MODE
    std::cout << "当前为非正式发布版本。" << std::endl;
#endif

    return 0;
}
```

### 2. 多分支逻辑判断：`#if` ... `#elif` ... `#else` ... `#endif`
当有多个条件需要判断（类似普通的 `if-else` 语句）时，通常使用 `#if`。
* **注意细节：** `#if` 和 `#elif` 后面跟的必须是**常量表达式**（通常是宏定义的值的比较）。

**语法结构与示例：**
```cpp
#include <iostream>

#define VERSION 2   // 定义版本号为 2

int main() {
#if VERSION == 1
    std::cout << "编译并运行 V1.0 基础功能代码" << std::endl;

#elif VERSION == 2
    // 因为 VERSION 等于 2，所以只有这段代码会被真正编译进程序
    std::cout << "编译并运行 V2.0 升级版功能代码" << std::endl;

#else
    std::cout << "编译并运行最新实验性功能代码" << std::endl;

#endif
    return 0;
}
```

## 三、 强制终止编译：`#error` 指令

`#error` 是一个非常强硬的预处理指令，用于**在编译期间主动抛出一个错误信息，并立刻终止程序的编译**。
这通常用于检查编译器环境是否符合要求，防止程序在错误的环境下被强行编译出来并引发严重的运行崩溃。

* **语法：** `#error error_message`
* **注意细节：** `#error` 后面的错误信息不需要（也不建议）加双引号。

**应用示例：**
假设我们的程序强制要求在 64 位系统下编译，如果在 32 位系统下编译，我们就主动阻断它：

```cpp
// 假设 __x86_64__ 是 64 位编译器特有的宏
#ifndef __x86_64__
    // 如果没有检测到 64 位环境，直接报错并停止编译
    #error This program requires a 64-bit compiler environment!
#endif

int main() {
    // 如果是 32 位系统，编译根本走不到这一步
    return 0;
}
```
