# 📝 C++ 内存分区模型与动态内存管理

## 1. 内存四区总览
C++ 程序在执行时，会将内存大方向划分为四个区域。
* **意义**：不同区域存放不同的数据，赋予不同的生命周期，给我们更大的灵活编程空间。


* **代码区**：存放所有的二进制代码，由操作系统进行管理。
* **全局区**：存放全局变量、静态变量以及常量。
* **栈区**：由编译器自动分配和释放，存放函数的参数值、局部变量等。
* **堆区**：由程序员手动分配和释放。若程序员不释放，程序结束时由操作系统统一回收。

---

## 2. 程序运行前（编译生成 `.exe` 后，未执行前）
此时只存在两个区域：

### ① 代码区
* **内容**：存放 CPU 执行的机器指令（即你写的代码翻译成的二进制数据）。
* **特点**：
  * **共享的**：对于频繁被执行的程序，内存中只需要有一份代码即可（多次双击运行的是同一份底层代码）。
  * **只读的**：防止程序意外地修改了它的指令。

### ② 全局区
* **内容**：主要存放**全局变量**和**静态变量 (`static`)**。
* **包含常量区**：字符串常量（如 `"hello"`）和其他全局常量也存放在此。
* **特点**：该区域的数据生命周期贯穿整个程序，在**程序结束后由操作系统统一释放**。
* **🔥 核心区分考点**：
  * **不在全局区的数据**：局部变量、`const` 修饰的局部变量（局部常量）。它们在栈区。
  * **在全局区的数据**：全局变量、静态变量（`static` 关键字）、字符串常量、`const` 修饰的全局变量（全局常量）。

---

## 3. 程序运行后
程序开始执行，系统开始调度分配另外两个区域：

### ③ 栈区 (Stack)
* **机制**：由编译器**自动分配释放**。
* **内容**：存放函数的参数值（形参）、局部变量等。
* **⚠️ 致命注意事项**：**绝对不要返回局部变量的地址（或引用）！** 因为函数执行完毕后，栈区分配给该函数的内存会自动被释放。如果强行保留其地址去访问，会拿到一堆乱码或直接导致程序崩溃。

### ④ 堆区 (Heap)
* **机制**：由程序员**手动分配释放**。若程序员忘记释放（内存泄漏），程序一直运行就会一直占用内存，直到程序彻底结束才由 OS 回收。
* **用法**：在 C++ 中，主要利用 `new` 关键字在堆区开辟内存。

---

## 4. `new` 和 `delete` 操作符
`new` 操作符会在堆区申请内存，并**返回该数据对应类型的指针**。如果堆区的数据不需要了，必须使用 `delete` 关键字手动销毁。

### 场景 A：开辟和释放单个变量
```cpp
// 1. 在堆区开辟一块内存，存放整数 10，并将地址赋给指针 p
int* p = new int(10);

// 2. 释放这块内存
delete p; 
// 良好习惯：释放后将指针置空，防止野指针
// p = nullptr; 
```

### 场景 B：开辟和释放数组
```cpp
// 1. 在堆区创建一个包含 10 个整型元素的数组（10 代表元素个数）
int* arr = new int[10];

// 2. 释放数组（⚠️ 必须加 []，告诉编译器这是一个数组的连续内存）
delete[] arr; 
```

## 5. 核心面试题：`new/delete` 与 `malloc/free` 的本质区别 (🔥 必考重点)

虽然它们都能在堆区申请和释放内存，但在 C++ 面向对象编程中，它们有着不可跨越的鸿沟：

| 核心维度 | `new` / `delete` (现代 C++ 专属) | `malloc` / `free` (C 语言遗留) |
| :--- | :--- | :--- |
| **身份本质** | 是 C++ 的**运算符/关键字** | 只是 C 语言的**标准库函数** |
| **🔥 构造与析构** | **会**自动调用类的构造函数和析构函数 | **绝对不会**调用，只管分配/释放物理裸内存 |
| **返回类型** | 自动返回带具体类型的指针（如 `int*`），极其安全 | 永远返回 `void*`，必须手动**强制类型转换** |
| **空间大小** | 编译器**自动计算**所需大小（如 `new int`） | 必须手动利用 `sizeof` 计算精确的字节数 |
| **失败反馈** | 内存分配失败时，会抛出异常 (`std::bad_alloc`) | 内存分配失败时，直接返回空指针 `NULL` |

### 💻 代码直观对比

**1. C 语言老派做法 (`malloc` / `free`)：写起来极其繁琐**
```cpp
// 1. 必须手动算字节 (sizeof(int))
// 2. 必须手动把 void* 强转为 int*
// 3. 内存里存的是乱码，还得手动赋值
int* p1 = (int*)malloc(sizeof(int)); 
*p1 = 10;
free(p1);
```

**2. C++ 现代做法 (`new` / `delete`)：优雅且安全**
```cpp
// 自动计算大小、自动返回 int*、甚至顺手就把 10 给装进去了
int* p2 = new int(10); 
delete p2;
```

> **💡 终极防雷法则**：
> 在 C++ 项目中，**全面封杀** `malloc/free`，无脑使用 `new/delete`。千万不要把它们混用（比如用 `new` 申请的空间，用 `free` 去释放），否则在处理复杂对象时会导致析构函数没被调用，引发严重的内存泄漏！
