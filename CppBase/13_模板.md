# C++ 核心学习笔记：泛型编程与模板 (Templates)

## 一、 模板的核心概念与底层原理

### 1. 核心概念
* **什么是模板？** 模板是 C++ 中实现**泛型编程**和**代码重用**的核心机制。它允许我们编写与数据类型无关的通用代码。
* **主要优势：** 极大地提高了代码的复用性。我们不需要为了 `int`、`double`、`float` 分别写三个逻辑完全相同的重载函数，只需要写一个模板，减少了工作量，并在一定程度上代替了带参数的宏定义（且比宏更安全，因为有类型检查）。

### 2. 底层原理（模具的本质）
* **模板不是函数：** 记住，函数模板或类模板**本身并不是真正的代码**，它只是一套“模具”。
* **实例化生成：** 在编译阶段，编译器会根据你**实际传入的数据类型**，利用这套模具，偷偷为你自动生成一份甚至多份具体类型的真实函数（或类）。这个过程叫做**模板实例化**。生成的这些具体函数，在内存中拥有各自独立、不同的地址。

## 二、 函数模板 (Function Templates)

### 1. 定义格式
`typename` 和 `class` 在这里的含义完全等价，可以互换使用（但现代 C++ 推荐在函数模板中使用 `typename` 以表意清晰）。

```cpp
template <typename T1, typename T2, ...>
返回类型 函数名(参数列表) {
    // 函数体
}
```

### 2. 实例化方式与类型推演机制
* **隐式实例化：** 你不指明类型，直接传参数，让编译器自己去猜（推演）参数的真实类型。
    * **⚠️ 避坑：** 编译器在隐式推演时**非常死板，通常不会做隐式类型转换**。如果你只定义了一个类型 `T`，却传了一个 `int` 和一个 `double`，编译器会因为“到底该把 `T` 认定为谁”而直接报错。
    * **解决方案：** 就像你笔记里提到的，引入多个模板参数（`T1`, `T2`），让编译器分别推导它们，不去打架。
* **显式实例化：** 在函数名后加上尖括号 `<类型>`，强制告诉编译器使用什么类型来生成函数。

### 3. 代码示例：函数模板

```cpp
#include <iostream>
using namespace std;

// 1. 单参数类型模板
template <typename T>
T myMax(T a, T b) {
    return (a > b) ? a : b;
}

// 2. 多参数类型模板 (解决推导冲突)
// 注意：这里最终是否发生类型转换，取决于你的【返回类型】定的是 T1 还是 T2
template <typename T1, typename T2>
T1 addValues(T1 a, T2 b) {
    return a + b; // 因为返回类型是 T1，所以最终的运算结果会被强制转换为 T1
}

int main() {
    // 【隐式实例化】
    cout << myMax(10, 20) << endl;     // 编译器自动推导 T 为 int
    cout << myMax(3.14, 2.5) << endl;  // 编译器自动推导 T 为 double

    // 【显式实例化】
    // 强制把 10.5 截断成 int 传入
    cout << myMax<int>(10.5, 20) << endl; 

    // 【多类型模板参数】
    int n1 = 10;
    double n2 = 5.5;
    
    // T1被推导为int，T2被推导为double。
    // 因为函数写死了返回 T1(int)，所以 10 + 5.5 = 15.5 最后会被强转成 int(15) 返回
    cout << addValues(n1, n2) << endl; 

    return 0;
}
```

## 三、 类模板 (Class Templates)

### 1. 核心概念
使得一个类的成员变量、成员函数可以基于通用的数据类型，而不需要在编写类的时候就将数据类型写死。

### 2. 定义格式与使用规范
* **实例化强制要求：** 与函数模板不同，类模板在使用时**不能隐式推演**（C++17 之前），**必须在类名后面显式跟上尖括号 `<具体类型>`** 来进行实例化。

```cpp
#include <iostream>
using namespace std;

// 定义类模板
template <class T1, class T2>
class PairBox {
private:
    T1 first;
    T2 second;
public:
    PairBox(T1 a, T2 b) : first(a), second(b) {}
    
    void show() {
        cout << "First: " << first << ", Second: " << second << endl;
    }
};

int main() {
    // 类模板实例化：必须显式加上 <类型>
    PairBox<int, string> box1(1, "Apple");
    box1.show();

    PairBox<double, char> box2(3.14, 'A');
    box2.show();

    return 0;
}
```

## 四、 模板别名 (Template Aliases)

当模板的类型嵌套非常长时（比如 `std::vector<std::map<int, std::string>>`），我们通常需要给它起个别名。

* **`typedef`：** 传统的 C 语言做法，但在处理带有未定义类型的模板时非常繁琐且支持度不好。
* **`using`：** C++11 引入的现代语法，极其适合给模板起别名，语法更直观，可读性极强(推荐使用)。

### 代码示例：模板别名

```cpp
#include <iostream>
#include <map>
#include <string>
using namespace std;

// 假设我们需要一个字典，键永远是 int，但值可以是任意类型

// 【旧方式】使用 typedef 的奇技淫巧 (必须包在一个结构体里)
template <typename T>
struct TypedefDict {
    typedef map<int, T> type;
};

// 【现代方式】使用 using (优雅、直接)
template <typename T>
using ModernDict = map<int, T>;

int main() {
    // 使用 using 定义的别名
    ModernDict<string> myDict;
    myDict[1] = "Hello";
    myDict[2] = "World";

    cout << myDict[1] << " " << myDict[2] << endl;

    return 0;
}
```
