# C++ 核心学习笔记：多文件编程

## 一、 多文件编程的基本原则与文件分类

为了方便团队分工合作以及后期的代码维护，我们在进行多文件编程时应遵循一个**基本原则**：
> **实现相同（或相关）功能的代码应存储在同一个文件中。**

根据代码文件后缀名的不同，C++ 中的文件可以大致分为以下几类：

*   **`.h`**：头文件（Header File），通常用于存放变量声明、函数声明、类定义等。
*   **`.hpp`**：头文件，通常是将 `.h` 的声明与 `.cpp` 的实现（定义）混杂写在一起。常用于模板类或开源库（例如 OpenCV 就大量采用了这种格式）。
*   **`.cpp`**：源文件（Source File），用于存放具体的函数实现和功能代码，在 Windows 环境下最为常见。
*   **`.cc`**：源文件，功能与 `.cpp` 完全一样，但在 Unix/Linux 环境下更常见。

## 二、 `#include <>` 与 `#include ""` 的区别

在引入头文件时，尖括号和双引号在底层的查找逻辑是不同的：

*   **`#include <filename.h>`**：
    *   **查找路径**：直接去**编译器自带的类库路径**（系统头文件目录）里面寻找。
    *   **适用场景**：通常用于引用标准库头文件（如 `<iostream>`、`<vector>` 等）。
*   **`#include "filename.h"`**：
    *   **查找路径**：首先在**当前项目所在的相对路径**下寻找。如果没有找到，才会退而求其次，去编译器自带的头文件目录中查找。
    *   **适用场景**：通常用于引用开发者**自定义的头文件**。

## 三、 防止头文件重复包含的三种方法

在多文件项目中，如果头文件被互相嵌套引用，极易导致“头文件重复包含”的编译错误。为了避免这个问题，我们通常有以下三种处理方式：

### 1. 使用宏定义 (`#ifndef` ...) 避免重复引用
这是最传统、兼容性最好的做法。通过定义一个独一无二的宏来拦截重复的解析。
**注意**：设置的宏名必须是独一无二的（通常基于文件名大写），千万不要和项目中其他宏的名称发生冲突。

**代码示例：**
```cpp
// 文件名: my_math.h
#ifndef MY_MATH_H    // 如果没有定义过 MY_MATH_H 这个宏
#define MY_MATH_H    // 那么就定义它，并继续向下编译

// -------- 头文件实际内容 --------
int add(int a, int b);
class Calculator {
public:
    void calculate();
};
// ------------------------------

#endif               // 结束条件编译
```

### 2. 使用 `#pragma once` 避免重复引入
这是一种相对现代的做法。编译器只要遇到这行代码，就会立刻知道当前文件在整个编译过程中**只引入一次**。
**优点**：不需要去想一个唯一的宏名字，也不涉及宏展开，编译效率很高。

**代码示例：**
```cpp
// 文件名: my_math.h
#pragma once         // 放在文件的最开头位置

// -------- 头文件实际内容 --------
int add(int a, int b);
class Calculator {
public:
    void calculate();
};
// ------------------------------
```

### 3. 使用 `_Pragma` 操作符
`_Pragma("once")` 的作用与 `#pragma once` 完全相同，但它是一个**操作符**而不是预处理指令。
**优点**：因为它是一个操作符，所以它可以写在代码的任何地方，甚至可以和宏定义嵌套搭配使用，比 `#pragma` 更加灵活。

**代码示例：**
```cpp
// 文件名: my_math.h
_Pragma("once")      // 同样放在文件的开头位置

// -------- 头文件实际内容 --------
int add(int a, int b);
class Calculator {
public:
    void calculate();
};
// ------------------------------
```
