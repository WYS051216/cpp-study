# C++ 核心学习笔记：命名空间 (Namespace)

## 一、 为什么要使用命名空间？（背景与痛点）

*   **C 语言的历史遗留问题：** 在 C 语言中，全局变量和函数**不能重名**。当项目变得庞大，或者引入多个第三方库时，极易发生“命名冲突”（Name Collision）。
*   **传统的笨办法：** 为了避免冲突，C 语言程序员通常会在全局变量或函数名前加上特定的前缀，例如 `cv_len`、`cv_myfun`。这种做法会让代码显得冗长且难以维护。
*   **C++ 的解决方案：** C++ 引入了**命名空间 (namespace)** 的概念。它就像是给代码划分了不同的“文件夹”或“作用域”。不同命名空间里的变量和函数，即使名字完全一样，也不会互相冲突。

## 二、 命名空间的定义与访问

### 1. 定义语法
使用关键字 `namespace` 来定义一个专属的命名空间：
```cpp
namespace cv {
    int len = 10;          // 属于 cv 空间的变量
    void myfun() {         // 属于 cv 空间的函数
        // 函数体
    }
}
```

### 2. 访问语法：作用域解析符 `::`
要访问某个特定命名空间里的符号（变量、函数等），需要使用 `::`（作用域解析运算符）。
*   **访问规则：** `命名空间名称::符号名称`

**示例对比：**
```cpp
int len = 200; // 这是一个【全局变量】

// 访问全局变量
cout << len << endl;       // 默认输出全局的 200 (或者写成 ::len)

// 访问命名空间 cv 中的变量
cout << cv::len << endl;   // 明确指定，输出 cv 里面的 10

// 调用命名空间 cv 中的函数
cv::myfun();               
```

## 三、 标准命名空间 `std` 

C++ 的标准库（如输入输出流、各种容器等）将其所有的内容都放在了一个名为 **`std` (Standard)** 的命名空间中。为了使用标准库里的功能，我们有以下几种方式：

### 方式一：全量引入（最省事，但有风险）
```cpp
using namespace std;
```
*   **含义：** 告诉编译器，自动把 `std` 命名空间里的所有前缀都暴露出来。如果在当前文件中找不到某个名字，就去 `std` 里面找。
*   **优缺点：** 写代码非常方便（可以直接写 `cout` 而不是 `std::cout`），但在大型项目中，把 `std` 里成千上万的符号全部暴露，容易引发新的命名冲突。

### 方式二：按需引入（最推荐，安全严谨）
```cpp
using std::cout;
using std::endl;
```
*   **含义：** 只把标准库里的 `cout` 和 `endl` 引入当前作用域，标准库里的其他内容依然处于隐藏状态。
*   **优缺点：** 写法稍显繁琐，但是最安全的做法，能有效避免命名污染。

### 方式三：直接指定前缀（偶尔使用）
如果你既没有写 `using namespace std;`，也没有单独引入，那就必须在每次使用时带上 `std::` 前缀：
```cpp
std::cout << "Hello World" << std::endl;
```

---

### 💡 综合代码演示

```cpp
#include <iostream>

// 1. 按需引入标准库的输出流
using std::cout;
using std::endl;

// 2. 定义自定义命名空间 cv
namespace cv {
    int len = 10;
    void myfun() {
        cout << "Calling cv::myfun()" << endl;
    }
}

// 3. 定义全局变量
int len = 200;

int main() {
    // 访问全局变量
    cout << "Global len: " << len << endl;

    // 访问 cv 命名空间里的变量
    cout << "cv namespace len: " << cv::len << endl;

    // 调用 cv 命名空间里的函数
    cv::myfun();

    return 0;
}
```
