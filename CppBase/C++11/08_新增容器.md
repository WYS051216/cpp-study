# C++ 现代容器笔记：`array` 与 `forward_list`

---

## 一、 `std::array` (固定大小数组)

### 1. 核心特性
* **栈内存分配**：`array` 的数据保存在栈中（相比存放在堆内存中的 `vector`），省去了动态内存分配的开销，从而获得更高的访问和运行性能。
* **编译期固定大小**：在编译时就必须确定大小，且不可更改。
* **更安全的封装**：C++ 封装的数组模板类，不同于 C 风格数组，它**不会自动隐式退化成 `T*` 指针**，更加安全。
* **聚合初始化**：它是一个聚合类型，支持直接使用花括号 `{}` 进行初始化。
* **索引机制**：封装了固定大小的数组，元素下标索引从 0 开始。

### 2. 定义与初始化
```cpp
// 1. 声明（未初始化，里面是随机垃圾值）
std::array<类型, 长度> 名称; 

// 2. 列表初始化
// ps: 如果提供的值不够长度，剩余的元素会被自动赋值为 0 (或该类型的默认值)
std::array<类型, 长度> 名称 = {值1, 值2, ...}; 
```

### 3. 访问 `array` 元素的 6 种方式 (🌟已补充)
除了常见的下标，C++ 还提供了更安全、更底层的访问接口：

1. **`[]` 运算符**：最常用，但不做越界检查（速度快）。
2. **`at()` 成员函数**：**安全访问**。会进行越界检查，越界时抛出 `std::out_of_range` 异常。
3. **`std::get<Index>()`**：**编译期访问**。传入的索引必须是常量，如果越界，代码直接**无法编译通过**。
4. **`front()` (补充)**：直接获取数组的第一个元素的引用。
5. **`back()` (补充)**：直接获取数组的最后一个元素的引用。
6. **`data()` (补充)**：获取指向底层数组的**原生指针 (`T*`)**，方便与遗留的 C 语言 API 交互。

### 4. `array` 代码示例
```cpp
#include <iostream>
#include <array>

int main() {
    // 初始化一个长度为 5 的 array，只给了前 3 个值，后面自动补 0
    std::array<int, 5> arr = {10, 20, 30}; 

    // 1. 普通访问 []
    std::cout << "arr[0] = " << arr[0] << std::endl;

    // 2. 安全访问 at()
    try {
        std::cout << "arr.at(1) = " << arr.at(1) << std::endl;
        // std::cout << arr.at(10); // 这行如果取消注释，会抛出异常而不会直接让程序崩溃
    } catch (const std::out_of_range& e) {
        std::cout << "越界啦: " << e.what() << std::endl;
    }

    // 3. 编译期获取 get<>
    std::cout << "get<2> = " << std::get<2>(arr) << std::endl;

    // 4. 首尾访问 (补充)
    std::cout << "第一个元素 front() = " << arr.front() << std::endl;
    std::cout << "最后一个元素 back() = " << arr.back() << std::endl; // 输出 0

    // 5. 获取底层指针并像 C 语言一样使用 (补充)
    int* p = arr.data();
    std::cout << "指针访问第一个元素 = " << *p << std::endl;

    return 0;
}
```

## 二、 `std::forward_list` (单向链表)

### 1. 核心特性
* **单向链表实现**：与 `std::list`（双向链表）不同，`forward_list` 底层是单向链表。
* **极致轻量**：具有插入/删除速度快、消耗内存空间极少的特点，多被用于排序算法或对内存极其敏感的场景。
* **特性局限**：
  * 只能向前遍历，不支持反向遍历。
  * 不支持快速随机访问（不能用 `[]` 或 `at()`，想找第 5 个元素只能从头遍历 5 次）。
  * 🌟 **唯一特例：它是 C++ 标准库中唯一一个没有 `size()` 方法的容器**。因为如果要维持一个 size 计数器，会增加额外的内存和更新成本，违背了它“极致轻量”的设计初衷。
* **操作效率**：提供了 $O(1)$ 复杂度的头部插入（`push_front`）和指定节点后的插入/删除。

### 2. 创建方式的 5 种形态
```cpp
#include <forward_list>
#include <array>

// ① 创建一个空的 forward_list
std::forward_list<int> flist1;

// ② 创建包含 n 个元素的 forward_list（默认初始化为 0）
std::forward_list<int> flist2(10); 

// ③ 创建包含 n 个元素，且指定初始值
std::forward_list<int> flist3(10, 5); // 10 个元素，全都是 5

// ④ 拷贝构造（用现有的初始化新的）
std::forward_list<int> flist4(flist3);

// ⑤ 迭代器范围拷贝（非常强大，可以跨容器拷贝）
// 从原生数组拷贝:
int a[] = {1, 2, 3, 4, 5};
std::forward_list<int> flist5(a, a + 5);

// 从 array 或 vector 拷贝指定区域:
std::array<int, 5> arr = {11, 12, 13, 14, 15};
// 拷贝 12, 13, 14, 15
std::forward_list<int> flist6(arr.begin() + 1, arr.end()); 
```

### 3. 访问 `forward_list` 中的元素 (🌟核心难点)

由于 `forward_list` 是单向链表，它的设计哲学是“极致的轻量和极速的头插”，这导致它在元素访问上有着极其严苛的限制：

* **不支持随机访问**：绝对不能使用 `[]` 运算符或 `at()` 方法。
* **不支持尾部访问**：它没有 `back()` 方法（要想知道尾部是谁，只能从头走到尾）。
* **唯一的高效访问**：它只提供了 **`front()`** 方法，用来以 $O(1)$ 的速度直接访问第一个元素。

那么，如果我们要访问链表中间的元素怎么办？只能使用**迭代器（Iterator）**配合遍历来实现。

#### 常见的访问与遍历方式：
1. **直接获取头部**：使用 `front()`。
2. **全遍历**：最推荐使用前面笔记提到的 **For-each 循环**。
3. **精准定位第 N 个元素**：由于不能用 `flist[N]`，C++ 提供了专门的辅助工具 **`std::next()`** 或 **`std::advance()`**（需要引入 `<iterator>` 头文件），让迭代器模拟“往前走 N 步”的操作。

#### 补充代码示例：如何花式访问单向链表
```cpp
#include <iostream>
#include <forward_list>
#include <iterator> // 必须引入这个头文件才能使用 std::next

int main() {
    std::forward_list<int> flist = {10, 20, 30, 40, 50};

    // 1. 唯一的高效直接访问：获取头部元素
    std::cout << "第一个元素 (front): " << flist.front() << std::endl;
    
    // 我们可以直接修改头部元素
    flist.front() = 99; 
    std::cout << "修改后的头部元素: " << flist.front() << std::endl;

    // ---------------------------------------------

    // 2. 想要访问第 3 个元素（值为 30）怎么办？
    // 错误写法：flist[2]; // 编译直接报错！
    
    // 正确写法：使用 std::next 让迭代器“走两步”
    // flist.begin() 指向第 1 个元素，往前走 2 步，就到了第 3 个元素
    auto it = std::next(flist.begin(), 2); 
    std::cout << "第 3 个元素是: " << *it << std::endl;

    // 我们同样可以通过解引用来修改它
    *it = 333;

    // ---------------------------------------------

    // 3. 常规遍历（查看全部元素）
    std::cout << "当前链表的所有元素: ";
    for (const auto& val : flist) {
        std::cout << val << " ";
    }
    std::cout << std::endl;

    return 0;
}
```

## 三、 `std::unordered` 无序容器系列

### 1. 核心特性与底层机制
* **哈希表实现**：`unordered` 系列容器的内部不进行任何自动排序，其底层完全基于**哈希表（Hash Table）**实现。
* **惊人的性能**：得益于哈希映射，其插入、删除和搜索元素的平均时间复杂度达到了 **$O(1)$** 常数级别。
* **最坏情况**：如果发生严重的“哈希冲突”（即大量不同的元素被哈希函数算出了相同的桶索引），所有的元素都会挤在同一个桶的单向链表里，此时时间复杂度会退化为 **$O(n)$**。
* **高阶概念（负载因子）**：无序容器有一个概念叫**负载因子（Load Factor）**，计算公式为 `元素总数 / 桶的总数`。当负载因子超过一定阈值（默认通常是 1.0）时，容器会自动触发 **Rehash（重新哈希）**，让桶数翻倍，并重新分配所有元素，以维持 $O(1)$ 的高效。

### 2. 四大无序容器分类

| 容器名称 | 元素形式 | 键（Key）是否允许重复 | 适用场景 |
| :--- | :--- | :--- | :--- |
| **`std::unordered_set`** | 仅有键（Key） | **不允许**重复 | 快速去重，高频查找某个元素是否存在 |
| **`std::unordered_multiset`** | 仅有键（Key） | **允许**重复 | 允许重复元素的快速查找与计数 |
| **`std::unordered_map`** | 键值对（Key-Value） | **键不允许**重复 | 经典的哈希字典，通过 Key 极速存取 Value |
| **`std::unordered_multimap`** | 键值对（Key-Value） | **键允许**重复 | 一个键可以映射到多个不同的值（如一对多关系） |

---

### 3. 完整代码示例

下面这段代码一网打尽了这四个无序容器的基础声明、插入、查找以及遍历操作：

```cpp
#include <iostream>
#include <unordered_set>
#include <unordered_map>
#include <string>

int main() {
    // ==========================================
    // 1. std::unordered_set（唯一键集合）
    // ==========================================
    std::unordered_set<int> uset = {10, 20, 30};
    uset.insert(20); // 重复插入 20，会被自动忽略，因为键必须唯一

    std::cout << "--- unordered_set 遍历 ---" << std::endl;
    for (const auto& val : uset) {
        std::cout << val << " "; // 输出是无序的
    }
    std::cout << "\n\n";


    // ==========================================
    // 2. std::unordered_multiset（可重复键集合）
    // ==========================================
    std::unordered_multiset<int> umset = {10, 20, 30};
    umset.insert(20); // 允许重复，此时集合里有两个 20

    std::cout << "unordered_multiset 中 20 的个数: " << umset.count(20) << std::endl; // 输出 2
    std::cout << "\n";


    // ==========================================
    // 3. std::unordered_map（唯一键值对字典）
    // ==========================================
    std::unordered_map<std::string, int> umap;
    umap["Apple"] = 5;
    umap["Banana"] = 3;
    umap["Apple"] = 8; // 覆盖原本的 5，因为键 "Apple" 必须唯一

    std::cout << "--- unordered_map 查找与访问 ---" << std::endl;
    // 查找经典的写法：使用 find() 迭代器
    auto it = umap.find("Banana");
    if (it != umap.end()) {
        std::cout << "找到了！Banana 的数量是: " << it->second << std::endl;
    }
    std::cout << "\n";


    // ==========================================
    // 4. std::unordered_multimap（可重复键值对）
    // ==========================================
    std::unordered_multimap<std::string, std::string> umultimap;
    // 一个学生可以选多门课，键 "Joel" 可以重复出现
    umultimap.insert({"Joel", "C++"});
    umultimap.insert({"Joel", "Linux"});
    umultimap.insert({"Alex", "Java"});

    std::cout << "--- unordered_multimap 遍历 ---" << std::endl;
    // 注意：multimap 不支持 [] 运算符，必须用 insert 或 pair 遍历
    for (const auto& pair : umultimap) {
        std::cout << pair.first << " 选了 " << pair.second << " 课" << std::endl;
    }

    return 0;
}
```

### 4. 🌟 面试/开发高频考点：`std::map` 与 `std::unordered_map` 怎么选？

| 特性 | `std::map` | `std::unordered_map` |
| :--- | :--- | :--- |
| **底层实现** | 红黑树（平衡二叉树） | 哈希表（开链桶数组） |
| **时间复杂度** | 插入/查找均为 **$O(\log n)$** | 平均 **$O(1)$**，最坏 **$O(n)$** |
| **元素顺序** | **严格有序**（从小到大） | **完全无序** |
| **额外空间开销** | 较低（每个节点只需要存左右指针和颜色标记） | 较高（需要预留大量的桶数组空间） |
| **如何选择** | 只要不需要数据有序，**一律首选 `unordered_map`**，其性能优势巨大。 |
