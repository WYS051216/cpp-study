# C++ 现代特性笔记：`decltype` 与 左右值引用

## 一、 `decltype` 关键字解析

**1. 核心定义**
* `decltype` 是 C++11 引入的操作符，专门用于**查询表达式的数据类型**。
* **编译期行为**：它与 `sizeof` 非常相似，编译器只在编译阶段分析表达式并推导它的类型，**绝不会实际计算（执行）该表达式的值**。

**2. 诞生背景与设计目的**
* **服务泛型编程**：在模板编程中，某些类型是由传入的模板参数动态决定的，在代码编写时难以提前写死，`decltype` 可以完美提取这些未知类型。
* **弥补 `auto` 的缺陷**：`auto` 虽然能自动推导类型，但**必须伴随变量的初始化**（例如 `auto x = 10;`）。而 `decltype` 可以在不初始化变量的情况下，直接提取出类型用于声明。
* **推导 Lambda 表达式类型**：由于每个 Lambda 表达式在底层都有一个独一无二的匿名类类型，如果我们需要存储或使用这个特定类型，只能依靠 `decltype`。

**3. 代码示例**
```cpp
auto num1 = 100, num2 = 200;

// 编译器仅推导出 num1 + num2 的结果是 int 类型，
// 相当于执行了：int num3;
// 这一步绝对不会真正去执行 100 + 200。
decltype(num1 + num2) num3; 
```

## 二、 尾随返回类型 (Trailing Return Type)

在泛型编程中，如果函数的返回值类型需要通过参数运算来确定，我们可以结合 `auto` 和 `decltype` 将返回类型“后置”，让编译器在参数已知之后再去推导返回值类型。

**代码示例：**
```cpp
// 因为 t 和 u 的类型未知，只能等参数列表确立后，
// 再利用 decltype(t + u) 计算出返回值类型。
template <typename T, typename U>
auto add(T t, U u) -> decltype(t + u) 
{
    return t + u;
}
```

## 三、 左值 (Lvalue) 与 右值 (Rvalue)

这是 C++ 内存管理和性能优化的基础概念。判断左值和右值最简单的方法是：**能不能对它取地址（使用 `&` 操作符）？能取地址的就是左值，不能的就是右值。**

**1. 左值 (Locator Value)**
* **概念**：占据内存中可识别的固定地址、有名字、且生命周期超出单个表达式的变量或对象。
* **特征**：通常可以放在赋值等号 `=` 的左边。

**2. 右值 (Read Value)**
* **概念**：没有固定内存地址、没有名字的**临时数据**（比如字面常量、表达式计算的临时结果、函数返回的非引用临时对象）。它们在表达式结束后就会立即被系统销毁。
* **特征**：通常只能放在赋值等号 `=` 的右边。

**代码示例：**
```cpp
int main() {
    int a = 10; // 'a' 是左值（有名字，有地址），'10' 是右值（临时字面量）
    int b = a;  // 'b' 是左值，'a' 在这里作为左值被读取

    // 10 = a;  // 编译错误！'10' 是右值，没有内存地址来接收赋值
    // &10;     // 编译错误！不能对右值取地址

    int c = a + b; // 'a + b' 算出来的那一瞬间产生的一个临时结果是右值，随后赋值给左值 'c'
    
    return 0;
}
```

## 四、 右值引用 (`&&`)

**1. 核心概念**
* 右值引用是 C++11 引入的划时代特性，使用 `&&` 表示。它的主要作用是**绑定到一个即将消亡的右值（临时对象）上，从而人为地延长这个临时对象的生命周期**。
* **终极目的（移动语义）**：它的出现是为了解决 C++ 中对象传递时产生的大量“无意义的深拷贝”问题。通过右值引用，我们可以直接“偷取（Move）”临时对象底层的内存资源，实现极高的性能提升。

**2. 代码示例**
```cpp
#include <iostream>
using namespace std;

int getTempData() {
    return 100; // 返回一个临时值（右值）
}

int main() {
    // 1. 常规左值引用，不能绑定到右值！
    // int& ref1 = 10; // 编译报错：非常量左值引用不能绑定到右值

    // 2. 右值引用 &&，专门用来绑定右值（临时对象）
    int&& r_ref1 = 10; 
    
    // 绑定函数返回的临时对象，原本这个 100 在本行结束后就会销毁，
    // 现在它的生命周期被延长，和 r_ref2 一样长了。
    int&& r_ref2 = getTempData(); 

    // 注意：右值引用变量本身是一个左值！因为它有了名字 r_ref1，并在内存中固定了。
    r_ref1 = 20; // 完全合法，可以修改
    cout << "r_ref1: " << r_ref1 << endl; // 输出 20

    return 0;
}
```
