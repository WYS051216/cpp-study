# C++ 现代特性笔记：Lambda 表达式与函数对象

---

## 一、 Lambda 表达式 (匿名函数)

### 1. 核心概念
Lambda 表达式本质上是一个**匿名类函数**（或者叫闭包）。在编译阶段，C++ 编译器会在底层自动将这个表达式转换成一个未命名类的匿名对象。它让我们可以极其方便地在函数内部定义另一个简短的函数。

### 2. 完整语法结构
```cpp
[capture-list] (parameters) mutable -> return-type { statement };
```
* **`[捕获列表]`**：**绝不能省略**。它是 Lambda 的标志，编译器靠它来识别 Lambda 表达式。它能够把 Lambda 外部作用域的上下文变量“抓”进来，供内部使用。
* **`(参数列表)`**：与普通函数参数一致。如果不需要传参，可以连同 `()` 一起省略。
* **`mutable`**：默认情况下，按值捕获的变量在 Lambda 内部是**只读（`const`）**的。如果想在函数体内部修改这些按值捕获的变量的副本，必须加上 `mutable` 关键字。（注意：使用 `mutable` 时，参数列表的 `()` 即使为空也不能省略）。
* **`-> 返回值类型`**：C++11 引入的追踪返回类型声明。如果没有返回值，或者编译器能明确推断出返回值类型（比如只有一条 `return` 语句），这部分**可以省略**。
* **`{函数体}`**：**绝不能省略**。在内部可以使用捕获列表抓进来的变量，以及传进来的参数。

---

### 3. 捕获列表详解 (`[]`)
这是 Lambda 最强大也最容易迷惑的地方，主要分为以下 5 种情况：

| 写法 | 含义说明 |
| :--- | :--- |
| **`[]`** | **不捕获任何外部变量**。大部分纯逻辑计算的场景下，不捕获即可。 |
| **`[a, &b]`** | **混合捕获**。变量 `a` 按**值**（复制）捕获，变量 `b` 按**引用**捕获。 |
| **`[this]`** | **捕获当前类对象**。将类成员指针 `this` 传进来，从而可以直接访问类的成员变量和函数。 |
| **`[=]`** | **全值捕获**。将 Lambda 所在作用域的所有局部变量按**值**复制进来，并且隐式包含 `[this]`。 |
| **`[&]`** | **全引用捕获**。将 Lambda 所在作用域的所有局部变量按**引用**抓进来，并且隐式包含 `[this]`。 |

---

### 4. 捕获列表示例代码
为了演示 `[this]`，我们必须把代码写在一个类的成员函数里。下面这段代码完美涵盖了你笔记中的 5 种捕获情况：

```cpp
#include <iostream>
using namespace std;

class LambdaTest {
public:
    int m_class_var = 100; // 类的成员变量

    void testCapture() {
        int a = 1;
        int b = 2;
        int c = 3;

        // ① [] 不捕获：只能使用内部定义的变量或传入的参数
        auto f1 = [](int x) { 
            cout << "① [] 不捕获，传入参数 x: " << x << endl; 
        };
        f1(10);

        // ② [a, &b]：a 按值复制，b 按引用
        // 注意：如果要修改按值捕获的 a，必须加 mutable 关键字
        auto f2 = [a, &b]() mutable {
            a = 10; // 成功：因为加了 mutable，可以修改本地副本 a
            b = 20; // 成功：按引用捕获，直接修改外部的 b
            cout << "② [a, &b] 内部 a: " << a << ", b: " << b << endl;
        };
        f2();

        // ③ [this]：以引用捕获当前类对象
        auto f3 = [this]() {
            m_class_var = 200; // 成功：直接修改类的成员变量
            cout << "③ [this] 内部 m_class_var: " << m_class_var << endl;
        };
        f3();

        // ④ [=]：按值捕获所有外部自动变量 (隐式包含 this)
        auto f4 = [=]() {
            // a 和 c 都是外面的值复制进来的，只读
            cout << "④ [=] 内部 a: " << a << ", c: " << c 
                 << ", m_class_var: " << m_class_var << endl;
        };
        f4();

        // ⑤ [&]：按引用捕获所有外部自动变量 (隐式包含 this)
        auto f5 = [&]() {
            c = 30; // 成功：直接修改了外部真实的 c
            m_class_var = 300; 
            cout << "⑤ [&] 内部将 c 修改为: " << c << endl;
        };
        f5();
    }
};

int main() {
    LambdaTest t;
    t.testCapture();
    return 0;
}
```

## 二、 函数对象 (仿函数 Functor)

### 1. 概念
**函数对象（又称仿函数）**，本质上是一个**普通类的对象**。它之所以被称为“函数”，是因为这个类**重载了函数调用运算符 `operator()`**。
这使得这个类的对象可以像普通函数一样，加个括号就能直接调用执行。

### 2. Lambda 与仿函数的关系
从底层实现和最终的使用方式来看，**Lambda 表达式与函数对象完全等价**。事实上，C++ 编译器在遇到 Lambda 表达式时，在底层就是自动帮你生成了一个匿名的仿函数类。

### 3. 代码对比示例
下面这段代码直观地展示了仿函数和 Lambda 表达式在调用时是何等的一致：

```cpp
#include <iostream>

// 1. 定义一个传统的函数对象（仿函数）类
class AddFunctor {
public:
    // 重载 () 运算符
    int operator()(int x, int y) const {
        return x + y;
    }
};

int main() {
    // 实例化仿函数对象
    AddFunctor myFunctor;

    // 写法 A：使用仿函数
    int result1 = myFunctor(10, 20);
    std::cout << "仿函数计算结果: " << result1 << std::endl;

    // 写法 B：使用 Lambda 表达式
    auto myLambda = [](int x, int y) {
        return x + y;
    };
    int result2 = myLambda(10, 20);
    std::cout << "Lambda计算结果: " << result2 << std::endl;

    // 结论：两者的调用语法长得一模一样！
    return 0;
}
```
