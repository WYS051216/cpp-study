# C++ 现代特性笔记：可变参数模板与成员函数控制

---

## 一、 可变参数模板 (Variadic Templates)

### 1. 核心概念
在 C++11 之前，类模板和函数模板只能包含**固定数量**的模板参数。
C++11 引入了可变参数模板，允许在创建函数模板和类模板时，接受**任意数量（包含 0 个）和任意类型**的参数。这种参数被称为**参数包 (Parameter Pack)**。

### 2. 代码示例：基础定义
在声明参数包时，我们在 `typename` 后面加上省略号 `...`。

```cpp
#include <iostream>

// 这是一个可变参数函数模板
// Args 是一个模板参数包；args 是一个函数参数包
template <typename... Args>
void showSize(Args... args) {
    // sizeof...(args) 可以直接获取参数包里有多少个参数
    std::cout << "传入了 " << sizeof...(args) << " 个参数" << std::endl;
}

int main() {
    showSize();                 // 输出: 传入了 0 个参数
    showSize(1, "Hello");       // 输出: 传入了 2 个参数
    showSize(1.5, 'c', 10, 20); // 输出: 传入了 4 个参数
    return 0;
}
```

## 二、 参数包的展开方式

参数包不能像普通的数组那样用 `for` 循环直接遍历，必须通过特定的机制将它“解包”展开。常用的展开方式有两种：传统的递归展开和现代的 `if constexpr` 展开。

### 方式 1：递归函数展开 (C++11 标准做法)

**原理**：利用函数重载，把参数包拆分成**“第一个参数 (first)”**和**“剩下的参数包 (rest...)”**。处理完第一个参数后，将剩下的参数包再次传给函数自己，直到参数包为空，触发边界条件（无参的重载函数）停止递归。

```cpp
#include <iostream>

// 1. 递归的出口（边界条件）：当参数包被完全剥离，没有参数时调用这个函数
void print() {
    std::cout << "--- 展开结束 ---" << std::endl;
}

// 2. 递归展开函数
template <typename T, typename... Args>
void print(T first, Args... rest) {
    std::cout << "处理参数: " << first << std::endl;
    // 递归调用，把剩下的参数包解包后继续传给自己
    print(rest...); 
}

int main() {
    print(1, "Hello", 3.14); 
    // 执行顺序推演：
    // print(1, "Hello", 3.14) -> 打印 1, 调用 print("Hello", 3.14)
    // print("Hello", 3.14)    -> 打印 "Hello", 调用 print(3.14)
    // print(3.14)             -> 打印 3.14, 调用 print()
    // print()                 -> 命中出口函数，打印 "--- 展开结束 ---"
    return 0;
}
```

### 方式 2：使用 `if constexpr` 展开 (C++17 特性)
**原理**：`if constexpr` 是编译期的条件判断。它可以在编译阶段计算参数包的长度，直接在同一个函数体内优雅地完成展开，**彻底抛弃了上面那种必须额外写一个“出口函数”的麻烦做法。**

```cpp
#include <iostream>

template <typename T, typename... Args>
void printModern(T first, Args... rest) {
    std::cout << "处理参数: " << first << std::endl;
    
    // 在编译期判断：如果剩下的参数包里还有东西，就继续递归
    if constexpr (sizeof...(rest) > 0) {
        printModern(rest...); 
    }
}

int main() {
    printModern("现代C++", 100, 'A');
    return 0;
}
```

## 三、 默认成员函数控制 (`= default` 和 `= delete`)

在 C++ 中，如果我们没有自己写，编译器会自动为类生成 6 个默认的成员函数（默认构造、拷贝构造、移动构造、拷贝赋值、移动赋值、析构）。
C++11 引入了 `= default` 和 `= delete`，让我们能够精确夺回对这些默认函数的控制权。

### 1. `= default` (显式要求编译器生成默认版本)
* **作用**：如果我们自己写了带参数的构造函数，编译器就不会再帮我们生成无参的默认构造函数了。用 `= default` 可以强制让编译器恢复生成，并且它的运行效率比我们手写一个空的 `{}` 还要高。

### 2. `= delete` (显式禁用该函数)
* **作用**：明确告诉编译器**绝对不要**生成这个函数，且禁止任何人调用它。常用于设计**“禁止拷贝”**的类（例如管理独占资源、文件句柄的类）。

### 3. 代码示例
```cpp
#include <iostream>

class NonCopyableTask {
public:
    int taskId;

    // 我们写了一个带参数的构造函数
    NonCopyableTask(int id) : taskId(id) {}

    // 1. 使用 = default 强制恢复无参默认构造函数
    NonCopyableTask() = default;

    // 2. 使用 = delete 彻底禁用拷贝构造函数
    NonCopyableTask(const NonCopyableTask& other) = delete;

    // 3. 使用 = delete 彻底禁用拷贝赋值运算符
    NonCopyableTask& operator=(const NonCopyableTask& other) = delete;
};

int main() {
    // 正常调用 = default 恢复的无参构造
    NonCopyableTask t1;       
    
    // 正常调用带参构造
    NonCopyableTask t2(100);  

    // NonCopyableTask t3 = t2; // 编译直接报错！因为拷贝构造被 = delete 禁用了
    // t1 = t2;                 // 编译直接报错！因为拷贝赋值被 = delete 禁用了

    return 0;
}
```
