## `auto` 关键字

### 核心概念
* **作用**：让编译器根据**初始值**自动推导变量的类型。
* **时机**：推导发生在**编译期**，完全**不影响**程序的运行效率。
* **语法**：`auto 变量名 = 初始值;` （必须初始化，否则编译器猜不出来）。

### 推导规则汇总 (核心考点) ⚠️
这是面试中最喜欢问的细节，只需记住这三条铁律：

1.  **`auto` 单独使用（值拷贝）**：
    * **规则**：会**丢弃**顶层的 `const` 和引用 `&`。
    * **结果**：变量是一个全新的副本，修改它不会影响原数据。
2.  **`auto&`（引用）**：
    * **规则**：会**保留** `const` 属性。
    * **结果**：变量是原数据的别名，如果原数据是 const，这里也是 const。
3.  **`auto*`（指针）**：
    * **规则**：会推导为指针类型，且**保留** `const` 属性。

### 代码演示

```cpp
int x = 10;
const int cx = 20;
const int& rx = x;

// --- 规则 1: auto (值拷贝) ---
auto a = cx;    // a 是 int (const 被丢弃)
auto b = rx;    // b 是 int (const 和引用都被丢弃)

// --- 规则 2: auto& (引用) ---
auto& c = cx;   // c 是 const int& (const 被保留)
auto& d = rx;   // d 是 const int& (原样保留)

// --- 规则 3: auto* (指针) ---
auto* e = &cx;  // e 是 const int* (保留 const)
```
## decltype 关键字

### 核心概念
* **作用**：选择并返回操作数的数据类型。
* **特点**：在此过程中，编译器**只分析表达式的类型**，**绝对不会执行**该表达式（Runtime 不执行）。
* **语法**：`decltype(表达式) 变量名;`

### decltype 与 auto 的核心区别 (面试必考)

| 特性 | auto | decltype |
| :--- | :--- | :--- |
| **初始化** | **必须**初始化 (要根据值推导) | **可以不**初始化 (根据表达式推导) |
| **推导依据** | 赋值操作右边的**值** | 括号内**表达式**的类型 |
| **Const / 引用** | 默认**丢弃**顶层 const 和引用 | **原样保留** (所见即所得) |

### 代码实战演示

```cpp
#include <iostream>
using namespace std;

// 用于测试的函数
int func() {
    cout << "func 被调用了" << endl;
    return 10;
}

int main() {
    int a = 10;
    const int& b = a; // b 是 a 的常引用

    // --- 1. 基础用法 ---
    decltype(a) x;       // x 是 int
    x = 20;

    // --- 2. 不执行代码特性 ---
    // 编译器只看 func() 返回值是 int，所以 y 是 int
    // 运行程序时，终端【不会】打印 "func 被调用了"
    decltype(func()) y;  

    // --- 3. 保留引用和 const (重要区别) ---
    // auto z = b;       // 如果用 auto，z 会变成 int (丢弃引用和 const)
    
    decltype(b) z = a;   // z 是 const int& (原样保留 b 的属性)
    // z = 20;           // ❌ 报错：不能给常量赋值

    return 0;
}
```
