# C++ 核心学习笔记：异常处理 (Exception Handling)

## 一、 异常处理的核心概念

**什么是异常处理？**
异常处理是 C++ 处理**运行时错误**的一种机制。它允许程序在遇到无法处理的错误时，将控制权从出错点转移到能够处理该错误的地方，从而防止程序因为一个小错误就直接崩溃，保持应用程序的正常流转。

**异常的本质：**
C++ 中的异常是在运行时抛出的**事件或对象**。如果不捕获异常，程序会默认调用 `std::terminate()`，打印异常消息并强制终止。

---

## 二、 异常处理的三大支柱 (`try`, `catch`, `throw`)

* **`try`：** 用于包裹一段“可能会出错的代码”。如果这段代码在执行时发生了异常，程序会立刻停止执行 `try` 块内后续的代码，并跳转到对应的 `catch` 块。
* **`throw`：** 用于“抛出异常”。当程序检测到错误时，使用 `throw` 关键字将一个对象（异常信息）扔出去。
* **`catch`：** 用于“捕获并处理异常”。它紧跟在 `try` 块之后，定义了当某种类型的异常发生时，程序该如何应对。

---

## 三、 C++ 标准异常类 (`std::exception`)

所有 C++ 标准库抛出的异常都派生自 `std::exception` 类。通过捕获基类 `std::exception`，你可以捕获到几乎所有标准异常。

| 异常类 | 说明 |
| :--- | :--- |
| `std::bad_alloc` | 内存分配失败（如 `new` 失败）时抛出 |
| `std::out_of_range` | 数组或容器访问越界时抛出 |
| `std::invalid_argument` | 传递了无效参数时抛出 |
| `std::runtime_error` | 运行时发生的逻辑错误 |
| `std::logic_error` | 程序逻辑错误 |

---

## 四、 `try-catch` 语句的应用



### 代码示例：捕获异常
```cpp
#include <iostream>
#include <stdexcept> 

using namespace std;

int main() {
    int a = 10, b = 0;
    try {
        if (b == 0) {
            throw runtime_error("除数不能为 0！"); // 抛出异常
        }
        cout << "结果: " << a / b << endl;
    } 
    catch (const runtime_error& e) { // 捕获特定异常
        cout << "捕获到运行时错误: " << e.what() << endl;
    } 
    catch (...) { // 捕获所有类型的异常
        cout << "捕获到未知异常。" << endl;
    }

    return 0;
}
```

## 五、 自定义异常

在实际开发中，我们可以通过继承 `std::exception` 类并重载其 `what()` 虚函数来定义属于自己的异常类型。

### 代码示例：自定义异常
```cpp
#include <iostream>
#include <exception> // 包含异常处理的标准库
#include <string>

using namespace std;

/**
 * 1. 【报警器类定义】
 * 通过继承 exception，并重写 what() 方法，我们定义了一个“智能报错包裹”。
 */
class MyException : public exception {
private:
    string errorMsg; // 我们可以往里面塞任何想要的信息

public:
    // 构造函数：初始化报错信息
    MyException(string msg) : errorMsg(msg) {}

    /* * what() 规范写法说明：
     * 1. const: 确保此函数不修改对象的状态，它是“只读”查看标签的。
     * 2. noexcept: 承诺此函数不会再抛出新异常。
     * 3. override: 显式确认我们在重写父类的虚函数，防止拼写错误。
     */
    const char* what() const noexcept override {
        return errorMsg.c_str(); // 返回我们自定义的报错字符串
    }

    // 这里展示了：自定义异常类不仅可以重写 what，还可以自定义其他函数
    int getErrorCode() const { return 1001; }
};

/**
 * 2. 【触发报警】
 * 危险源函数：逻辑出错时，将“异常对象”丢出去。
 */
void riskyFunction(int b) {
    if (b == 0) {
        // 创建一个 MyException 对象并抛出，系统会将其护送到 catch 块
        throw MyException("检测到除数为0的错误！");
    }
}

/**
 * 3. 【监控室处理】
 * 这里通过 catch 块的类型匹配，接收对象并调用其方法。
 */
int main() {
    try {
        cout << "尝试执行危险操作..." << endl;
        riskyFunction(0); // 触发异常
    } 
    // 捕获异常：e 就是那个被抛出的“包裹”（对象）
    catch (const MyException& e) {
        // 这里我们主动通过 e.what() 和 e.getErrorCode() 提取信息
        // 系统并没有自动运行这些函数，而是按需读取
        cout << "--- 监控室处理异常 ---" << endl;
        cout << "报警信息: " << e.what() << endl;      // 读取“标签”
        cout << "错误代码: " << e.getErrorCode() << endl; // 读取“编号”
    }
    
    return 0;
}
```

## 六、 嵌套类 (Nested Class)

### 1. 核心概念与作用
* **定义：** 将一个类的定义放在另一个类的内部，这个被放在内部的类就被称为**嵌套类**，包含它的类被称为**外围类**。
* **作用：** 外围类通常需要使用嵌套类对象作为底层实现。将类嵌套可以仅仅为外围类服务，同时对外部用户**隐藏**该底层实现，起到良好的封装作用。
* **物理独立：** 嵌套类仅仅是**语法上的嵌入**，它和外围类在物理内存上是完全独立的两个类。嵌套类的对象并不包含外围类的子对象，反之亦然。

### 2. 作用域与定义
* **作用域隐藏：** 嵌套类的类名被隐藏在外围类的作用域中。如果嵌套类被声明为 `public`，在外围类之外的作用域使用它时，必须加上**名字限定**（例如 `Outer::Nested`）。
* **函数外置定义：** 嵌套类的成员函数可以写在它的类体外面（甚至外围类的外面）进行定义，但需要写明多重作用域。

### 3. 双方的访问权限（⚠️ 重点避坑）
| 访问方向 | 权限说明 |
| :--- | :--- |
| **嵌套类 $\rightarrow$ 外围类** | **可以访问！**（自 C++11 起）嵌套类作为外围类的一个成员，可以直接访问外围类的所有成员（包括私有成员）。但如果访问的是非静态成员，必须通过外围类的对象或指针来访问。 |
| **外围类 $\rightarrow$ 嵌套类** | **不可以越权！** 外围类只能通过嵌套类的对象访问其 `public` 成员，**绝对不能**直接访问嵌套类的 `protected` 和 `private` 成员（除非嵌套类将外围类声明为 `friend` 友元）。 |

### 4. 代码示例：嵌套类

```cpp
#include <iostream>
using namespace std;

// 外围类
class Outer {
private:
    int outerPrivateData = 100; // 外围类的私有成员

public:
    // 嵌套类 (默认仅供 Outer 内部使用，也可设为 public 供外部调用)
    class Nested {
    public:
        // 嵌套类可以访问外围类的私有成员（需要传入外围类对象）
        void accessOuter(Outer& outObj) {
            cout << "嵌套类访问外围类的私有数据: " << outObj.outerPrivateData << endl;
        }
    };

    void useNested() {
        Nested n;          // 外围类实例化嵌套类对象
        n.accessOuter(*this); // 调用嵌套类公有成员
    }
};

int main() {
    Outer out;
    out.useNested();

    // 如果嵌套类是 public 的，外部也可以直接使用，但必须加作用域限定
    Outer::Nested outsideNested;
    outsideNested.accessOuter(out);

    return 0;
}
```

## 七、 局部类 (Local Class)

### 1. 核心概念
* **定义：** 定义在**函数体里面**的类，被称为局部类。
* **可见性：** 局部类只在定义它的那个**局部作用域**（即该函数体内部）可见。出了这个函数，别的代码根本不知道有这个类的存在。

### 2. 使用限制（极为严苛）
局部类由于受限于函数的生命周期，有非常严格的语法限制：
1.  **函数体必须内联：** 局部类的所有成员函数**必须直接定义在类体内部**，不允许在类外定义。
2.  **禁止静态成员：** 局部类中**绝对不能**有静态数据成员或静态成员函数，因为局部类没有全局的作用域来为静态成员分配空间和初始化。
3.  **外部变量访问限制：** 局部类不能访问所在函数中的普通局部变量，**只能访问所在函数中的静态变量（`static`）**、全局变量或枚举常量。

### 3. 代码示例：局部类

```cpp
#include <iostream>
using namespace std;

// 全局变量
int globalVar = 50;

void myFunction() {
    static int funcStaticVar = 20; // 函数内部的静态变量
    int funcNormalVar = 10;        // 函数内部的普通局部变量

    // 在函数体内部定义局部类
    class LocalClass {
    public:
        // 成员函数必须在类内部实现
        void display() {
            cout << "访问全局变量: " << globalVar << endl;
            cout << "访问函数的静态变量: " << funcStaticVar << endl;
            
            // ⚠️ 下面这行如果取消注释会报错，局部类不能访问函数的普通局部变量
            // cout << "函数的普通变量: " << funcNormalVar << endl; 
        }
    };

    // 只能在该函数内部实例化并使用局部类
    LocalClass lc;
    lc.display();
}

int main() {
    cout << "调用函数开始：" << endl;
    myFunction();
    
    // LocalClass lc2; // 错误！出了 myFunction，LocalClass 就不存在了，无法访问

    return 0;
}
```
