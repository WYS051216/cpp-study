# C++ 现代并发编程笔记：线程间的数据快递 (promise 与 future)

---

## 一、 核心概念：一对一的数据管道

在多线程编程中，最头疼的问题就是：“主线程派了一个子线程去干活，子线程干完活之后，怎么把结果安全地交回给主线程？” 

`std::promise` 和 `std::future` 就是专门为了解决跨线程数据传输而生的。它们俩必须成对出现，在底层共享同一个“状态通道”。我们可以用**“寄快递”**来完美比喻它们：

### 1. `std::promise` (快递发件人 / 生产者)
* **作用**：它存在于**干活的子线程**中，负责将计算结果或者异常装进通道里。
* **核心方法**：
  * `.set_value(数据)`：活干完了，把结果打包发出去。
  * `.set_exception(...)`：活干砸了（发生异常），把报错信息打包发出去，让主线程知道。

### 2. `std::future` (取件凭证 / 消费者)
* **作用**：它存在于**等待结果的主线程**中。主线程手里拿着这个凭证，随时可以去取结果。
* **核心方法**：
  * `.get()`：**最关键的函数！** 拿着凭证去取货。如果快递还没送到（子线程还在算），调用 `.get()` 会让当前线程**立刻挂起（阻塞等待）**，直到数据到了才会被唤醒并拿到数据。
  * *注意*：一个 `future` 的 `.get()` 只能被调用一次，取完货凭证就作废了。

## 二、 代码示例：跨线程安全获取数据

下面这段代码展示了如何利用 `promise` 和 `future`，让主线程优雅地等待子线程的计算结果，全程不需要写哪怕一行令人头秃的 `mutex`（锁）代码。

```cpp
#include <iostream>
#include <thread>
#include <future>
#include <chrono>

// 模拟一个极其耗时的后台计算任务
// 注意：promise 是独占的资源，不能拷贝，只能通过 std::move 移动传参
void complexCalculation(std::promise<int> prom) {
    std::cout << "[子线程] 收到任务，开始吭哧吭哧计算..." << std::endl;
    
    // 模拟耗时 2 秒
    std::this_thread::sleep_for(std::chrono::seconds(2)); 

    try {
        int final_result = 42; // 假设这是经过复杂计算得出的结果
        
        // 【核心】将计算结果塞入管道
        prom.set_value(final_result); 
        std::cout << "[子线程] 计算完成，数据已发货！" << std::endl;
        
    } catch (...) {
        // 如果中间崩溃了，可以将异常传给主线程
        prom.set_exception(std::current_exception());
    }
}

int main() {
    // 1. 创建一个发件人 (promise)
    std::promise<int> my_promise;

    // 2. 提前从发件人那里获取对应的取件凭证 (future)
    std::future<int> my_future = my_promise.get_future();

    // 3. 启动子线程去干活！
    // 必须用 std::move 把 promise 的所有权转移给子线程
    std::thread worker(complexCalculation, std::move(my_promise));

    // 4. 主线程可以先干点别的事
    std::cout << "[主线程] 安排好子线程了，我先泡杯咖啡..." << std::endl;
    std::cout << "[主线程] 准备去取结果..." << std::endl;

    // 5. 【核心】提取结果。如果子线程还没发货，这里会死等（阻塞）
    try {
        int result = my_future.get(); 
        std::cout << "[主线程] 成功拿到数据！结果是: " << result << std::endl;
    } catch (const std::exception& e) {
        std::cout << "[主线程] 糟糕，子线程抛出了异常: " << e.what() << std::endl;
    }

    // 6. 善后工作：等待子线程彻底销毁
    worker.join(); 
    
    return 0;
}
```

### 💡 进阶小贴士：`std::async`
虽然 `promise` 和 `future` 配合 `thread` 已经很优雅了，但在很多简单场景下，C++ 还提供了一个终极偷懒神器：**`std::async`**。
它可以帮你把“创建线程 + 创建 promise + 返回 future”这三步全自动打包完成！如果你只是想“丢一个函数到后台运行，然后等它返回结果”，优先查阅 `std::async` 的用法，代码还能再精简一半！
