# C++20 现代特性笔记（一）：模块 (Modules) 与 协程 (Coroutines)

---

## 一、 模块 (Modules)：终结头文件的历史

### 1. 传统 `#include` 机制的痛点
C++ 继承了 C 语言的 `#include` 机制，本质上是极其粗暴的**文本复制粘贴**。这在大型项目中引发了三大致命问题：
1. **不够卫生**：头文件里包含的宏定义（Macro）会不受控制地泄漏到引用它的所有文件中，导致各种奇怪的命名冲突。
2. **分离编译的不一致性**：头文件和源文件分离，如果定义和实现稍微不一致，极易在链接期报错。
3. **编译次数过多（极慢）**：如果一个头文件被 100 个源文件 `#include`，编译器就要把这个头文件重复解析 100 次！

### 2. 模块 (Modules) 的核心优势
模块是一个用于在翻译单元间分享声明和定义的全新语言特性：
* **没有头文件**：不再需要 `.h` 和 `.cpp` 的强制分离。
* **按需导出**：可以显式指定导出哪些类或函数，未导出的内容对外绝对不可见（完美封装）。
* **宏的隔离**：预处理宏只在模块内部有效，绝对不会污染引入该模块的其他代码。
* **极速编译**：模块只会被编译器处理一次，生成预编译的二进制格式，其他文件引入时直接读取，编译速度呈指数级提升。
* **顺序无关**：引入模块的顺序再也不会影响编译结果。
* **无名称冲突**：不同模块内部可以有同名的函数，互不干扰。

### 3. 创建模块的步骤与细节
**步骤 1：创建模块接口文件**
不同编译器对模块文件的后缀名有不同偏好（MSVC 常用 `.ixx`，Clang/GCC 常用 `.cppm`）。
在此文件中，使用 `export module 模块名;` 声明这是一个模块，并使用 `export` 关键字暴露出你想让别人用的函数或类。

**步骤 2：在主程序中导入模块**
使用全新的 `import 模块名;` 语法，代替古老的 `#include`。

### 4. 代码示例

**文件 A: `math_module.cppm` (模块接口与实现)**
```cpp
// 声明这是一个名为 MyMath 的模块
export module MyMath;

// 模块内部的私有辅助函数（没有 export，外界绝对调不到）
int internal_add(int a, int b) {
    return a + b;
}

// 导出函数，外界可以使用
export int add(int a, int b) {
    return internal_add(a, b);
}

// 导出整个类
export class Calculator {
public:
    int multiply(int a, int b) { return a * b; }
};
```

**文件 B: `main.cpp` (调用模块)**
```cpp
import <iostream>;
import MyMath; // 全新语法：导入模块

int main() {
    std::cout << "加法结果: " << add(5, 3) << std::endl; // 成功调用
    
    Calculator calc;
    std::cout << "乘法结果: " << calc.multiply(4, 2) << std::endl;

    // internal_add(1, 2); // ❌ 编译报错：找不到该标识符，完美隐藏！
    return 0;
}
```
*(注意：编译模块目前需要较新的编译器，并加上 `-std=c++20 -fmodules-ts` 等专门的编译指令。)*

## 二、 协程 (Coroutines)：异步编程的终极答案

### 1. 概念补充
**协程是一个可以挂起（Suspend）和恢复（Resume）的特殊函数。**
* 它可以让你像写**同步代码**一样，去写极其复杂的**异步代码**（比如网络请求、文件读写）。
* **无栈协程**：C++20 的协程是轻量级的“无栈协程”（Stackless），它挂起时状态保存在堆内存中，而不是占用昂贵的线程栈。
* **调度关系**：协程本身并不会抢占内核调度，它只是函数调用的泛化，最终依然是由线程来执行它。
* **限制**：`main` 函数不能是协程。

**💡 核心痛点说明：**
C++20 仅仅提供了协程的**底层机制/骨架**（比如 `std::coroutine_handle` 等极其底层的概念），**并没有提供现成好用的协程库**。在日常开发中，我们通常需要结合第三方库（如 `cppcoro`、`asio`）或等 C++23 的标准库来使用它们。

### 2. 三个核心新关键字的作用

为了控制协程的暂停与执行，C++20 引入了三个极其强大的关键字：

* **`co_await`（挂起并等待）**：
  * **作用**：用于暂停当前协程的执行，直到其等待的异步任务（比如网络下载）完成。
  * **大白话**：“我在这里歇会儿，等那个耗时的活儿干完了，你再把我叫醒，我接着往下跑。”

* **`co_yield`（产出并挂起）**：
  * **作用**：向调用者返回一个值，然后**暂停**协程。下一次调用时，从暂停的地方继续往下走。非常适合用来编写“生成器（Generator）”（比如无限产生斐波那契数列）。
  * **大白话**：“这是你要的一个数据，先给你，我先睡了。你下次再找我要，我再给你算下一个。”

* **`co_return`（彻底结束）**：
  * **作用**：用于在协程中代替传统的 `return`，返回最终结果并彻底终结协程的生命周期。
  * **大白话**：“我的活儿全干完了，这是最终结果，以后别来叫我了。”

### 3. 概念性代码示例 (生成器)
由于实现一个完整的 C++20 协程底层结构极其冗长（需要手写 `promise_type` 等底层构件），这里展示一段高层次的**伪代码/概念代码**，让你体会协程的美感：

```cpp
#include <iostream>
// 假设这里引入了一个封装好的生成器库，比如 <generator> (C++23) 或 cppcoro

// 这是一个协程，因为它内部使用了 co_yield
Generator<int> generateNumbers() {
    std::cout << "协程开始执行..." << std::endl;
    co_yield 1; // 产出 1，然后立刻挂起暂停！
    
    std::cout << "协程被唤醒，继续执行..." << std::endl;
    co_yield 2; // 产出 2，再次挂起暂停！
    
    std::cout << "协程准备结束..." << std::endl;
    co_return;  // 彻底结束
}

int main() {
    auto gen = generateNumbers(); // 此时协程还没有开始跑！

    for (int val : gen) { 
        // 每次循环迭代，都会唤醒协程，跑到下一个 co_yield 处拿值
        std::cout << "主函数收到: " << val << std::endl;
    }
    
    return 0;
}
```
