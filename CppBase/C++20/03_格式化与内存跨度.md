# C++20 现代特性笔记：格式化库 `<format>` 与内存跨度视图 `<span>`

---

## 一、 格式化库 (std::format)

### 1. 概念补充与痛点解决
在 C++20 之前，我们组装字符串通常面临极其纠结的二选一：
* **`printf` 家族**：写法极其简洁（如 `"%s is %d"`），速度极快。但它**类型不安全**，如果你把 `%d` 对应到了一个字符串变量上，程序运行时直接崩溃。
* **`std::stringstream` / `cout`**：类型绝对安全，但**写法极其丑陋冗长**，疯狂使用 `<<` 拼接，而且速度很慢。

C++20 引入了 `<format>` 库（深受 Python 的 `str.format` 启发），它完美结合了这两者的优点：
* **类 `printf` 的极简写法**：使用 `{}` 作为占位符，代码阅读极其顺畅。
* **绝对的类型安全**：编译器会自动推导 `{}` 里的变量类型，再也不用去记 `%d`、`%f`、`%s` 了。
* **极高的性能**：在底层直接操作缓冲区，性能碾压传统的流输出。

### 2. 代码示例
```cpp
#include <iostream>
#include <string>
#include <format> // 必须引入此头文件

int main() {
    std::string name = "亚瑟";
    int level = 18;
    double health = 2500.55;

    // 1. 基础用法：全自动类型推导，按顺序填入 {}
    std::string result = std::format("玩家 {} 的等级是 {}，血量为 {}", name, level, health);
    std::cout << result << std::endl;

    // 2. 进阶用法：指定填入顺序（适合国际化翻译）
    // {1} 代表取第二个参数 (level)
    std::string order_result = std::format("等级 {1} 属于玩家 {0}", name, level);
    std::cout << order_result << std::endl;

    // 3. 高级格式化控制（例如：保留一位小数、对齐、转换十六进制）
    // {:.1f} 表示保留一位小数，{:X} 表示大写十六进制
    std::string advanced = std::format("血量: {:.1f}, 等级(Hex): {:X}", health, level);
    std::cout << advanced << std::endl;

    return 0;
}
```

## 二、 连续内存跨度 (std::span)

### 1. 概念补充与痛点解决
在传统的 C/C++ 中，有一个臭名昭著的问题叫做**“数组退化（Array Decay）”**。
当你把一个数组 `int arr[10]` 传给一个函数 `void process(int* p)` 时，它立刻就退化成了一个光秃秃的指针。函数内部**根本不知道这个数组到底有多长**，极其容易发生“越界访问（缓冲区溢出）”。过去为了解决这个问题，我们被迫每次都要多传一个参数：`void process(int* p, size_t length)`。

C++20 引入了 `std::span<T>`，它是一个极其轻量级的**非拥有型视图（Non-owning view）**：
* **结构极简**：它的底层仅仅封装了**一个指针**和**一条大小信息（长度）**。
* **自动打包**：无论你传进来的是普通 C 数组、`std::vector` 还是 `std::array`，它都能**全自动**提取首地址和长度，完美打包成一个对象。
* **零成本抽象**：它不拥有这段内存空间，绝对不拷贝数据，传递它就像传递两个数字一样快。

### 2. 代码示例
```cpp
#include <iostream>
#include <vector>
#include <array>
#include <span> // 必须引入此头文件

// 现代 C++ 推荐写法：统一使用 std::span 接收连续内存数据
// 它能自动记住传入数据的长度，彻底消灭“数组退化”问题！
void printData(std::span<int> data) {
    std::cout << "接收到数据，长度为: " << data.size() << " -> ";
    
    // 因为 span 内部记录了 size，所以可以极其安全地使用范围 for 循环
    for (int val : data) {
        std::cout << val << " ";
    }
    std::cout << std::endl;
}

int main() {
    // 数据源 1：传统的 C 风格数组
    int raw_array[] = {1, 2, 3};
    
    // 数据源 2：标准库的动态数组 vector
    std::vector<int> vec = {4, 5, 6, 7};
    
    // 数据源 3：标准库的静态数组 array
    std::array<int, 2> std_arr = {8, 9};

    // 极其震撼的地方：同一个函数，通吃所有连续内存容器！
    // 编译器会自动把底层指针和长度提取出来，组装成 span 传进去。
    printData(raw_array); 
    printData(vec);       
    printData(std_arr);   

    // 还可以只截取其中一部分传递（视图切片）
    // std::span 的 .subspan(起点, 长度) 可以极速截取
    std::span<int> sub_view = std::span{vec}.subspan(1, 2);
    std::cout << "--- 截取视图测试 ---" << std::endl;
    printData(sub_view); // 仅输出 5 6

    return 0;
}
```
