# C++20 现代特性笔记（二）：飞船运算符、范围库与日历时区

## 一、 三向比较运算符 `<=>` (飞船运算符)

### 1. 概念补充与痛点解决
在过去，如果你自己写了一个类（比如 `struct Point { int x, y; };`），为了让它能支持排序或比较，你必须极其痛苦地手动重载 6 个运算符（`==`, `!=`, `<`, `<=`, `>`, `>=`）。这不仅代码极其臃肿，而且极易写出 Bug。

C++20 引入了 `<=>`，因为它长得像科幻电影里的 UFO，所以被亲切地称为**“飞船运算符”**。
* **返回值**：它的返回值不是 `true` 或 `false`，而是专门的比较结果类（类似于 C 语言里的 `strcmp()` 返回 `-1, 0, 1`）。
  * `(a <=> b) < 0`：表示 a 小于 b
  * `(a <=> b) == 0`：表示 a 等于 b
  * `(a <=> b) > 0`：表示 a 大于 b
* **终极杀手锏：`= default`**：只要你在类里写一行 `auto operator<=>(const ClassName&) const = default;`，编译器就会**全自动为你生成所有 6 个比较运算符**！

### 2. 代码示例
```cpp
#include <iostream>
#include <compare> // 必须引入此头文件

struct Player {
    int level;
    int score;

    // C++20 神奇的一行代码：自动生成全部 6 种比较运算！
    // 默认比较规则：先比 level，如果 level 相同再比 score。
    auto operator<=>(const Player&) const = default; 
};

int main() {
    Player p1{10, 500};
    Player p2{10, 600};
    Player p3{10, 500};

    // 1. 直接使用传统运算符（编译器通过 <=> 自动推导）
    if (p1 < p2) { std::cout << "p1 实力弱于 p2" << std::endl; }
    if (p1 == p3) { std::cout << "p1 和 p3 实力完全一样" << std::endl; }

    // 2. 显式使用飞船运算符，获取底层三向比较结果
    auto result = (p1 <=> p2);
    if (result < 0) { // 类似于 strcmp 返回 -1
        std::cout << "底层判定：p1 < p2" << std::endl;
    }

    return 0;
}
```

## 二、 范围库 (Ranges)

### 1. 概念补充与痛点解决
在 C++20 之前，调用标准库算法极其繁琐。例如对一个 `vector` 排序，你必须写 `std::sort(vec.begin(), vec.end());`。如果需要对数据进行过滤、变形，往往要写好几个冗长的循环，不仅容易写错边界，还会产生一堆临时变量。

C++20 的 `<ranges>` 彻底革新了数据处理的方式：
* **彻底告别迭代器对**：可以直接传容器。`std::ranges::sort(vec);` 一步搞定！
* **视图 (Views) 与管道操作符 `|`**：引入了类似于 Linux 命令行或者 Python 的链式管道。你可以用 `|` 把多个操作（过滤、转换等）串起来。
* **延迟计算 (Lazy Evaluation)**：视图并不产生新的数据拷贝，它只是戴上了“有色眼镜”去看原数据，直到你真正遍历它时才进行计算，极其节省内存。

### 2. 代码示例
```cpp
#include <iostream>
#include <vector>
#include <ranges>

int main() {
    std::vector<int> nums = {1, 2, 3, 4, 5, 6, 7, 8};

    // --- 痛点解决 1：极简排序 ---
    // 以前：std::sort(nums.begin(), nums.end(), std::greater<>());
    std::ranges::sort(nums, std::greater<>()); 

    // --- 痛点解决 2：管道式组合操作（极其优雅） ---
    std::cout << "将偶数筛选出来，并计算平方: " << std::endl;
    
    auto resultView = nums 
        | std::views::filter([](int n) { return n % 2 == 0; }) // 步骤1：过滤出偶数
        | std::views::transform([](int n) { return n * n; });  // 步骤2：将偶数平方

    // 遍历视图（此时才真正触发上述的筛选和平方计算）
    for (int val : resultView) {
        std::cout << val << " "; 
    }
    // 输出: 64 36 16 4 (因为前面从大到小排过序了)

    return 0;
}
```

## 三、 日期和时区 (Chrono 增强)

### 1. 概念补充与痛点解决
C++11 虽然引入了 `<chrono>` 库，但它里面只有“时间段（比如 5秒）”和“时间点（比如程序启动后运行了多久）”，却唯独没有**人类看得懂的日历和时区**！以前要在 C++ 里处理真实的年月日和跨国时区极其痛苦。

C++20 对 `<chrono>` 进行了史诗级增强：
* **日历类型**：新增了 `year`, `month`, `day`, `year_month_day` 等极其直观的类型。
* **极其易读的字面量语法**：甚至可以直接用 `/` 运算符来拼接日期！
* **时区支持**：新增 `zoned_time` 和系统时区数据库，支持完美跨时区转换（再也不用怕夏令时了）。
* **格式化输出**：可以配合 `std::format` 直接输出人类友好的格式。

### 2. 代码示例
```cpp
#include <iostream>
#include <chrono>

// 引入 chrono 的命名空间和字面量支持 (比如 2023y)
using namespace std::chrono;
using namespace std::chrono_literals;

int main() {
    // 1. 极其优雅的日期拼接法 (使用 / 运算符)
    // 语法: 年 / 月 / 日
    year_month_day date = 2023y / October / 15d;
    
    // 你甚至可以这样写：找出 2023 年 2 月的最后一天
    year_month_day last_day = 2023y / February / last;
    
    std::cout << "设定日期: " << date.year() << "-" << unsigned(date.month()) << "-" << date.day() << std::endl;

    // 2. 获取当前时间并带上时区
    // 获取当前的系统绝对时间 (UTC时区)
    auto now = system_clock::now(); 

    // 获取当前系统的本地时区，并将 UTC 时间转换过去
    const time_zone* current_tz = current_zone();
    zoned_time local_time{current_tz, now};

    // 打印出来的就是你所在地的真实时间（注意：C++20 的输出通常推荐搭配 std::format）
    // 为了兼容纯控制台打印，可以直接通过 local_time 拿到信息
    std::cout << "当前所在地时区: " << current_tz->name() << std::endl;
    // 实际项目中，配合 std::format 打印时间字符串最为强大

    return 0;
}
```
