# 📝 C/C++ 指针核心知识点整理

## 1. 指针的作用
* **核心目的**：间接访问内存。
* **机制**：通过指针保存的内存地址，可以找到并操作内存中特定位置的数据。

## 2. 定义与初始化指针
```cpp
int a = 10;
int* p;     // 1. 定义一个整型指针变量 p
p = &a;     // 2. 将变量 a 的内存地址赋给 p

// 以上两行等价于：
int* p = &a; 
```
* **原理解析**：`p` 里面保存的内容和 `&a` 是一模一样的，**都是 `a` 的物理内存地址**。

## 3. 使用指针（解引用）
* **操作符**：在指针变量前加 `*` 代表**解引用**（去这个地址里掏数据）。
* **修改数据**：
```cpp
*p = 20; // 顺着指针 p 找到 a 的内存，将其修改为 20。此时 a 的值也变成了 20。
```

> 🚨 **纠错警告**：
> 原笔记中提到的“通过指针的解引用修改内容，这就是浅拷贝”是**完全错误**的！
> 这仅仅是正常的**“通过指针修改同一块内存”**。真正的**浅拷贝**是指两个不同的指针变量保存了相同的地址（例如 `int* p2 = p;`），导致它们指向同一块堆内存，在释放（`delete`）时容易引发重复释放（Double Free）导致程序崩溃。

## 4. 指针所占内存空间
指针本身也是一个变量，用来存地址，因此它也占用内存空间。
* **32位操作系统下**：无论什么类型的指针（`int*`, `double*`, `char*` 等），统统占 **4 个字节**。
* **64位操作系统下**：无论什么类型的指针，统统占 **8 个字节**。
> **底层补充**：这与计算机体系结构中 CPU 的寻址能力（地址总线宽度）直接相关。

## 5. 空指针和野指针（⚠️ 高危）
* **空指针 (Null Pointer)**：指针变量指向内存中编号为 0 的空间。
  * **用途**：用于初始化指针变量，表示该指针目前“哪里都没指”。
  * **注意**：空指针指向的内存（0~255系统保留区）**绝对不允许访问**，一旦对其解引用（`*p`），程序会立刻引发段错误崩溃。
  * **现代 C++ 补充**：强烈建议使用 C++11 引入的 `nullptr` 代替传统的 `NULL` 或 `0` 来初始化空指针（如：`int* p = nullptr;`）。
* **野指针 (Wild Pointer)**：指针变量指向了非法的、未知的内存空间（如未初始化的指针，或指向了已被释放的内存）。
#### 1. 怎样用才算是野指针？
野指针是指**指向了一个“不合法”或“不属于你管辖”的内存地址**的指针。使用野指针之所以危险，是因为它不会像空指针那样 100% 稳定崩溃，它可能会暗中改写其他数据，导致程序出现极难排查的“玄学 Bug”。
```cpp
int* p;
```

以下是开发中最容易制造野指针的**三大重灾区**：

* **情况 A：指针定义后未初始化**
    局部指针变量如果不初始化，它的值是一个随机的垃圾地址（在 Debug 模式下可能是特定标记，但在 Release 模式下完全随机）。此时它指向哪里没人知道。
* **情况 B：所指内存已被释放（Dangling Pointer / 悬空指针）**
    使用 `delete` 或 `free` 释放了一块堆内存后，**指针变量本身的值（即那个内存地址）并不会被自动抹去**。它依然顽固地指着那块已经被系统回收的内存。如果你再次去读写它，就是在访问非法内存。
* **情况 C：指针指向了已被销毁的局部变量**
    当一个函数执行完毕后，它的局部变量在栈内存中会被自动销毁。如果你的指针指向了这类局部变量，那么在函数结束后，这个指针就变成了野指针。

---

#### 2. 代码示例：三大野指针场景排查

```cpp
#include <iostream>

// 制造情况 C 的辅助函数
int* getLocalAddress() {
    int temp = 100; // 局部变量，存放在栈区
    return &temp;   // ❌ 极其危险：返回局部变量的地址
} // 函数结束，temp 占用的内存被系统收回

int main() {
    // ----------------------------------------------------
    // 场景 A：未初始化的指针
    // ----------------------------------------------------
    int* p1; 
    // ❌ 此时 p1 是野指针，它指向一个随机的不可知地址
    // *p1 = 10; // ❌ 极其危险：可能会导致程序直接崩溃，或者暗中破坏其他数据


    // ----------------------------------------------------
    // 场景 B：释放内存后未置空的指针 (悬空指针)
    // ----------------------------------------------------
    int* p2 = new int(42); // 成功在堆区申请内存
    std::cout << "释放前的值: " << *p2 << std::endl;

    delete p2; 
    // ❌ 此时 p2 变成了野指针！内存虽然释放了，但 p2 里面还记录着那个地址。
    
    // std::cout << *p2 << std::endl; // ❌ 危险：此时访问的行为是未定义的（可能拿到垃圾值，也可能崩溃）

    // ✅ 正确防御手段：释放后必须立刻将指针洗干净
    p2 = nullptr; 


    // ----------------------------------------------------
    // 场景 C：指向已销毁的局部变量
    // ----------------------------------------------------
    int* p3 = getLocalAddress(); 
    // ❌ 此时 p3 是野指针。因为 getLocalAddress 已经执行完毕，temp 已经被销毁了。
    
    // std::cout << *p3 << std::endl; // ❌ 危险：指向了一块已经交公的栈空间

    return 0;
}
```

---

#### 3. 终结野指针的黄金三原则
为了彻底在以后的开发中远离野指针，请务必养成以下肌肉记忆：
1. **定义指针时必初始化**：如果一时间不知道让它指哪，统一写 `int* p = nullptr;`。
2. **释放内存后必位置空**：只要写了 `delete p;`，紧接着后面必须写 `p = nullptr;`。
3. **绝不返回局部变量的指针或引用**。

## 6. const 修饰指针 (经典考点)
> 💡 **防晕口诀**：看 `const` 右边紧挨着的是什么。紧挨着 `*`，就是指向的值不能改；紧挨着指针名，就是指针的指向不能改。

* **① 常量指针（`const` 修饰指针指向的值）**
  * `const int* p = &a;`
  * **特点**：指针的**指向可以修改**，但是**不能通过 `*p` 去修改**它指向的值。
* **② 指针常量（`const` 修饰指针变量本身）**
  * `int* const p = &a;`
  * **特点**：指针的**指向不可以修改**（终生绑定），但可以通过 `*p` 修改它指向的值。
* **③ 既修饰指针，又修饰常量**
  * `const int* const p = &a;`
  * **特点**：指针的指向、指针指向的值，**统统不可以修改**。
---
PS:
```cpp
int a = 10;
const int* p = &a; // p 答应不修改 a
a = 20;            // 但是 a 本身不是常量，可以自己改！
cout << *p;        // 此时打印出来是 20
```

**本质理解：**

`const int*` 只是给这个指针颁发了一个**“只读许可证”**。

指针 `p` 只能保证**“我不去修改你”**，但如果别人（比如 `a` 自己，或者别的普通指针）去修改了那块内存，`p` 看到的值照样会变。

**编译器的强制要求：谁必须初始化？**

因为 `const` 的特性就是“一旦赋值终生不变”，所以在 C++ 中，**只要涉及到修饰变量本身是常量，就必须在定义时初始化**。

* **常量指针：** `const int* p;` —— **合法**。（指针本身可以变，以后再指别人也行）
* **指针常量：** `int* const p;` —— **报错！**（指针本身是常量，必须立刻绑定，比如 `int* const p = &a;`）
* **双重 const：** `const int* const p;` —— **报错！**（同理，必须立刻初始化）

## 7. 指针和数组
* **作用**：利用指针遍历和访问数组中的每一个元素。
* **核心机制补充**：在 C++ 中，**数组名本身就是一个指向数组首元素地址的常量指针**（即 `arr` 等价于 `&arr[0]`）。
```cpp
int arr[5] = {1, 2, 3, 4, 5};
int* p = arr; // p 指向数组的第一个元素
p++;          // 指针向后偏移，指向第二个元素
```

## 8. 指针和函数

* **作用**：利用指针作为函数形参，可以在函数内部直接修改函数外部实参的值。
* **值传递**：传入的是变量的副本。在函数内修改形参，**不会影响**外部的实参。
* **地址传递**：调用函数时，传入变量的地址（变量前面加 `&`）；函数定义时，形参必须写成指针类型接收。在函数内通过解引用（`*`）操作，即可**修改外部实参的原件**。

#### 💻 代码示例：值传递 vs 地址传递

**1. 值传递 (Pass by Value) —— 失败的交换**
传入的是“复印件”，函数内部的修改对外部毫无影响。

```cpp
#include <iostream>
using namespace std;

// 形参是普通的 int 变量
void swapByValue(int a, int b) {
    int temp = a;
    a = b;
    b = temp;
}

int main() {
    int x = 10;
    int y = 20;
    
    // 直接传入变量名
    swapByValue(x, y); 
    
    // 输出: x = 10, y = 20 (外部毫无变化)
    cout << "值传递: x = " << x << ", y = " << y << endl; 
    return 0;
}
```

**2. 地址传递 (Pass by Address) —— 成功的交换**
传入的是“钥匙（地址）”，函数内部通过解引用 `*` 直接修改外部的原件。

```cpp
#include <iostream>
using namespace std;

// 形参必须是指针类型 (int*)
void swapByAddress(int* p1, int* p2) {
    int temp = *p1; // 顺着 p1 拿到 x 的值
    *p1 = *p2;      // 把 y 的值覆写到 x 的房间
    *p2 = temp;     // 把 temp 里的值覆写到 y 的房间
}

int main() {
    int x = 10;
    int y = 20;
    
    // 必须传入变量的地址 (前缀加 &)
    swapByAddress(&x, &y); 
    
    // 输出: x = 20, y = 10 (外部成功交换)
    cout << "地址传递: x = " << x << ", y = " << y << endl; 
    return 0;
}
```
