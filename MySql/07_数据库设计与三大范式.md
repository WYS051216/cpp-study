# 数据库设计与三大范式核心笔记

## 1. 数据库设计基础
* **实体 (Entity)**: 就是在软件开发过程中所涉及到的事物，通常都是一类数据对象的个体。
* **数据库设计**: 将实体与实体之间的关系进行规划和结构化的过程。好的数据库设计可以减少不必要的数据冗余。
* **设计步骤**:
  1. 收集信息。
  2. 标识实体。
  3. 标识实体的详细属性。
  4. 标识实体之间的关系。

## 2. ER 图 (实体关系图)
* **矩形** $\square$：表示实体。
* **椭圆** $\circ$：表示实体属性。
* **菱形** $\diamond$：表示实体关系。

## 3. 数据库模型图
* **关系模式**: 实体关系的描述称为关系模式。例如：`学生(学号, 姓名, 性别...)`
* **关系模式转换为数据库模型图步骤**:
  1. 将每个实体转换为对应的表，将实体属性转换为表对应的列。
  2. 标识出表的主键。
  3. 在表之间建立外键，体现可见性。

---

## 4. 数据库三大范式 (Normal Forms)
规范化数据库设计的基本规则：

* **第一范式 (1NF)**: 
  最基本的范式。**确保每个字段保持原子性**，也就是说字段是不可再分割的。

* **第二范式 (2NF)**: 
  在第一范式基础上，**确保表中的属性完全依赖于主键**。也就是要确保表只描述一件事（不能是组合主键中的一部分属性只依赖于部分主键）。

* **第三范式 (3NF)**: 
  在第二范式基础上，**确保属性直接依赖于主键，而不是间接依赖于主键**。不能存在传递依赖（即非主键列不能依赖于另一个非主键列）。s
