# SQL 数据增删改 (DML) 核心笔记

## 1. 数据操作 (DML)
* **INSERT (插入)**:
    * **指定列插入**: `INSERT INTO 表名 (字段1, 字段2...) VALUES (值1, 值2...);`
    * **全表插入**: `INSERT INTO 表名 VALUES (值1, 值2...);` （注意：值的顺序必须与表结构完全一致）
    * **一次性插入多条数据**: 
      `INSERT INTO 表名 (字段1,...) VALUES (值1,...), (值1,...);`
      或 `INSERT INTO 表名 VALUES (值1,...), (值1,...);`
* **UPDATE (更新)**: 
    * **语法**: `UPDATE 表名 SET 字段1=值1, 字段2=值2... [WHERE 条件];`
    * *注意：可以同时修改多个字段，中间用逗号隔开；如果不加 WHERE 条件，会更新整张表的所有数据！*
* **DELETE (删除)**: 
    * **语法**: `DELETE FROM 表名 [WHERE 条件];`
    * *注意：如果不加 WHERE 条件，会删除整张表的所有记录！*

---

## 2. 数据删除与清空 (DELETE vs TRUNCATE)
* **`DELETE FROM 表名 [WHERE 条件];`** (DML)
  * 可以带条件删除特定行。
  * 删除操作会被记录在日志中，执行后**可以使用事务回滚**。
* **`TRUNCATE TABLE 表名;`** (DDL)
  * 直接清空整张表的数据，重置自增计数器。
  * 速度比 DELETE 快，但不写详细日志，**执行后不可回滚**。
