## 1. 容器数据卷 (Data Volumes)

### 1.1 基本概念与核心作用
**概念：** 容器卷就是目录或文件，存在于一个或多个容器中，由 Docker 挂载到容器。它**不属于**联合文件系统（UnionFS），因此能够绕过 UnionFS 提供一些用于持续存储或共享数据的特性。
**作用：** * **数据持久化**：卷的设计目的就是持久化数据。它完全独立于容器的生命周期，即使容器实例被删除，挂载的数据卷也依然存在。
* **防止数据丢失**：如果不使用数据卷，Docker 容器产生的数据会随着容器的删除而自然消失。为了保存核心数据，必须使用数据卷。

### 1.2 数据卷的四大特点
1. **共享与重用**：数据卷可在不同容器之间共享或重用数据。
2. **实时生效**：卷中的更改可以直接、实时生效（宿主机变，容器变；容器变，宿主机也变）。
3. **独立更新**：数据卷中的更改不会被包含在镜像的更新或提交（commit）中。
4. **持久存在**：数据卷的生命周期一直持续，直到没有容器使用它为止。

### 1.3 挂载实战基本案例
在宿主机和容器之间添加数据卷，主要通过 `run` 命令加上 `-v` 参数来实现。通常会加上 `--privileged=true` 赋予最高权限以防止挂载目录出现权限不足的报错。

| 挂载模式 | 终端指令语法 | 说明 |
| :--- | :--- | :--- |
| **默认双向挂载** | `docker run -it --privileged=true -v /宿主机绝对路径:/容器内绝对路径 镜像名` | 宿主机和容器双方都可以对该目录内的文件进行修改，且实时同步。 |
| **只读挂载 (ro)** | `docker run -it --privileged=true -v /宿主机绝对路径:/容器内绝对路径:ro 镜像名` | 加上 `:ro` (read-only) 后，**容器内部只能读取**该目录，没有修改权限。仅宿主机可以修改。 |
| **卷的继承 (共享)** | `docker run -it --privileged=true --volumes-from 父容器名 --name 子容器名 镜像名` | 启动子容器时，直接继承父容器的数据卷挂载规则，实现多个容器间的数据互通。 |

## 2. 常见软件安装实战 (以 Tomcat 为例)

在 Docker 中安装常用软件，基本都遵循“搜 -> 拉 -> 查 -> 跑”的标准流程。

**Step 1: 搜索镜像**
```bash
docker search tomcat
```

**Step 2: 拉取镜像**
```bash
docker pull tomcat
```

**Step 3: 检查本地镜像**
查看本地是否已经成功拉取到了 tomcat 镜像：
```bash
docker images tomcat
```

**Step 4: 创建并运行容器实例**
```bash
docker run -d -p 8080:8080 --name=t1 tomcat
```
> **💡 参数深度解析：**
> * `-d`：后台守护式运行。
> * `--name=t1`：给这个新运行的容器起个别名叫 `t1`。
> * `-p 8080:8080`：**端口映射 (`宿主机端口:容器内端口`)**。冒号左边的 `8080` 是**宿主机（你的物理机/云服务器）向外暴露的端口**，用户在浏览器输入的端口号就是这个；冒号右边的 `8080` 是 **Tomcat 容器内部本身运行的默认端口**。Docker 会将外界对宿主机 8080 的访问，自动转发到容器内部的 8080 端口上。

**Step 5: 验证访问与排错 (⚠️ Tomcat 必踩坑补全)**
运行成功后，在物理机浏览器输入 `http://宿主机IP:8080` 进行访问。
* **现象：** 你大概率会看到一个 **HTTP 404** 报错页面，提示找不到网页。
* **原因：** Docker 官方为了保证镜像轻量化，最新版的 Tomcat 镜像中默认把 `webapps` 目录清空了，默认的欢迎页面被放在了旁边的 `webapps.dist` 目录中。
* **解决方案**（进入容器，将文件拷回原位）：
  ```bash
  # 1. 进入运行中的 tomcat 容器
  docker exec -it t1 /bin/bash
  
  # 2. 将 webapps.dist 目录下的所有文件复制到 webapps 目录下
  cp -r webapps.dist/* webapps/
  
  # 3. 退出容器
  exit
  ```
执行完上述操作后，刷新浏览器页面，就能正常看到熟悉的 Tomcat 欢迎页（那只猫）了！
