# 🐳 Docker 核心基础命令速查笔记

---

## 一、 服务与系统管理类指令（Systemctl & Help）
这些命令主要用于在 Linux 系统层面控制 Docker 引擎（大管家）的启停。

| 操作目标 | 终端指令 |
| :--- | :--- |
| **启动** Docker 服务 | `systemctl start docker` |
| **停止** Docker 服务 | `systemctl stop docker` |
| **重启** Docker 服务 | `systemctl restart docker` |
| 查看 Docker **运行状态** | `systemctl status docker` |
| 查看 Docker **帮助文档** | `docker --help` |

## 二、 镜像管理命令（Image）
镜像（Image）是容器运行的只读模板。

### 1. 查看与搜索镜像
* **列出本地主机的镜像：** `docker images [选项]`
    * `-a`：列出所有镜像（含历史映像层）。
    * `-q`：静默模式，只显示镜像 ID。
    * *输出字段说明：* `REPOSITORY` (仓库源) | `TAG` (版本标签) | `IMAGE ID` (镜像ID) | `CREATED` (创建时间) | `SIZE` (大小)。
* **在仓库中搜索镜像：** `docker search [选项] 镜像名称`
    * `--limit int`：限制最多显示前几个搜索结果。
    * `--filter`：按条件过滤搜索结果。
    * `--format`：格式化输出。
    * `--no-trunc`：显示完整的镜像描述信息（不截断）。

### 2. 下载与删除镜像
| 操作目标 | 终端指令 | 说明 |
| :--- | :--- | :--- |
| **下载镜像** | `docker pull 镜像名称:版本号` | 不加版本号默认下载 `latest`（最新版）。 |
| **删除指定镜像** | `docker rmi -f 镜像名称或ID` | `-f` 代表强制删除；支持空格隔开同时删除多个。 |
| **清空所有镜像** | `docker rmi -f $(docker images -qa)` | 组合命令：将查出来的所有镜像 ID 传给删除命令。 |

## 三、 容器基本命令（Container）
容器（Container）是基于镜像运行起来的实体实例。

### 1. 启动与创建容器
**基础语法：** `docker run [选项] IMAGE [COMMAND] [ARG...]`

| 常用参数 | 作用说明 |
| :--- | :--- |
| `--name` | 为新启动的容器指定一个自定义名字。 |
| `-d` | **守护式运行**：后台运行容器并返回容器 ID。 |
| `-i` | **交互式运行**：以交互模式运行容器（通常与 `-t` 连用）。 |
| `-t` | **分配终端**：为容器重新分配一个伪输入终端（通常与 `-i` 连用）。 |
| `-P` | **随机端口映射**：将容器内部端口随机映射到宿主机端口。 |
| `-p` | **指定端口映射**：格式为 `宿主机端口:容器内端口`。 |

> **💡 经典组合（交互式启动）：**
> 希望启动一个带有终端的容器并直接在里面敲命令时使用：
> `docker run -it ubuntu /bin/bash` （`/bin/bash` 是希望交互式执行的 Shell 解释器）。

### 2. 查看容器状态
**基础语法：** `docker ps [选项]`

| 参数 | 作用说明 |
| :--- | :--- |
| **无参数** | 仅列出当前**正在运行**的容器。 |
| `-a` | 列出**所有**容器（包含正在运行和已经停止的）。 |
| `-l` | 仅罗列最近创建运行的一个容器。 |
| `-n 2` | 罗列最近创建运行的两个容器（数字可变）。 |
| `-q` | 静默模式，只显示容器的 ID。 |

### 3. 容器的退出与进入（⚠️ 核心易错点）

**① 退出当前正在交互的容器：**
* `exit`：退出终端，**同时容器也会停止运行**。
* `Ctrl + P + Q`（快捷键）：退出终端，**但容器保持在后台继续运行**（分离模式）。

**② 重新进入正在后台运行的容器：**
* `docker exec -it 容器ID /bin/bash`：**（强烈推荐）**在容器中打开新的终端。使用 `exit` 退出时，**容器不会停止**。
* `docker attach 容器ID`：直接进入容器启动命令的终端。使用 `exit` 退出时，**容器会停止**。

### 4. 容器运维与排错
| 操作目标 | 终端指令 |
| :--- | :--- |
| **启动已停止**的容器 | `docker start 容器ID或名称` |
| 查看容器的**运行日志** | `docker logs 容器ID` |
| 查看容器内的**进程信息** | `docker top 容器ID` |
| 查看容器的**底层详细配置** | `docker inspect 容器ID` |
