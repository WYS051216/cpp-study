## 一、 Docker 镜像底层原理与自定义构建

### 1. 镜像的“千层饼”架构：联合文件系统 (UnionFS)
Docker 镜像并不是一个单一的大文件，而是由多层文件系统叠加而成的分层系统。以 `docker pull tomcat` 为例，每个分层镜像都有单独的功能，最后汇聚成一个整体。

* **核心定义**：联合文件系统是一种分层、轻量级并且高性能的文件系统。它支持对文件系统的修改作为一次提交来一层层地叠加，同时可以将不同目录挂载到同一个虚拟文件系统下。UnionFS 是 Docker 镜像的基础。
* **对外透明（叠加特性）**：它可以一次同时加载多个文件系统，但从外面看起来，**只能看到一个统一的文件系统**。联合加载会把各层文件系统融合叠加，最终的文件系统会包含所有底层的文件和目录。
* **资源共享与继承**：镜像可以通过分层来进行继承。基于同一个基础镜像，可以制作各种不同的具体应用镜像，极大节约磁盘空间。

---

### 2. 镜像加载的底层逻辑：bootfs 与 rootfs
理解 Docker 为什么轻量且启动极快，关键在于区分以下两层：

| 核心组件 | 包含内容 | 作用与加载原理 |
| :--- | :--- | :--- |
| **底层引擎**<br>`bootfs`<br>*(Boot File System)* | 包含 `bootloader` (引导加载器) 和 `kernel` (系统内核) | 类似典型 Linux 刚启动时加载的引导系统。**当 boot 加载完成之后，整个内核就都在内存中了，此时系统会卸载 bootfs。**（*注：Docker 容器启动快，正是因为它直接复用宿主机的内核，无需重新加载 bootfs*）。 |
| **上层系统**<br>`rootfs`<br>*(Root File System)* | 包含典型 Linux 系统中的 `/dev`, `/proc`, `/bin` 等标准目录和文件 | 建立在 bootfs 之上。它其实就是各种不同的操作系统发行版（比如 Ubuntu, CentOS 等的纯文件目录环境）。 |

> **💡 核心铁律：读写权限区分**
> * **镜像 (Image)**：是底层的原始文件，永远是**不能修改的（只读）**。
> * **容器 (Container)**：当容器启动时，会在只读镜像的最顶层加上一层读写层，因此**容器是可以修改的**。

---

### 3. Docker 镜像 commit 操作（自定义打包）
当我们在容器内进行了自定义修改后，可以将其当前状态提交并打包成一个新的镜像。

**【实战案例：向纯净版容器内添加 Vim 功能并打包】**

**Step 1: 进入容器内部并安装功能**
```bash
# 更新源包列表
apt-get update
# 安装 vim 编辑器
apt-get -y install vim
```

**Step 2: 将该容器制作成新的镜像**
* **基本语法格式**：
  ```bash
  docker commit -m="提交的描述信息" -a="作者" 容器ID 要创建的目标镜像名:[标签名]
  ```
* **具体应用示例**：
  ```bash
  docker commit -m="add vim tool" -a="your_name" f8a9b2c3 my_ubuntu_vim:1.0
  ```
