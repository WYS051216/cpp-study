## 一、 镜像发布与分发（推送至云端与私服）

### 1. 将本地镜像推送到阿里云 (云端公有/私有库)
阿里云提供了非常稳定的容器镜像服务，适合将镜像托管在云端。

**前期准备（网页端操作）：**
1. 登录阿里云控制台，进入**“容器镜像服务 (ACR)”**。
2. 选择**“个人实例”**。
3. 进入个人实例，创建**“命名空间”**。
4. 在该命名空间下创建对应的**“镜像仓库”**。

**终端操作指令（以阿里云控制台提供的操作指南为准）：**

* **① 登录阿里云 Docker Registry**
  ```bash
  # 替换为你自己所在地域的地址，回车后输入阿里云 Registry 独立密码
  docker login --username=你的阿里云账号 registry.cn-hangzhou.aliyuncs.com
  ```
* **② 给本地镜像打上云端专属标签 (Tag)**
  ```bash
  # 语法：docker tag [本地镜像ID/名:版本] [阿里云仓库地址]/[命名空间]/[仓库名]:[新版本号]
  docker tag f8a9b2c3 [registry.cn-hangzhou.aliyuncs.com/my_space/my_project:1.0](https://registry.cn-hangzhou.aliyuncs.com/my_space/my_project:1.0)
  ```
* **③ 将镜像推送到阿里云**
  ```bash
  docker push [registry.cn-hangzhou.aliyuncs.com/my_space/my_project:1.0](https://registry.cn-hangzhou.aliyuncs.com/my_space/my_project:1.0)
  ```
* **④ 从阿里云拉取镜像（验证）**
  ```bash
  docker pull [registry.cn-hangzhou.aliyuncs.com/my_space/my_project:1.0](https://registry.cn-hangzhou.aliyuncs.com/my_space/my_project:1.0)
  ```

### 2. 将本地镜像推送到私有库 (内网 Docker Registry)
企业内部通常搭建私有镜像仓库以保证网络安全和内网拉取速度。

**① 搭建私服**
* **下载官方 Registry 镜像**：
  ```bash
  docker pull registry
  ```
* **运行私服容器**（映射端口并挂载数据卷以防镜像重启丢失）：
  ```bash
  docker run -d -p 5000:5000 -v /opt/data/registry:/var/lib/registry --name private-registry registry
  ```

**② ⚠️ 核心配置：配置 Docker 信任 HTTP 私服**
默认情况下 Docker 只允许向 HTTPS 仓库推送，内网私服通常是 HTTP 的，如果不配置信任，推送时会报错。
* **修改 daemon.json 配置文件**：
  ```bash
  # 在原有配置大括号内，增加 insecure-registries 节点
  {
    "registry-mirrors": ["[https://hub-mirror.c.163.com](https://hub-mirror.c.163.com)"],
    "insecure-registries": ["192.168.x.x:5000"] 
  }
  ```
* **重启 Docker 服务使配置生效**：
  ```bash
  systemctl daemon-reload
  systemctl restart docker
  ```

**③ 将镜像推送到私服库**
* **给本地镜像打上私服的标签 (Tag)**：
  ```bash
  # 语法：docker tag [本地镜像名:版本] [私服IP:5000]/[镜像名:版本]
  docker tag my_ubuntu:1.0 192.168.x.x:5000/my_ubuntu:1.0
  ```
* **执行推送命令**：
  ```bash
  docker push 192.168.x.x:5000/my_ubuntu:1.0
  ```

**④ 验证私服库镜像**
通过 `curl` 命令向私服 API 请求数据，查看是否成功包含了刚推送的镜像列表：
  ```bash
  curl -XGET [http://192.168.](http://192.168.)x.x:5000/v2/_catalog
  ```
