## 1. 虚拟机网络模式：NAT 网络连接原理

在使用 VMware 等虚拟机软件时，最常用的网络连接模式就是 **NAT (Network Address Translation，网络地址转换)** 模式。

### 1.1 核心组件解析（基于架构图）
从图中可以看出，数据要从虚拟机发送到公网（如 www.baidu.com），需要经过以下几个关键节点：

1.  **Linux 虚拟机 (`192.168.10.137`)**：
    这是我们在 VMware 中安装的系统，它被分配了一个虚拟的局域网 IP。
2.  **VMnet8 虚拟网卡 (`192.168.10.1`)**：
    这是 VMware 在 Windows 宿主机上虚拟出来的一个交换机/网关。
    * **互相通信**：Linux 虚拟机和 VMnet8 处于同一个网段（`192.168.10.x`），因此它们之间可以直接互通。对于 Linux 来说，VMnet8 就是它的“默认网关”。
3.  **Windows 本地地址 (`192.168.1.4`)**：
    这是你真实的物理电脑连接真实路由器（如家里 WiFi）获取到的 IP 地址。
4.  **网关与外网**：
    真实的物理路由器，负责将物理机的数据接入互联网（`www.baidu.com`）。

---

### 1.2 数据包的“越狱”流程（NAT 是如何工作的）

当你在 Linux 终端输入 `ping www.baidu.com` 时，底层发生了什么？

* **第一步（内部转发）**：Linux (137) 发现百度不在自己的局域网内，于是把数据包扔给它的网关 VMnet8 (1)。
* **第二步（地址转换 - 核心！）**：VMnet8 收到数据后，会将数据包交给 Windows 的物理网卡。在这个交接过程中，**VMware 的 NAT 服务偷偷把数据包的“源 IP 地址”从虚拟机的 `192.168.10.137` 替换成了 Windows 宿主机的 `192.168.1.4`**。
* **第三步（发送外网）**：真实路由器看到的是宿主机 (4) 发来的数据，于是正常转发给百度。
* **第四步（原路返回）**：百度将数据返回给宿主机，VMware 的 NAT 服务查阅内部的“转换记录表”，再把数据包准确地还给虚拟机 (137)。

---

### 1.3 NAT 模式的优缺点总结

* **优点（为什么推荐）**：
    * **不占用真实 IP**：无论你在电脑里开多少台虚拟机，真实路由器都只认为你这一台物理机在上网，不会造成物理局域网的 IP 冲突。
    * **配置极其简单**：只要宿主机（Windows）能上网，虚拟机就一定能上网（这也是为什么前面我们只需要把 Windows 的 VMware NAT 服务设置为自动启动，Linux 就有网了）。
    * **安全性高**：外部网络无法直接看到虚拟机的内部 IP，虚拟机被隐藏在宿主机背后。
* **缺点**：
    * 外部网络（包括真实局域网里的其他物理电脑）不能直接主动访问这台虚拟机，必须通过宿主机的端口映射才能连入。

### 2. Linux 网络配置的指令（以 Ubuntu 固化/静态 IP 为例）

在现代的 Ubuntu 系统（18.04 及以上版本）中，网络配置统一交由 **Netplan** 工具来管理，不再使用旧版的 `/etc/network/interfaces`。以下是配置固定（静态）IP 的完整步骤：

**步骤 1：确认当前网卡名称**
*   **指令**：输入 `ip a` 或 `ifconfig`。
*   **目的**：找到你正在使用的网卡名称（通常是有线网卡，如 `ens33`、`eth0` 等）。

**步骤 2：找到并编辑 Netplan 配置文件**
*   **指令**：`ls /etc/netplan/` （查看该目录下的 `.yaml` 配置文件名，比如 `01-network-manager-all.yaml`）。
*   **指令**：`sudo nano /etc/netplan/你的配置文件名.yaml` （使用超级管理员权限编辑该文件）。

**步骤 3：修改配置文件（关键步骤）**
在文件中填入以下内容。**注意：YAML 文件对缩进极其严格，只能用空格，绝对不能用 Tab 键！**
```yaml
network:
  version: 2
  renderer: NetworkManager  # 桌面版通常是 NetworkManager，服务器版可能是 networkd
  ethernets:
    ens33:                  # 这里必须改成你自己在步骤1查到的网卡名
      dhcp4: no             # 关闭自动获取 IP（关闭 DHCP）
      addresses:
        - 192.168.1.100/24  # 设定你的固定 IP 地址和子网掩码（/24 代表 255.255.255.0）
      routes:
        - to: default
          via: 192.168.1.1  # 设定默认网关（通常是路由器的IP）
      nameservers:
        addresses: [8.8.8.8, 114.114.114.114] # 设定 DNS 解析服务器
```

**步骤 4：应用配置并生效**
*   **指令**：`sudo netplan apply`
*   如果没有报错，说明配置成功。可以输入 `ping www.baidu.com` 测试网络是否通畅。

### 3. 主机名称和 Host 映射

Host 映射的作用是：在本地电脑上强制规定“某个域名/主机名”对应“某个 IP 地址”，相当于给电脑塞了一本私人的“迷你电话本”。系统在上网时会**优先**查这个本子，查不到才会去找网上的 DNS 服务器。

#### (1) Linux 的主机名称与 Host 映射
*   **查看当前主机名**：直接输入 `hostname`。
*   **永久修改主机名**：输入 `sudo hostnamectl set-hostname 新名字` （修改后重新登录终端即可看到变化）。
*   **配置 Host 映射的步骤**：
    1.  输入指令打开文件：`sudo nano /etc/hosts`
    2.  在文件最下方新起一行，按照 `IP地址 主机名` 的格式添加。例如：
        `192.168.96.128 my-ubuntu-server`
    3.  保存退出。现在你在 Linux 里直接 `ping my-ubuntu-server`，系统就会自动去找 `192.168.96.128`。

#### (2) Windows 的主机名称与 Host 映射
*   **查看与修改主机名**：右键点击桌面的“此电脑” -> “属性”，找到“重命名这台电脑”，修改后重启即可。
*   **配置 Host 映射的步骤**：
    1.  **获取权限**：因为该文件受系统保护，必须按 `Win` 键搜索“记事本”，**右键选择“以管理员身份运行”**。
    2.  **打开文件**：在记事本中点击 左上角文件 -> 打开，路径导航到 `C:\Windows\System32\drivers\etc`。在右下角把“文本文档(*.txt)”改为“**所有文件(*.*)**”，找到并打开 `hosts` 文件。
    3.  **添加映射**：在最下方添加记录，例如：`192.168.96.128 my-ubuntu-server`。
    4.  **避坑指南**：保存时务必确认编码格式是 **ANSI** 或无 BOM 的 **UTF-8**；且最后一行**必须敲一个回车留出空行**。
    5.  **刷新缓存**：打开 cmd 命令行，输入 `ipconfig /flushdns` 让修改立即生效。

### 4. 核心原理解析：在浏览器输入 www.baidu.com 是怎么找到服务器地址的？

这在计算机网络中被称为 **DNS 解析（Domain Name System Resolution）** 的全过程。由于计算机底层只认识 IP 地址（如 `110.242.68.3`），不认识人类易读的域名（如 `www.baidu.com`），因此需要一个“查电话本”的过程。

当你敲下回车的那一瞬间，计算机在后台经历了以下由近及远的“连环追问”：

**第一步：本地“翻箱倒柜”（查找本地缓存）**
1.  **浏览器缓存**：浏览器首先回忆自己最近有没有访问过百度，如果有记录且没过期，直接拿 IP 地址去访问。
2.  **操作系统缓存与 Hosts 文件**：如果浏览器忘了，它会去问操作系统。操作系统首先会查看刚才我们在第 3 点配置的本地 **`hosts` 文件**，如果有强制映射，直接使用；如果没有，再查看系统的本地 DNS 缓存记录。

**第二步：求助片区管理员（请求本地 DNS 服务器，LDNS）**
如果在你自己的电脑上都没找到，你的电脑就会向你连接的**路由器**，或者**运营商（电信/联通/移动）分配的本地域名服务器 (LDNS)** 发出求助。这台 LDNS 服务器通常能解答 80% 以上的日常访问请求，如果它缓存里有，就会直接告诉你。

**第三步：终极寻根问底（迭代查询，针对 LDNS 找不到的情况）**
如果运营商的 LDNS 也不知道，它就会代替你，去互联网的最顶层开始“层层递进”地打听：
1.  **问“根”服务器 (Root Server)**：LDNS 首先去问全球的根域名服务器（全世界只有 13 组）。根服务器看了一眼说：“我不知道百度的具体 IP，但我看到它后缀是 `.com`，你去问负责管理 `.com` 的顶级域名服务器吧，这是它的地址。”
2.  **问“顶级域名”服务器 (TLD Server)**：LDNS 接着去找 `.com` 顶级服务器。顶级服务器说：“我也不知道 www.baidu.com 具体在哪，但我知道 `baidu.com` 这个域名的管家（权威服务器）是谁，你去问它吧。”
3.  **问“权威”服务器 (Authoritative Server)**：LDNS 终于找到了百度公司自己维护的权威 DNS 服务器。权威服务器一查自家的花名册，说：“找到了，`www.baidu.com` 对应的 IP 地址是 `110.242.68.3`。”

**第四步：满载而归与建立连接**
1.  LDNS 拿到这个正确的 IP 地址后，马上把它存进自己的缓存里（方便下一个用户查询），然后把结果发给你的电脑。
2.  你的电脑拿到 IP 后，浏览器终于知道了目标，正式向百度的服务器发起 **TCP 三次握手**建立网络连接，随后发送 HTTP 请求，百度服务器返回网页代码，最终你的屏幕上就渲染出了百度的搜索页面。
