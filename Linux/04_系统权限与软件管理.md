## 1. Linux 用户管理
### 1.1 root 用户与 su/sudo
- **root**：系统的超级管理员，拥有最大权限。
- **`su` 切换用户**：`su [-] [用户名]`（`-` 表示同时加载环境变量，省略用户名则切换到 root）。
- **`sudo` 提权**：在命令前加 `sudo`，临时以 root 身份执行。
  - **配置 sudo 认证**：root 下执行 `visudo`，在文件末尾添加 `用户名 ALL=(ALL) NOPASSWD: ALL`。

- 从一个低级别的用户是需要输入密码的，从高级别的用户切换到低级别的用户，不需要输入密码

### 1.2 Linux 用户注销 (Log out)
注销是指结束当前用户的登录会话，返回登录界面，而**不影响**系统的运行状态。

#### ① 常用指令
* **`logout`**：最标准的注销指令。
* **`exit`**：
    * 如果是远程连接（Xshell/SSH），表示断开当前连接。
    * 如果是从普通用户切换到 root（使用 `su` 命令），表示退回原用户。
    * 如果在图形界面终端，表示关闭当前窗口。

#### ② 快捷键（高效推荐）
* **`Ctrl + D`**：等同于执行 `exit`。在终端中连续按下，可以逐层退出直到关闭会话。

#### whoami可以获取当前系统的登录用户名称
#### who am i 查看当前用户的详细信息


### 1.3 用户管理 (需 root)
创建用户的权限只有管理员才有，我们一般在root用户下面去创建用户
- **创建用户**：`useradd [-g 初始组 -d HOME目录] 用户名`
- **创建用户的同时自定义用户目录的名称**：`useradd -d 目录名称 用户名`(一般用户目录和用户名是同名的)
- **给用户设置密码**：`passwd 用户名` 
- **删除用户**：`userdel [-r] 用户名`（`-r` 同时删除 HOME 目录）
- **查看与修改**：
  - `getent passwd` (看用户) / `getent group` (看组)
  - `id [用户名]` (看所属组)
  - `usermod -aG 用户组 用户名` (加入附加组)

### 1.4 用户组管理

#### 1.4.1 用户组 (Group) 基本概念
在 Linux 中，用户组是权限管理的核心单位。通过将具有相同任务需求的用户归入同一个组，可以实现权限的统一分配。
- **关联关系**：一个组可以包含多个用户，一个用户也可以属于多个组。
- **默认机制**：创建用户时若未指定组，系统会自动创建一个与用户名同名的“私有组”。



### 1.4.2 用户组管理指令
这部分操作涉及系统安全，通常需要 `sudo` 权限执行。

| 功能 | 指令语法 | 说明 |
| :--- | :--- | :--- |
| **创建组** | `groupadd 组名` | 在系统中新增一个用户组。 |
| **删除组** | `groupdel 组名` | 彻底删除该组。 |
| **修改组名** | `groupmod -n 新组名 旧组名` | 仅修改组的名称，不影响组 ID 和成员。 |

### 1.4.3 用户与组的关联管理
通过指令控制用户在不同组之间的流转。

- **创建用户时指定主组**：
  ```bash
  sudo useradd -g 组名称 用户名
  ```
- **修改现有用户的主组**：
  ```bash
  sudo usermod -g 组名称 用户名
  ```
  *(注：原笔记中为 `usermd`，正确拼写应为 `usermod`，代表 User Modify)*
- **将用户添加到附加组**：
  ```bash
  sudo usermod -aG 组名称 用户名
  ```
  *(`-aG` 用于追加组，不会覆盖原有的主组，这在赋予用户 sudo 权限时非常常用。)*

### 1.4.4 组信息查询与验证
在执行完上述管理指令后，需要通过以下方式验证结果。

- **查看当前用户所属组**：
  ```bash
  groups
  ```
- **查看特定用户的详细 ID 信息**：
  ```bash
  id 用户名
  ```
  *(该命令会显示 UID、GID 以及所属的所有组名。)*
- **查看系统所有组的清单**：
  ```bash
  cat /etc/group
  ```

### 1.5 找回丢失的 root 密码
当忘记 root 密码时，可以通过进入“单用户模式”或“紧急模式”进行重置。此操作必须在虚拟机本地界面操作，无法通过 Xshell 远程完成。

#### 操作步骤详解：

1.  **进入编辑模式**：
    启动 Linux，在进入开机选择界面时（倒计时结束前），迅速按下键盘上的 **`e`** 键进入内核编辑界面。
    
2.  **修改启动指令**：
    找到以 `linux` 或 `linux16` 开头的那一行，将光标移至行末，空格后输入：
    ```text
    init=/bin/sh
    ```
    
3.  **启动进入环境**：
    按下快捷键 **`Ctrl + X`**，系统将引导进入一个只有 `#` 提示符的单用户 Shell 环境。

4.  **挂载根目录为读写状态**：
    由于此时系统默认是只读的，无法修改密码，需要执行：
    ```bash
    mount -o remount,rw /
    ```
    *(注意：是 `remount` 而不是 `remout`。)*

5.  **重置密码**：
    输入 `passwd` 并回车。系统会提示你输入新密码并确认。
    > **注意**：在 Linux 终端输入密码时，屏幕**不会**显示任何字符，输完直接回车即可。

6.  **标记 SELinux 重新打标签**：
    为了确保修改后的密码在重启后能正常生效（解决权限锁定问题），输入：
    ```bash
    touch /.autorelabel
    ```

7.  **重启系统**：
    输入以下指令完成最后的重置并自动重启：
    ```bash
    exec /sbin/init
    ```

## 2. 权限控制 (`chmod`, `chown` & `chgrp`)

在 Linux 中，每个文件或文件夹都有明确的“所有者”和“访问权限”。通过 `ls -l` 命令可以查看到详细的权限字符串。

### 2.1 权限字符串的四段式拆解
在 Linux 中，执行 `ls -l` 看到的 `drwxr-xr-x` 共有 10 个字符，其具体含义如下：

| 段落 | 字符位置 | 对应对象 | 含义说明 |
| :--- | :--- | :--- | :--- |
| **第一段** | 第 1 位 | **文件类型** | `d`: 目录；`-`: 普通文件；`l`: 软链接（类似快捷方式）。 |
| **第二段** | 第 2-4 位 | **所有者 (u)** | 文件所有者（Owner）拥有的权限。 |
| **第三段** | 第 5-7 位 | **所属组 (g)** | 文件所属用户组（Group）内其他成员的权限。 |
| **第四段** | 第 8-10 位 | **其他人 (o)** | 既不是所有者，也不在所属组内的其他用户（Others）。 |

---

### 权限字符与操作对比表
每一组（3 位）都由 `r`、`w`、`x` 组成，如果某个位置是 `-`，则表示不具备该权限。

| 字符 | 权限名称 | 对文件的意义 | 对目录（文件夹）的意义 | 数字权重 |
| :--- | :--- | :--- | :--- | :--- |
| **r** | **读 (Read)** | 可以读取、查看文件内容 | 可以 `ls` 查看目录下的文件名 | **4** |
| **w** | **写 (Write)** | 可以修改文件内容 | 可以在目录内**创建、删除、重命名**文件 | **2** |
| **x** | **执行 (Execute)** | 该文件可以作为程序运行 | 可以 `cd` 进入该目录 | **1** |
| **-** | **无权限** | 没有任何操作权限 | 没有任何操作权限 | **0** |
ps:删除文件的前提条件是对该文件所在的目录有写权限，才能删除文件

### 2.2 修改权限 (`chmod`)
修改文件或目录的读、写、执行权限。

#### ① 数字法（最常用）
权限对应数值：**r = 4, w = 2, x = 1, - = 0**。
*   **计算方式**：将每组权限相加。例如 `rwx` = 4+2+1 = 7；`r-x` = 4+0+1 = 5。
*   **常见组合**：
    *   `777`：所有人都有所有权限（慎用，不安全）。
    *   `755`：属主全开，其他人只读和执行（常用）。
    *   `644`：属主读写，其他人只读（普通文件标准权限）。
*   **示例**：
    ```bash
    chmod 755 test.cpp    # 修改单个文件
    chmod -R 777 /home/my # 递归修改文件夹及其下所有内容
    ```

#### ② 符号法（更直观）
*   **对象**：`u` (属主), `g` (属组), `o` (其他), `a` (全部)。
*   **操作**：`+` (增加), `-` (取消), `=` (唯一设定)。
*   **示例**：
    ```bash
    chmod u+x file.sh     # 给属主增加执行权限
    chmod g-w file.txt    # 取消属组的写权限
    ```
#### ③直接操作
* **示例**：
    ```bash
    chmod u = rwx, g = rx, o = rx 文件名; //当前用户user为可读可写可操作
    //当前用户组group为可读可操作不可写
    //当前用户组之外的其他用户other为可读可操作不可写

    chmod a+r 文件名 //全部加上可读操作
    ```

## 3. 软件管理与服务控制
### 3.1 包管理器 (yum / apt)
自动化安装配置 Linux 软件，自动解决依赖问题。
- **CentOS**：`yum [-y] [install | remove | search] 软件名称`
- **Ubuntu**：`apt [-y] [install | remove | search] 软件名称`

### 3.2 服务控制 (`systemctl`)
控制软件的启动、停止、开机自启等。
- **语法**：`systemctl start | stop | status | enable | disable 服务名`
- **常见服务**：`NetworkManager` (主网络), `firewalld` (防火墙), `sshd` (SSH连接)。
