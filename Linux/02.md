# Linux 常用命令与权限管理学习笔记

## 1. 查找与检索命令

### 1.1 `which` 命令
通过此命令可以查看所使用的命令的程序文件存放在哪里。
- **语法**：`which 要查找的命令`
- **示例**：`which cd`

### 1.2 `find` 命令
用于在指定目录下搜索文件。
- **按文件名查找**：
  - **语法**：`find 起始路径 -name "被查找的文件名"`
  - **说明**：支持使用通配符 `*`（例如 `*.txt` 查找所有 txt 文件）。
- **按文件大小查找**：
  - **语法**：`find 起始路径 -size +|-n[kMG]`
  - **说明**：
    - `+` 表示大于，`-` 表示小于。
    - `n` 表示大小数字。
    - `kMG` 表示单位（小写 `k` 为 KB，大写 `M` 为 MB，大写 `G` 为 GB）。
  - **示例**：`find / -size +100M`（查找大于 100MB 的文件）。

### 1.3 `grep` 命令
从文件中通过关键字过滤出包含关键字的行。
- **语法**：`grep [-n] "关键字" 文件路径`
- **选项**：
  - `-n`：可选，表示在结果中显示匹配行的行号。
- **参数**：
  - **关键字**：必填，带有空格或其他特殊符号时，建议使用 `""` 将关键字包围起来。
  - **文件路径**：必填，表示要过滤内容的文件路径（也可省略，作为管道符的输入端）。

## 2. 文本处理与输出命令

### 2.1 `wc` 命令
统计文件的行数、单词数量、字节数等。
- **语法**：`wc [-c -m -l -w] 文件路径`
- **选项**：
  - `-c`：统计 bytes (字节) 数量。
  - `-m`：统计 characters (字符) 数量。
  - `-l`：统计 lines (行) 数。
  - `-w`：统计 words (单词) 数量。

### 2.2 管道符 `|`
- **含义**：将管道符 **左边命令的输出结果**，作为 **右边命令的输入**。
- **示例**：`cat test.txt | grep "hello"`。

### 2.3 `echo` 命令
在命令行输出指定内容。
- **语法**：`echo 输出的内容`

### 2.4 反引号 `` ` ``
- **含义**：被 `` ` `` 包围的内容将会被作为**命令**执行，而非普通的字符串。
- **示例**：`echo "当前工作目录是：`pwd`"`。

## 3. vi/vim 编辑器

vi/vim 是 Linux 中最经典的文本编辑器。vim 是 vi 的加强版本，兼容 vi 的所有指令，不仅能编辑文本，还具有代码高亮显示功能，可以以不同颜色的字体来辨别语法的正确性。

### 3.1 三种工作模式
1. **命令模式 (Normal Mode)**：默认模式，通过键盘快捷键进行复制、粘贴、删除等操作。
2. **输入模式 (Insert Mode)**：对文件内容进行具体的文本编辑。
3. **底线命令模式 (Command-line Mode)**：用于文件的保存、退出、搜索等。

- **语法**：`vi/vim 文件路径`
  - *注：如果文件不存在，会用于编辑新文件；如果存在，则编辑已有文件。*

### 3.2 常用快捷键补充
**命令模式下：**
- `i`：在当前光标前进入**输入模式**。
- `a`：在当前光标后进入**输入模式**。
- `o`：在当前光标下方新起一行，并进入**输入模式**。
- `dd`：删除（剪切）当前行。
- `yy`：复制当前行。
- `p`：粘贴。
- `u`：撤销上一步操作。
- `gg`：跳到文件开头。
- `G`：跳到文件末尾。
- `/关键字`：搜索指定关键字（按 `n` 找下一个，`N` 找上一个）。

**其他模式切换：**
- `Esc`：退出输入模式或底线命令模式，回到**命令模式**。
- `:`：在命令模式下按冒号，进入**底线命令模式**。

**底线命令模式下：**
- `:w`：保存。
- `:q`：退出。
- `:wq`：保存并退出。
- `:q!`：强制退出且不保存。

## 4. Linux 用户和权限管理

### 4.1 root 用户与 su / sudo 命令
- **root 用户**：系统的超级管理员，拥有最大的系统操作权限。
- **`su` (switch user) 命令**：用于切换账户。
  - **语法**：`su [-] [用户名]`
  - `-`：可选，表示切换用户后同时加载该用户的环境变量（建议带上）。
  - **用户名**：省略时默认切换到 root 用户。
  - *注：切换后可通过 `exit` 命令退回上一个用户。*
- **`sudo` 命令**：临时以 root 身份执行某条普通命令。
  - **用法**：在需要提权的命令前加上 `sudo`（例如 `sudo yum install wget`）。
  - *前提：需要为普通用户配置 sudo 认证。*

### 4.2 为普通用户配置 sudo 认证（补充）
需要切换到 root 用户进行配置：
1. 切换到 root 用户：`su -`
2. 执行 `visudo` 命令，这会自动使用 vi 编辑器打开 `/etc/sudoers` 文件。
3. 在文件末尾（或找到 `root ALL=(ALL) ALL` 这一行下面）添加：
   `用户名 ALL=(ALL) NOPASSWD: ALL`
   *(注：`NOPASSWD: ALL` 表示使用 sudo 时不需要输入密码；如果想要求输入密码，写成 `用户名 ALL=(ALL) ALL` 即可)*
4. 保存并退出（`:wq`）。该用户即可使用 `sudo` 命令。

### 4.3 用户与用户组管理
*(注：以下命令需 root 用户执行)*
- **用户管理**：
  - **创建用户**：`useradd [-g -d] 用户名`
    - `-g`：指定用户的初始主组（组必须存在）。不指定则自动创建同名组。
    - `-d`：指定用户的 HOME 目录路径。不指定则默认在 `/home/用户名`。
  - **删除用户**：`userdel [-r] 用户名`
    - `-r`：同时删除用户的 HOME 目录。
- **用户组管理**：
  - **创建用户组**：`groupadd 用户组名`
  - **删除用户组**：`groupdel 用户组名`
- **查看与修改**：
  - **查看系统中的用户**：`getent passwd`
  - **查看系统中的组**：`getent group`
  - **查看用户所属组**：`id [用户名]`
  - **修改用户所属附加组**：`usermod -aG 用户组 用户名`（将指定用户加入指定用户组）。


### 4.4 查看与修改权限控制（补充完整）
使用 `ls -l` 可以查看文件/文件夹的权限细节，输出结果的第一列通常是一个 10 位的字符，例如 `drwxr-xr-x`：
- **第 1 位**：文件类型（`-` 代表普通文件，`d` 代表文件夹，`l` 代表软链接）。
- **第 2-4 位**：所属用户 (User) 的权限。
- **第 5-7 位**：所属用户组 (Group) 的权限。
- **第 8-10 位**：其他用户 (Others) 的权限。

**rwx 的含义：**
- `r` (Read, 读)：对文件是查看内容；对文件夹是查看内部文件列表。
- `w` (Write, 写)：对文件是修改内容；对文件夹是可以在其中创建、删除、重命名文件。
- `x` (Execute, 执行)：对文件是可作为程序运行；对文件夹是可以 `cd` 进入。
- `-`：表示没有对应权限。

### 4.5 `chmod` 与 `chown` 命令
**1. 修改权限：`chmod`**
- **语法**：`chmod [-R] 权限 文件或文件夹`
- **选项**：`-R` 对文件夹内的全部内容递归应用同样的操作。
- **权限的数字符号（补充）**：
  权限可以用数字表示，`r=4, w=2, x=1, -=0`。将每组权限相加即可：
  - `7` = 4 + 2 + 1 (rwx)
  - `6` = 4 + 2 + 0 (rw-)
  - `5` = 4 + 0 + 1 (r-x)
  - **示例**：`chmod 755 file.txt` 
    - 赋予属主 7 (rwx)
    - 赋予属组 5 (r-x)
    - 赋予其他用户 5 (r-x)

**2. 修改归属：`chown`**
- **作用**：修改文件/文件夹的所属用户和用户组（仅限 root 用户执行）。
- **语法**：`chown [-R] [用户][:][用户组] 文件或文件夹`
- **选项**：`-R` 递归应用。
- **示例**：
  - `chown user1 file.txt` （仅修改所属用户）
  - `chown :group1 file.txt` （仅修改所属组）
  - `chown user1:group1 file.txt` （同时修改用户和组）

## 5. 终端快捷键与历史命令

- `Ctrl + d`：登出当前用户或退出终端。
- `history`：查看历史输入过的命令。
  - 常配合过滤使用：`history | grep 关键字`。
- `!命令前缀`：自动执行上一次匹配该前缀的历史命令。
- `Ctrl + r`：从下往上搜索历史命令。
- `Ctrl + a`：光标跳到命令开头。
- `Ctrl + e`：光标跳到命令结尾。
- `Ctrl + 左右方向键`：光标向左/右跳过一个单词。

## 6. 软件管理与服务控制

### 6.1 包管理器 (yum / apt)
**CentOS (`yum`)**
RPM 包软件管理器，用于自动化安装配置 Linux 软件，自动解决依赖问题（需 root 权限）。
- **语法**：`yum [-y] [install | remove | search] 软件名称`
- **选项**：`-y` 自动确认，无需手动输入 `y` 确认安装或卸载。

**Ubuntu (`apt`)**
- **语法**：`apt [-y] [install | remove | search] 软件名称`

### 6.2 `systemctl` 命令
控制系统服务的启动、停止、开机自启等。
- **语法**：`systemctl start | stop | status | enable | disable 服务名`
  - `start`：启动
  - `stop`：停止
  - `status`：查看状态
  - `enable`：设置开机自启
  - `disable`：关闭开机自启
- **常见系统内置服务**：
  - `NetworkManager`：主网络服务。
  - `network`：副网络服务。
  - `firewalld`：防火墙服务 (CentOS 常用)。
  - `sshd`：SSH 远程连接服务。
  *(除内置服务外，部分第三方软件如 Nginx、MySQL 安装后也可以通过 systemctl 控制)*

## 7. 软链接 (类似 Windows 快捷方式)

### `ln` 命令
用于创建软链接。
- **语法**：`ln -s 参数1 参数2`
- **选项**：`-s` 表示创建软链接 (Soft link)。不加 `-s` 则是硬链接。
- **参数**：
  - **参数1**：被链接的源文件或文件夹的绝对路径。
  - **参数2**：要创建的快捷方式的路径。
- **示例**：`ln -s /etc/sysconfig/network-scripts/ifcfg-ens33 ~/net_config`
