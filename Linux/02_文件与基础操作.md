## 1. Linux 搜索与查找指令
在 Linux 系统中，根据需求（按名称、按内容、按指令路径）的不同，可以选择不同的搜索工具。
### 1.1 find 指令 (功能最强大)
`find` 指令通过**递归遍历**指定的目录来查找文件。由于是实时遍历磁盘，虽然功能最全，但在超大目录下速度较慢。

#### ① 常用搜索方式示例
*   **按文件名查找**：
    ```bash
    # 在 /home 目录下查找名为 hello.py 的文件
    find /home -name hello.py
    ```
*   **按所属用户查找**：
    ```bash
    # 在 /root 目录下查找属于用户 cpp 的所有文件
    find /root -user cpp
    ```
*   **按文件大小查找**：
    ```bash
    # 在当前目录下查找大于 200M 的文件 (+n 大于, -n 小于, n 等于)
    find . -size +200M
    ```
---

### 1.2 locate 指令 (速度最快)
`locate` 不扫描文件系统，而是查询系统内建的**索引数据库** (`/var/lib/mlocate`)。

*   **特点**：查询速度极快，但无法实时找到刚刚创建的新文件。
*   **使用前提**：第一次使用或查找新文件前，必须手动更新数据库。
*   **示例**：
    ```bash
    sudo updatedb    # 必须先更新数据库
    locate hello.txt # 快速定位文件路径
    ```
---

### 1.3 which 指令 (查找指令路径)
`which` 专门用于查看某个**二进制指令**存放在系统的哪个位置。

*   **示例**：
    ```bash
    which ls     # 输出: /usr/bin/ls
    which python # 查看当前环境使用的 python 路径
    ```
---

### 1.4 grep 指令 与 管道符 `|`
`grep` 是一种强大的文本搜索工具，它能使用正则表达式搜索文本。

*   **管道符 `|`**：表示将前一个命令的处理结果输出传递给后面的命令处理。
*   **常用写法**：
    ```bash
    # 在 hello.txt 文件中查找包含 "hello" 的行，并显示行号 (-n)
    cat hello.txt | grep -n "hello"
    
    # 结合查询指令，过滤出特定的进程
    ps -ef | grep mysql
    ```
---

### 💡 查找指令对比总结

| 指令 | 搜索对象 | 原理 | 优缺点 |
| :--- | :--- | :--- | :--- |
| **find** | 任意文件 | 磁盘递归遍历 | 实时准确，功能极多，但较耗资源。 |
| **locate** | 任意文件 | 数据库索引查询 | 速度极快，但非实时，需更新数据库。 |
| **which** | 系统指令 | 搜索 PATH 变量路径 | 只找指令，不找普通文件。 |
| **grep** | 文件内容 | 文本过滤 | 配合管道符是 Linux 运维的“神技”。 |

## 2. 文本处理与输出
### 2.1 `wc` 命令
统计文件的行数、单词数量、字节数等。
- **语法**：`wc [-c -m -l -w] 文件路径`
  - `-c` (字节数), `-m` (字符数), `-l` (行数), `-w` (单词数)

### 2.2 管道符 `|` 与输出
- **管道符 `|`**：将左边命令的输出结果，作为右边命令的输入。示例：`cat test.txt | grep "hello"`
- **`echo` 命令**：在命令行输出指定内容。语法：`echo 输出的内容`
- **反引号 `` ` ``**：被包围的内容将会被作为命令执行。示例：`echo "当前目录是：`pwd`"`

## 3. 终端快捷键与历史命令
- `Ctrl + d`：登出当前用户或退出终端。
- `Ctrl + r`：从下往上搜索历史命令。
- `Ctrl + a` / `Ctrl + e`：光标跳到命令开头 / 结尾。
- `Ctrl + 左右方向键`：光标向左/右跳过一个单词。
- `history`：查看历史输入过的命令（常配合 `grep` 过滤）。
- `!命令前缀`：自动执行上一次匹配该前缀的历史命令。

## 4. 文件链接与传输
### 4.1 软链接 (`ln`)
类似 Windows 的快捷方式。
- **语法**：`ln -s 被链接的绝对路径 要创建的快捷方式路径`（不加 `-s` 是硬链接）

### 4.2 上传和下载 (Ubuntu 常用)
1. **使用 `lrzsz` 工具（基于 Xshell 等软件）**：
   - 安装：`sudo apt install lrzsz`
   - 上传：输入 `rz` 回车，弹窗选择。
   - 下载：输入 `sz 文件名`，弹窗保存。
2. **使用 `scp` 命令（基于 SSH，本地终端执行）**：
   - 上传：`scp 本地文件路径 用户名@服务器IP:/目标路径`
   - 下载：`scp 用户名@服务器IP:/服务器上的文件路径 本地存放路径`

## 5. 文件压缩与解压
### 5.1 `tar` 命令
- `.tar`：tarball 归档文件，仅打包不压缩。
- `.gz`：gzip 压缩文件，大幅减小体积。
- **语法**：`tar [-c -v -x -f -z -C] 参数...`
  - `-c` (创建压缩), `-x` (解压缩), `-v` (显示进度), `-z` (gzip模式), `-C` (指定解压目的地)。
  - **注意**：`-f`（指定文件名）必须放在选项最后一位。
- **常用组合**：
  - 压缩：`tar -zcvf 压缩包名.tar.gz 被压缩目录`
  - 解压：`tar -zxvf 被解压的文件名.tar.gz -C /解压目标路径`

### 5.2 `zip` 与 `unzip` 命令
- **压缩**：`zip [-r] 压缩包名.zip 参数...`（包含文件夹需加 `-r` 递归）。
- **解压**：`unzip [-d 目标路径] 压缩包名.zip`
