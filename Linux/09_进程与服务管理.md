# Linux 进程与服务管理核心笔记

## 一、 查看进程信息

### 1. 动态监控进程：`top` 指令
- **作用**：类似于 Windows 的任务管理器，能够实时、动态地监控系统的进程状态、CPU 和内存使用情况。
- **常用交互快捷键**：
  - `P`：按 CPU 使用率排序
  - `M`：按内存使用率排序
  - `q`：退出 `top` 监控面板

### 2. 查看静态进程快照：`ps` 指令
- **作用**：显示当前系统中进程的静态快照（类似于截图），通常用于查看后台进程或者过滤特定进程。
- **常用参数**：
  - `-a`：显示当前终端的所有进程信息。
  - `-u`：以用户的格式显示进程信息。
  - `-x`：显示没有控制终端的后台进程及其运行参数。
  - `-e`：显示所有进程（等同于 `-A`）。
  - `-f`：全格式显示，包含 UID、PID、PPID（父进程ID）等更多信息。
- **常用组合命令**：
  - `ps -aux | more`：分页全屏展示所有进程信息。
  - `ps -ef | grep 进程名`：使用全格式查看系统中包含特定名称的进程（常用于查询某个服务是否在运行）。

#### 💡 补充说明：`ps -aux` 的表头信息含义
当执行 `ps -aux` 时，系统会输出以下各列代表的含义：
* **USER**：运行该进程的用户名称。
* **PID**：进程的唯一标识符（Process ID）。
* **%CPU**：当前进程消耗的 CPU 占用百分比。
* **%MEM**：当前进程消耗的物理内存占用百分比。
* **VSZ**：进程占用的虚拟内存大小（单位：KB）。
* **RSS**：进程占用的实际物理内存大小（单位：KB）。
* **TTY**：该进程在哪个终端上运行。`?` 表示后台运行（与终端无关），`tty1-tty6` 是本机字符终端，`pts/0-255` 代表虚拟终端（如通过 SSH 远程连接）。
* **STAT**：当前进程的状态（常见状态如：`S` 睡眠，`R` 运行，`Z` 僵尸进程，`T` 停止）。
* **START**：进程被触发启动的时间。
* **TIME**：该进程实际消耗 CPU 的总时间。
* **COMMAND**：启动该进程的对应命令或程序的名称及参数。

### 3. 查看进程树：`pstree` 指令
- **作用**：以树状图的形式更加直观地查看进程及其父子关系。
- **常用参数**：
  - `-p`：显示每个进程的 PID。
  - `-u`：显示每个进程的所属用户。
  - **示例**：`pstree -pu`（既显示 PID 又显示用户）。

## 二、 终止/管理进程

- **`kill PID`**：通过进程 ID（PID）正常终止一个进程。
- **`kill -9 PID`**：强制杀死某个进程（当普通 `kill` 无法终止时使用，`-9` 代表 SIGKILL 信号）。
- **`killall 进程名称`**：通过进程的名称杀死该进程，并且**会将其下属的子进程一并杀死**（适用于停止大型服务或负载的多个子进程）。
- **`pkill 进程名称`**：（*补充*）根据进程名称来杀死进程，类似于 `killall`。

## 三、 Linux 的服务管理

### 1. 现代 Linux 的系统服务管理 (`systemctl`)
目前绝大多数主流的 Linux 发行版（如 CentOS 7/8/9、Ubuntu 16.04+ 等）已经使用 `systemd` 替代了传统的 `init` 进程来管理服务。
- **常用命令格式**：`systemctl [选项] 服务名`
- **示例：管理 SSH 服务 (sshd)**：
  - **启动服务**：`systemctl start sshd` （或绝对路径 `/bin/systemctl start sshd.service`）
  - **停止服务**：`systemctl stop sshd`
  - **重启服务**：`systemctl restart sshd`
  - **查看服务状态**：`systemctl status sshd`
  - **设置开机自启**：`systemctl enable sshd`
  - **关闭开机自启**：`systemctl disable sshd`

### 2. 传统经典的服务管理 (`service`)
在早期的 Linux 版本（如 CentOS 6 及其以前）中，使用 SysVinit 机制。
- 这些被 `service` 指令管理的服务脚本通常存放在 `/etc/init.d/` 目录下（用 `ls -l /etc/init.d/` 查看时，绿色的可执行文件代表服务脚本）。
- **常用命令格式**：`service 服务名 [start|stop|restart|status]`
- **示例**：查看 network 服务的状态：`service network status`

### 3. 图形化/交互式系统设置工具：`setup` 指令
- **作用**：`setup` 是 Red Hat 系列（CentOS、RHEL）自带的一个交互式文本图形管理工具，可用来配置防火墙、网络配置、系统服务管理等。
- **注意**：随着系统更新，`setup` 的功能逐渐被更专业的命令替代（如管理网络现在推荐使用 `nmtui` 工具）。

# Linux 服务管理与系统监控进阶笔记

## 四、 systemctl 服务管理指令

`systemctl` 是现代 Linux（CentOS 7+、Ubuntu 16.04+）管理系统服务的核心工具。系统的服务脚本（Unit 文件）通常存放在 `/usr/lib/systemd/system/` 目录下。

### 1. 基本服务控制语法
`systemctl [start | stop | restart | reload | status] 服务名`
- `start`：启动服务
- `stop`：停止服务
- `restart`：重启服务（会中断服务）
- `reload`：重载配置（不中断服务重新加载配置文件，适用于 Nginx 等）
- `status`：查看服务当前运行状态

### 2. 开机自启状态管理
- **查看所有服务的自启动状态**：`systemctl list-unit-files`
- **设置服务开机自启**：`systemctl enable 服务名`
- **关闭服务开机自启**：`systemctl disable 服务名`

### 3. 防火墙 (firewalld) 管理示例
- 查看防火墙相关服务：`systemctl list-unit-files | grep firewall`
- 查看防火墙状态：`systemctl status firewalld.service` （注意是 `firewalld`）
- 停止防火墙：`systemctl stop firewalld.service`
- 重启防火墙：`systemctl restart firewalld.service`

---

## 五、 Linux 防火墙指令 (firewall-cmd)

`firewall-cmd` 是基于 `firewalld` 服务的命令行配置工具，用于管理网络端口和协议的访问控制。

### 1. 基本语法
*注：添加 `--permanent` 参数表示永久生效，否则重启后失效。所有带 `--permanent` 的修改都必须执行 `--reload` 才会立刻生效。*

- **打开端口/协议**：`firewall-cmd --permanent --add-port=端口号/协议`
- **关闭端口/协议**：`firewall-cmd --permanent --remove-port=端口号/协议`
- **查询端口/协议是否开启**：`firewall-cmd --query-port=端口号/协议`
- **查询防火墙所有开放的端口**：`firewall-cmd --list-ports`
- **重载防火墙配置**：`firewall-cmd --reload`

### 2. 实战案例举例
- **案例 1：开放 Web 网站端口（HTTP 80 和 HTTPS 443）**
  ```bash
  firewall-cmd --permanent --add-port=80/tcp
  firewall-cmd --permanent --add-port=443/tcp
  firewall-cmd --reload   # 让配置立即生效
  ```

- **案例 2：临时开放 MySQL 3306 端口（不加 --permanent，重启失效）**
  ```bash
  firewall-cmd --add-port=3306/tcp
  ```

- **案例 3：关闭不再使用的 8080 端口**
  ```bash
  firewall-cmd --permanent --remove-port=8080/tcp
  firewall-cmd --reload
  ```
- **案例 4：检查 Redis 6379 端口是否被防火墙放行**
  ```bash
  firewall-cmd --query-port=6379/tcp
  # 返回 yes 表示放行，返回 no 表示拦截
  ```

## 六、 动态监控 (top 指令)

`top` 指令类似于 Windows 的任务管理器，用于实时监控系统的 CPU、内存以及各个进程的资源占用情况。

### 1. 基本语法与选项
`top [选项]`
- `-d 秒数`：指定 top 界面每隔几秒刷新一次（默认通常是 3 秒）。
- `-i`：不显示任何闲置或者僵死的进程（只看正在活跃的）。
- `-p PID`：通过指定进程 ID（PID）来仅仅监控某个特定进程的状态。

### 2. 交互操作（进入 top 界面后可按的快捷键）
- `P`：按 **CPU 使用率** 从大到小排序（默认状态）。
- `M`：按 **内存使用率** 从大到小排序。
- `N`：按 **PID（进程号）** 倒序排列。
- `u`：监控特定用户。按下 `u` 后，输入用户名回车，只显示该用户的进程。
- `k`：终止（Kill）进程。按下 `k`，输入要结束的进程 PID 回车，系统会提示输入信号量，输入 `9`（代表强制杀死）并回车。
- `q`：退出 `top` 面板。

### 3. `top` 界面信息深度解析
在 `top` 界面的**上半部分（系统总览）**：
- **第一行 (top)**：当前时间、系统运行总时长、当前登录用户数、**系统负载 (load average)**（三个数值分别代表 1分钟、5分钟、15分钟内的系统平均负载，数值越低越好）。
- **第二行 (Tasks)**：进程总数、正在运行数、睡眠数、停止数、僵尸进程数。
- **第三行 (%Cpu)**：CPU 状态。重点关注 `us`（用户空间占用率）、`sy`（内核空间占用率）和 `id`（CPU 空闲率，越高说明 CPU 越闲）。
- **第四/五行 (Mem/Swap)**：物理内存（Mem）和交换分区虚拟内存（Swap）的总量、空闲量、已用量。

在 `top` 界面的**下半部分（进程列表表头）**：
- **PID**：进程的唯一 ID 号。
- **USER**：运行该进程的用户名。
- **PR / NI**：进程优先级。
- **VIRT**：进程使用的虚拟内存大小。
- **RES**：进程使用的物理内存大小。
- **SHR**：共享内存大小。
- **S**：进程状态（`R`=运行，`S`=睡眠，`Z`=僵尸）。
- **%CPU**：上次更新到现在的 CPU 时间占用百分比。
- **%MEM**：进程使用的物理内存百分比。
- **TIME+**：进程自启动以来占用的总 CPU 时间。
- **COMMAND**：启动进程的命令名称。

## 七、 监控网络状态 (netstat 指令)

`netstat` 用于查看 Linux 系统的网络连接状态、路由表、接口统计等信息。（*注：现代 Linux 中也常推荐使用速度更快的 `ss` 命令替代 netstat*）

### 1. 基本语法
`netstat [选项]`
最常用的黄金组合是：`netstat -anp | grep 端口号或进程名`
- `-a`：显示所有的网络连接和监听端口（All）。
- `-n`：以数字（IP 和 端口号）形式显示，不尝试去解析域名，速度更快（Numeric）。
- `-p`：显示是哪个 PID 和进程名在调用该网络连接（Programs）。

### 2. 输出内容表头解析
执行 `netstat -anp` 后，输出结果的各列含义如下：
- **Proto (协议)**：该连接使用的协议，通常是 `tcp`、`udp` 或 `tcp6`。
- **Recv-Q (接收队列)**：网络接收队列。表示收到的数据已经在本地接收缓冲，但是还有多少字节没有被进程取走。正常情况下应为 0，如果持续非 0，说明程序处理缓慢或卡死。
- **Send-Q (发送队列)**：网络发送队列。对方没有收到的数据包字节数。正常情况下也应为 0。
- **Local Address (本地地址)**：本机当前的 IP 地址和端口号（例如 `0.0.0.0:22` 表示监听本机所有网卡的 22 端口）。
- **Foreign Address (外部地址)**：与本机建立网络连接的远端（对方）IP 地址和端口号。
- **State (状态)**：当前网络连接的状态。最常见的有：
  - `LISTEN`：表示本机正在监听该端口，等待别人来连接（通常是服务器状态）。
  - `ESTABLISHED`：表示双方已经成功建立连接，正在进行数据传输。
  - `TIME_WAIT`：表示连接刚刚断开，系统正在保留状态以确保数据包收发处理完毕。
- **PID/Program name (进程信息)**：是哪个进程（PID 号）和哪个程序名称占用了这个网络连接（需要有 root 权限或加 `-p` 参数才能看到）。
