# Linux Shell 编程核心笔记

## 一、 什么是 Shell
Shell 是一个命令行解释器，它为用户提供了一个向 Linux 内核发送请求以便运行程序的界面系统级程序。用户可以用 Shell 来启动、挂起、停止甚至是编写一些程序。

### Shell 脚本的编写与执行方式
- **脚本格式要求**：
  - 脚本必须以 `#!/bin/bash` 开头（指定解析器）。
- **常用执行方式**：
  - **方式一（推荐）**：赋予脚本可执行权限后运行。
    - `chmod +x 脚本文件.sh`
    - `./脚本文件.sh` （使用相对路径或绝对路径执行）
  - **方式二**：直接使用 `sh` 或 `bash` 指令执行（不需要赋予脚本可执行权限）。
    - `sh 脚本文件.sh` 或 `bash 脚本文件.sh`

## 二、 Shell 的变量
Linux Shell 中的变量分为**系统变量**和**用户自定义变量**。

- **系统变量**：`$HOME`、`$PWD`、`$SHELL`、`$USER` 等。
  - 查询指定的系统变量：`echo $HOME`
  - 查询所有的系统变量：`set | more`
- **用户自定义变量**：
  - **定义变量**：`变量名=值` （**注意：等号两侧绝对不能有空格！**）
  - **撤销变量**：`unset 变量名`
  - **声明静态变量**：`readonly 变量名` （注意：静态变量不能被 unset 撤销，重启或退出终端后失效）
- **将命令返回值赋值给变量**：
  - `A=$(ls -la)` 或 `A=`ls -la``（使用反引号或 `$()` 提取命令结果）

---

## 三、 设置环境变量
环境变量相当于系统级别的全局变量，可以在所有的子 Shell 中生效。

- **基本语法**：
  - `export 变量名=变量值`：将局部 shell 变量提升为环境变量/全局变量。
  - `source 配置文件`：让修改后的配置文件（如 `/etc/profile`）立即生效。
  - `echo $变量名`：查询环境变量的值。
- **Shell 中的注释**：
  - 单行注释：以 `#` 开头
  - 多行注释：
    ```bash
    :<<!
    这里是多行注释内容
    可以写很多行
    !
    ```

---

## 四、 位置参数变量
当我们执行一个 Shell 脚本时，如果希望获取到命令行传入的参数信息，就可以使用位置参数变量。

- `$n`：`n` 为数字，`$0` 代表脚本命令本身，`$1`-`$9` 代表第一个到第九个参数，十以上的参数需要用大括号包含，如 `${10}`。
- `$*`：代表命令行中的所有参数，**把所有的参数看成一个整体**。
- `$@`：代表命令行中的所有参数，**不过该命令会把每个参数区分对待**（在 for 循环中体现明显）。
- `$#`：代表命令行中传入的参数总个数。

---

## 五、 预定义变量
Shell 设计者事先已经定义好的变量，可以直接在 Shell 脚本中使用。

- `$$`：当前进程的进程号 (PID)。
- `$!`：后台运行的最后一个进程的进程号。
- `$?`：最后一次执行的命令的返回状态。如果值为 `0`，证明上一个命令正确执行；如果非 `0`，则证明执行失败或报错。

## 六、 Shell 中的运算符
用于在脚本中实现各种基本的数学运算操作。

- **基本语法**：
  - 第一种写法（推荐）：`$((表达式))`
  - 第二种写法：`$[表达式]`
- **示例：接收命令行的值进行加法运算**
  ```bash
  #!/bin/bash
  # 运行脚本时传入两个数字： ./test.sh 10 20
  NUM=$[$1+$2]
  echo "两个参数的和为: $NUM"
  ```

## 七、 Shell 中的条件判断

### 1. 常用判断条件
- **字符串比较**：`=`
- **整数比较**：
  - `-eq` (等于), `-ne` (不等于)
  - `-lt` (小于), `-le` (小于等于)
  - `-gt` (大于), `-ge` (大于等于)
- **按文件权限判断**：
  - `-r` 有读的权限
  - `-w` 有写的权限
  - `-x` 有可执行的权限
- **按文件类型判断**：
  - `-f` 文件存在，并且是一个常规文件
  - `-e` 文件或目录存在
  - `-d` 文件存在，并且是一个目录

### 2. if-else 判断语法
```bash
if [ 条件判断式 ] # 注意：中括号内部的两侧必须有空格！
then
    执行代码1
elif [ 条件判断式 ]
then
    执行代码2
else
    执行代码3
fi
```

### 3. case 语句语法与示例
```bash
#!/bin/bash
case $1 in
    "1")
        echo "星期一"
        ;;
    "2")
        echo "星期二"
        ;;
    *)
        echo "其他日子"
        ;;
esac
```

## 八、 Shell 中的循环语句 (for)

### 语法 1：遍历列表（类似于 for-each）
```bash
#!/bin/bash
# 打印命令行传入的所有参数
for i in "$@"
do
    echo "接收到的参数为: $i"
done
```

### 语法 2：C 语言风格的循环
```bash
#!/bin/bash
# 计算从 1 加到 100 的值
SUM=0
for (( i=1; i<=100; i++ ))
do
    SUM=$[$SUM+$i]
done
echo "1到100的总和为: $SUM"
```

## 九、 while 循环

- **基本语法与示例**：
```bash
#!/bin/bash
# 使用 while 循环计算 1 到 100 的和
SUM=0
i=1
while [ $i -le 100 ]
do
    SUM=$[$SUM+$i]
    i=$[$i+1] # 变量自增
done
echo "While循环结果: $SUM"
```

## 十、 read 读取控制台输入

- **基本语法**：`read [选项] 参数(变量名)`
- **选项**：
  - `-p`：指定读取值时的提示符。
  - `-t`：指定读取值时等待的时间（秒），如果没有在指定时间内输入，指令自动跳过。
- **示例代码**：
```bash
#!/bin/bash
# 提示用户输入，并设置10秒超时
read -t 10 -p "请在10秒内输入您的姓名: " NAME
echo "欢迎您, $NAME"
```

## 十一、 Shell 函数

### 1. 系统自带函数
- **`basename`**：获取路径中的文件名（去掉完整的绝对路径，甚至可以顺便去掉后缀）。
  - 语法：`basename [文件的绝对路径] [可选：要去掉的后缀]`
  - 示例：`basename /home/user/test.txt .txt` -> 返回 `test`
- **`dirname`**：返回完整路径中最前面的目录部分（去掉最后的文件名）。
  - 语法：`dirname [文件的绝对路径]`
  - 示例：`dirname /home/user/test.txt` -> 返回 `/home/user`

### 2. 自定义函数
```bash
#!/bin/bash
# 定义一个计算两数之和的函数
function getSum() {
    # 这里的 $1 和 $2 指的是传递给函数的参数，不是命令行的参数
    SUM=$[$1+$2]
    echo "两个数的和是: $SUM"
}

read -p "请输入第一个数字: " n1
read -p "请输入第二个数字: " n2

# 调用函数并传入参数
getSum $n1 $n2
```
