### 1. 虚拟机的克隆
克隆是快速创建多个相同环境的方法。
* **方式一：手动克隆**
    * 直接在 Windows 文件管理器中复制粘贴 Ubuntu/CentOS 的整个虚拟机文件夹。
* **方式二：VMware 自动克隆**（右键虚拟机 -> 管理 -> 克隆）
    * **链接克隆**：创建一个指向原始虚拟机的引用。占用空间极小，但必须保留原始虚拟机，原始虚拟机损坏则克隆机失效。
    * **完整克隆**：完全复制一份原始虚拟机的状态。占用空间大，但与原始虚拟机完全独立，互不影响。

### 2. 虚拟机的快照（系统“后悔药”）
快照用于记录虚拟机在某一时刻的状态，以便在系统崩溃或误操作后快速恢复。
**操作步骤：**
1.  **创建快照**：点击 VMware 菜单栏的“时钟”图标（或右键虚拟机 -> 快照 -> 拍摄快照），起一个易懂的名字（如“安装环境前”）。
2.  **快照管理**：进入“快照管理器”，可以查看到所有历史快照节点。
3.  **恢复快照**：选中目标节点，点击“转到”，虚拟机将立即回滚到当时的状态。

---

### 3. 虚拟机的迁移与删除
* **迁移**：在关闭虚拟机的情况下，直接将虚拟机所在的文件夹整体复制粘贴到移动硬盘或其他电脑上，然后在新的 VMware 中选择“打开虚拟机”即可。
* **删除**：
    * **从库中移除**：只是在列表中不显示，文件还留在磁盘上。
    * **从磁盘中删除**：彻底清空文件，释放磁盘空间（建议操作）。

---

### 4. VMware Tools 与文件共享
VMware Tools 是虚拟机增强工具，主要解决 Linux 和 Windows 之间的文件传输和操作便利性问题。

#### A. 实现 Windows 共享文件夹的步骤：
1.  **VMware 设置**：右键虚拟机 -> 设置 -> 选项 -> **共享文件夹**。
2.  **启用功能**：选择“总是启用”，点击“添加”，选择 Windows 上的一个文件夹作为共享目录，并起个名字（例如 `myshare`）。
3.  **Linux 安装支持**：确保 Linux 内安装了 `open-vm-tools-desktop`。

#### B. 在 Linux 中访问 Windows 文件的路径：
在 Linux 内部，Windows 共享过来的文件不会出现在桌面，而是挂载在特定的系统目录下。
* **访问路径**：**`/mnt/hgfs/`**
* **操作命令**：
    ```bash
    cd /mnt/hgfs/          # 进入挂载目录
    ls                     # 查看你刚才设置的共享文件夹名称（如 myshare）
    cd myshare             # 进入该目录即可看到 Windows 里的文件
    ```
> **注意**：如果在 `/mnt/hgfs` 下看不到文件夹，通常需要运行命令 `sudo mount -t fuse.vmhgfs-fuse .host:/ /mnt/hgfs -o allow_other` 进行手动挂载。

### 5. 远程连接 Linux 系统
在实际开发中，我们很少直接在虚拟机窗口操作，而是通过远程工具连接。这要求必须知道 Linux 的 **IP 地址**（可通过 `ifconfig` 命令查看）。

#### ① Xshell：终端命令行连接
用于远程输入命令，操作 Linux 服务器。
* **新建会话**：打开 Xshell，点击“新建”。
* **配置参数**：
    * **主机**：输入 Linux 的 IP 地址。
    * **协议**：SSH（默认）。
    * **端口**：22（默认）。
* **身份验证**：输入 Linux 的用户名（如 `cpp`）和密码。
* **连接成功**：看到提示符后，即可实现远程控制。

#### ② FileZilla：远程文件传输
用于在 Windows 和 Linux 之间互传文件（如安装包、源代码）。
* **快速连接**：
    * **主机**：输入 `sftp://IP地址`（注意 sftp 协议更安全且支持 22 端口）。
    * **用户名/密码**：输入 Linux 的登录凭证。
    * **端口**：22。
* **操作方法**：
    * 界面左侧为“本地站点”（Windows），右侧为“远程站点”（Linux）。
    * 直接**拖拽**文件即可完成上传或下载。

### 6. Linux 远程关机和重启操作
**核心注意**：远程操作必须具备管理员权限。如果使用普通用户登录，命令前必须加 `sudo`。

#### ① shutdown 命令（最安全的方式）
* `sudo shutdown -h now`：立即关机。
* `sudo shutdown -h 1`：1 分钟后关机（系统会向所有用户发送通知）。
* `sudo shutdown -r now`：立即重启。

#### ② 其他快捷命令
* **关机**：`sudo halt` 或 `sudo poweroff`。
* **重启**：`sudo reboot`。

#### ③ sync 数据同步（重要！）
* **功能**：将内存中尚未写入磁盘的数据强制同步到硬盘。
* **建议**：在执行关机或重启命令前，养成连刷两次 `sync` 命令的习惯，以防意外丢数据。

#### ④ 远程操作总结
* **权限问题**：若提示 `Permission denied`，请在命令前加上 `sudo`。
* **连接状态**：关机或重启会导致远程连接（Xshell/VS Code）立即中断。
* **无法开机**：通过命令只能远程“关”，不能远程“开”。若需开机，仍需在 VMware 中手动开启。

### 7. 运行级别的切换与查看

#### ① 临时切换级别 (init)
你可以使用 `init` 命令在不同级别间快速跳转。
- **示例**：
  - `init 3`：关闭图形界面，进入纯命令行模式。
  - `init 5`：从命令行切回图形桌面。
  - `init 0`：立即关机。

#### ② 查看与修改默认级别 (systemd)
在现代 Linux（如 Ubuntu 18.04+）中，推荐使用 `systemctl` 指令来操作，因为它更符合当前的系统架构。

- **查看当前默认级别**：
  ```bash
  systemctl get-default
  ```
- **修改默认启动级别**：
  - **设置为命令行模式启动**（对应级别 3）：
    ```bash
    sudo systemctl set-default multi-user.target
    ```
  - **设置为图形界面模式启动**（对应级别 5）：
    ```bash
    sudo systemctl set-default graphical.target
    ```

## 8. Linux 帮助指令

在 Linux 中，你不需要死记硬背所有命令及其参数，学会使用内置的帮助手册是最高效的学习方式。

### 8.1 man 指令（Manual Pages）
`man` 是 Linux 最核心的外部命令手册。

#### ① man 手册中文化步骤
由于默认手册是英文的，可以通过以下步骤安装中文支持包：
1. **更新软件源**：
   ```bash
   sudo apt update
   ```
2. **安装中文手册包**：
   ```bash
   sudo apt install manpages-zh
   ```
3. **设置中文别名（推荐方案）**：
   为了不覆盖原有的英文手册（英文版更新更及时），建议设置一个专门查看中文手册的命令：
   - 输入 `vim ~/.bashrc` 进入配置文件。
   - 在文件末尾添加：`alias cman='man -L zh_CN'`。
   - 保存退出后执行 `source ~/.bashrc` 使其生效。
   - **使用**：以后输入 `cman ls` 即可查看中文帮助。



---

### 8.2 help 指令
`help` 主要用于查看 Linux **内置命令** 的帮助信息。

#### ① 指令适用范围梳理
Linux 命令分为“内置命令”和“外部命令”，它们查看帮助的方式不同：

| 帮助工具 | 适用指令类型 | 常见查询指令举例 | 说明 |
| :--- | :--- | :--- | :--- |
| **man** | **外部命令** | `ls`, `cp`, `mv`, `tar`, `vim`, `gcc` | 绝大多数安装在 `/bin` 或 `/usr/bin` 下的独立程序。 |
| **help** | **内置命令** | `cd`, `pwd`, `exit`, `echo`, `alias`, `history` | 属于 Shell（如 bash）自身的一部分，执行速度极快。 |

> **提示**：如果不确定一个命令是内置还是外部，可以使用 `type 命令名`（如 `type cd`）来查看。

---

### 8.3 帮助指令对比总结

#### A. 使用方法
* **man 模式**：`man ls`
  - 进入后使用 `j/k` 翻页，`/关键词` 搜索。
  - 按 **`q`** 退出。
* **help 模式**：
  - 内置命令：`help cd`
  - 外部命令：大部分外部命令也支持 `--help` 参数（如 `ls --help`），这种方式输出较简练，适合快速查看参数。

#### B. 核心区别
* **man**：提供极其详尽的文档，包括命令描述、所有参数说明、作者信息及相关文件。
* **help**：提供精简的参数列表和基本用法，适合忘记某个参数字母（如 `-r` 还是 `-R`）时快速查阅。

## 9. Linux 时间与日历管理

### 9.1 date 指令 (系统时间)
`date` 指令主要用于显示或设置系统的日期和时间。

#### ① 显示当前时间
* **基本用法**：直接输入 `date`。
* **格式化输出**：使用 `+` 符号引导自定义格式。
    * `%Y`：四位年份
    * `%m`：月份 (01-12)
    * `%d`：按月计的天 (01-31)
    * `%H`：小时 (00-23)
    * `%M`：分钟 (00-59)
    * `%S`：秒数 (00-60)
* **常用范例**：
  ```bash
  # 以“年-月-日 时:分:秒”格式输出
  date "+%Y-%m-%d %H:%M:%S" 
  ```

#### ② 修改系统时间
在需要手动校准时间时使用（通常需要 root 权限）：
```bash
sudo date -s "2026-04-30 12:00:00"
```

---

### 9.2 cal 指令 (日历)
`cal` (Calendar) 用于在终端查看日历信息。

#### ① 基本用法
* **查看本月**：直接输入 `cal`，会高亮显示今天的日期。
* **查看指定年份**：`cal 2026`（输出该整年的日历）。



---

### 9.3 时间管理核心知识点对比

| 功能 | 指令语法 | 应用场景 |
| :--- | :--- | :--- |
| **查看时间** | `date` | 日常查看，或在脚本中记录操作日志。 |
| **设置时间** | `date -s` | 系统时间偏差过大时手动校准。 |
| **查看日历** | `cal` | 快速确认某年某月的天数或星期。 |
| **硬件时钟** | `hwclock` | (补充) 查看主板 BIOS 里的硬件时间。 |

---

### 💡 核心知识点补充：
1. **引号的使用**：在 `date "+%Y-%m-%d"` 中，由于格式字符串包含空格或特殊符号，必须使用**双引号**包裹。
2. **同步网络时间**：在实际生产环境中，很少手动 `date -s`。通常使用 `ntpdate` 或 `timedatectl` 命令自动与互联网上的时间服务器同步，以保证高精度。
3. **时区问题**：如果你发现 `date` 显示的时间和你的手表差了正好 8 小时，说明时区被设成了 UTC，可以使用 `timedatectl set-timezone "Asia/Shanghai"` 切回北京时间。

## 10. 定时任务调度 (crontab)

## 10.1 定时任务调度概述
定时任务调度是指系统在指定的时间，自动执行指定的命令或程序脚本。在 Linux 中，这一功能主要由 `cron` 守护进程管理。

---

## 10.2 crontab 核心命令
`crontab` 命令用于维护当前用户的任务列表。

*   **`crontab -e`**：编辑当前用户的定时任务列表。
*   **`crontab -l`**：查询并列出当前所有的定时任务。
*   **`crontab -r`**：删除当前用户的所有定时任务。
*   **`systemctl restart cron`**：重启任务调度服务（旧版系统使用 `service crond restart`）。

---

## 10.3 任务调度的时间规则
`crontab` 的每一行都由 **时间表达式 + 执行指令** 组成。

### 10.3.1 占位符说明 (五个星号)
格式：`* * * * * command`

| 占位符位置 | 含义 | 取值范围 |
| :--- | :--- | :--- |
| 第 1 个 `*` | 一个小时中的第几分钟 | 0 - 59 |
| 第 2 个 `*` | 一天中的第几个小时 | 0 - 23 |
| 第 3 个 `*` | 一个月中的第几天 | 1 - 31 |
| 第 4 个 `*` | 一年中的第几个月 | 1 - 12 |
| 第 5 个 `*` | 一周中的星期几 | 0 - 7（0 和 7 均代表周日） |



### 10.3.2 特殊符号说明
*   **`*` (星号)**：代表任何时间。例如第一个 `*` 表示每分钟执行一次。
*   **`,` (逗号)**：代表不连续的时间点。例如 `0,8,12,16 * * * *` 表示每小时的 0分、8分、12分、16分各执行一次。
*   **`-` (连字符)**：代表连续的时间范围。例如 `0 5 * * 1-6` 表示周一至周六的凌晨 5:00 执行一次。
*   **`*/n`**：代表每隔多久执行一次。例如 `*/10 * * * *` 表示每隔 10 分钟执行一次指令。

---

## 10.4. 生产环境示例：使用脚本备份数据
在实际开发（如 C++ 或数据库实验）中，通常将多个命令写进脚本，再由 crontab 调用。

### 第一步：编写 Shell 脚本
创建一个备份脚本 `backup_db.sh`：
```bash
#!/bin/bash
# 备份 MySQL 数据库并添加时间戳
DATE=$(date +%Y%m%d_%H%M%S)
# 假设备份到 /home/cpp/backups 目录下
tar -zcvf /home/cpp/backups/db_backup_$DATE.tar.gz /var/lib/mysql
```

### 第二步：赋予脚本执行权限
```bash
chmod +x /home/cpp/backup_db.sh
```

### 第三步：配置定时任务
执行 `crontab -e`，在末尾添加一行（每周一至周五的凌晨 3 点执行备份）：
```bash
0 3 * * 1-5 /home/cpp/backup_db.sh
```

## 11.一次性任务调度 (at)

## 11.1 at 指令简介
`at` 命令用于在指定的时间执行一次性的任务计划。与 `crontab` 的循环执行不同，`at` 任务执行完毕后就会从队列中删除。

### 11.2 工作原理
*   **atd 守护进程**：`at` 指令由 `atd` 守护进程负责。该进程以后台模式运行，默认每 **60 秒** 检查一次作业队列。
*   **匹配运行**：如果有作业的运行时间与当前时间匹配，该作业就会被执行。
*   **前置条件**：使用 `at` 之前必须保证 `atd` 进程已启动。
    *   查看指令：`ps -ef | grep atd`

---

## 11.3 at 指令语法
**基本格式**：
`at [选项] [时间]`

**操作流程**：
1.  输入 `at` 指令及时间，回车。
2.  进入交互模式，输入你要执行的指令。
3.  按 **`Ctrl + D`** 退出编辑并提交任务。

---

## 11.4 at 时间定义方式

| 类别 | 格式说明 | 示例 |
| :--- | :--- | :--- |
| **绝对计时** | `HH:MM` (若时间已过，则明天执行) | `at 16:30` |
| **模糊词汇** | `midnight`, `noon`, `teatime`, `tomorrow` | `at teatime` |
| **具体日期** | `月份名 日期 [年份]` | `at July 25 2026` |
| **时间戳** | `HH:MM MMDDYY` (或 `MM/DD/YY`) | `at 10:00 052026` |
| **相对计时** | `now + 数量 单位` (单位可选 minutes, hours, days, weeks) | `at now + 5 minutes` |

---

## 11.5 at 常用选项参数

*   **`-l`**：列出当前用户尚未执行的 `at` 任务队列（等同于 `atq` 命令）。
*   **`-d`**：删除指定的 `at` 任务（后面跟任务编号，等同于 `atrm` 命令）。
*   **`-c`**：输出指定任务的具体内容。
*   **`-m`**：任务执行完毕后，向用户发送邮件通知。
*   **`-f <文件>`**：从指定文件中读取指令，而不是通过手动输入。
*   **`-v`**：显示任务将被执行的详细时间。
*   **`-V`**：显示版本信息。

---

## 11.6 交互示例：5分钟后自动备份

假设你想在 5 分钟后快速备份一份代码：

1.  输入指令：
    ```bash
    at now + 5 minutes
    ```
2.  输入具体的备份逻辑：
    ```bash
    at> tar -zcvf /home/cpp/temp_backup.tar.gz /home/cpp/project
    at> <此处按下 Ctrl + D 提交>
    ```
3.  系统提示：
    `job 1 at Wed May 6 09:15:00 2026`

## 12.磁盘分区与挂载 (Mount)

## 12.1 磁盘分区与文件系统的关系
在 Windows 中，我们习惯通过盘符（C盘、D盘）访问硬盘；而在 Linux 中，所有的磁盘分区必须“嵌入”到统一的目录树结构中。

### 核心逻辑
1.  **单根结构**：Linux 只有一个根目录 `/`，其目录结构是独立且唯一的。所有的磁盘分区最终都是文件系统树的一部分。
2.  **挂载 (Mount)**：将一个磁盘分区与一个现有的目录（挂载点）建立关联的过程。访问该目录即等同于访问对应的磁盘空间。
3.  **根挂载点**：系统必须将根目录 `/` 挂载到某个硬盘分区上，Linux 系统才能正常启动和运行。
4.  **互斥原则**：不同的分区可以挂载到不同的目录下，但**不能**将多个分区同时挂载到同一个目录，否则会发生覆盖。

---

## 12.2 磁盘命名规则
Linux 中一切皆文件，硬盘被抽象为 `/dev/` 目录下的设备文件。

### 12.3 硬盘类型标识
目前主流服务器和虚拟机（如你的 VMware 环境）大多使用 **SCSI/SATA** 接口。

| 硬盘类型 | 标识符 | 命名规律 | 示例 |
| :--- | :--- | :--- | :--- |
| **IDE 硬盘** | `hd` | `hdx~` (x 为 a,b,c...; ~ 为分区号) | `hda1` (第1块IDE盘的第1分区) |
| **SCSI 硬盘** | `sd` | `sdx~` (x 为 a,b,c...; ~ 为分区号) | `sdb2` (第2块SCSI盘的第2分区) |

*   **x 的含义**：代表磁盘序号（a 为第1块，b 为第2块，以此类推）。
*   **~ 的含义**：代表分区序号（1-4 通常为物理/主分区，5 以后为逻辑分区）。

---

## 12.4 查看磁盘分区信息
在终端中使用以下指令查看当前系统的磁盘分布及挂载情况：

*   **`lsblk`**：(List Block Devices) 最常用的指令，以树状结构展示磁盘、分区及挂载点。
*   **`lsblk -f`**：显示更详细的信息，包括文件系统类型（如 ext4, xfs）和 UUID（分区的唯一标识符）。

**输出示例解读：**
```text
NAME   MAJ:MIN RM  SIZE RO TYPE MOUNTPOINT
sda      8:0    0   20G  0 disk 
├─sda1   8:1    0   18G  0 part /
└─sda2   8:2    0    2G  0 part [SWAP]
```
*这表示 `sda` 是一块 20G 的硬盘，其中第 1 个分区挂载在根目录 `/` 下。*

---

## 12.5 新硬盘分区、格式化与挂载

当你为虚拟机添加了一块新硬盘（假设为 `/dev/sdb`）后，需要经过以下三个核心步骤才能使用：

### ① 分区 (fdisk)
使用 `fdisk` 工具进入交互界面：
```bash
sudo fdisk /dev/sdb
```
* **常用交互指令**：
    * `m`：查看帮助名单。
    * `n`：新建分区。
    * `p`：指定为主分区（Primary）。
    * `w`：**写入并退出**（如果不输入 w，之前的操作都不会生效）。

### ② 格式化 (mkfs)
分区完成后，直接 `lsblk -f` 会发现没有 UUID，此时需要格式化注入文件系统：
```bash
# -t 指定文件系统类型，ext4 是 Linux 常用的格式
sudo mkfs -t ext4 /dev/sdb1
```

### ③ 挂载 (Mount) 步骤
挂载分为**临时挂载**和**永久挂载**：

1.  **临时挂载**（重启后失效）：
    ```bash
    mkdir /home/newdisk       # 先创建一个空的挂载点目录
    sudo mount /dev/sdb1 /home/newdisk
    ```
2.  **永久挂载**（开机自动挂载）：
    * 编辑系统挂载配置文件：`sudo vim /etc/fstab`
    * 在文件末尾添加一行：
        `/dev/sdb1  /home/newdisk  ext4  defaults  0  0`
    * 保存退出后，执行 `mount -a` 验证是否配置正确。

## 12.6 磁盘查询指令

### ① df -h (Disk Free)
* **功能**：查看全系统磁盘分区的整体使用情况。
* **输出内容**：容量、已用、可用、已用百分比、挂载点。
* **示例**：`df -h`（-h 表示以人类可读的 GB/MB 显示）。

### ② du -h (Disk Usage)
* **功能**：查看指定**目录或文件**所占用的磁盘空间。
* **参数补充**：
    * `-s`：只显示总计（Summary）。
    * `-a`：显示所有文件，而不只是目录。
    * `--max-depth=1`：子目录深度，只看当前这一层。
    * `-c`：增加一列总合计（Total）。

#### 实用例子：
* **查看当前文件夹总大小**：`du -sh`
* **查看 /opt 下第一层各个目录的大小**：`du -h --max-depth=1 /opt`
* **查看并显示合计**：`du -hc /home`

## 12.7 磁盘内容统计实战

在 Linux 中，通过组合 `ls`、`grep` 和 `wc` 可以实现强大的统计功能。

### ① 统计文件与目录数量
* **统计 /opt 下的文件个数**（不包含子文件夹）：
    `ls -l /opt | grep "^-" | wc -l`
* **统计 /opt 下的所有目录个数**（递归统计）：
    `ls -lR /opt | grep "^d" | wc -l`
* **统计 /opt 下的所有文件个数**（包含子文件夹里的文件）：
    `ls -lR /opt | grep "^-" | wc -l`

> **原理说明**：
> * `ls -lR`：递归列出详细信息。
> * `grep "^-"`：过滤出以 `-` 开头的行（代表普通文件）。
> * `grep "^d"`：过滤出以 `d` 开头的行（代表目录）。
> * `wc -l`：统计行数。

### ② 树状结构显示
* **命令**：`tree`
* **安装**：如果提示命令没找到，执行 `sudo apt install tree`。
* **使用**：`tree /home`（可以直观看到目录的层级关系）。
