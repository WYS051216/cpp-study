# 面向对象与设计模式核心笔记 (Q&A)

---

### Q1：如何理解面向对象？
**答：**
1. **概念**：面向对象是一种软件开发方法、编程范式和理解现实世界的抽象方法。它是计算机编程技术发展到一定阶段，为了应对软件复杂性而产生的产物。
2. **应用领域**：不仅限于软件开发，还涵盖数据库系统、交互式界面、应用平台、分布式系统等几乎所有现代计算机领域。
3. **对象的概念**：对象是具体存在的事物（现实或抽象）。在程序中，对象是**类**的实例。类定义了特征（**属性**，表示“有什么”）和行为（**方法**，表示“能做什么”）。
4. **与面向过程的区别**：
   - **面向过程**：分析解决问题的步骤，用函数一步步实现，依次调用。
   - **面向对象**：将问题涉及的事物分解成对象，建立对象的目标是为了描述该事物在解决整个问题中的行为。
   - **示例（五子棋）**：
     - **面向过程**：开始游戏（绘图）-> 决定先手 -> 玩家落子 -> 绘制棋子 -> 判断胜负 -> 循环。
     - **面向对象**：拆分为游戏面板对象（绘图）、玩家对象（下棋）、规则对象（判断输赢）。

### Q2：面向对象编程包含哪些特性？
**答：**
1. **封装 (Encapsulation)**：也叫信息隐藏。将属性和方法封装在一起，对外隐藏实现细节。
   - **优点**：减少耦合、方便独立开发测试、降低维护负担、专人专事、提升性能调优效率、降低大型系统风险。
2. **抽象 (Abstraction)**：隐藏方法的具体实现，让调用者只关心功能而非过程。函数本身就是一种抽象行为，包裹了具体的逻辑。
3. **继承 (Inheritance)**：基于 `is-a` 关系实现代码复用。子类复用父类的属性和方法，是支持多态的表现形式。
4. **多态 (Polymorphism)**：
   - **编译时多态**：主要指函数重载。
   - **运行时多态**：在程序运行时才确定对象具体指向哪一个具体类实例，使程序更灵活。

### Q3：什么是设计模式？
**答：**
1. **诞生与定义**：源于建筑行业（亚历山大）。定义为：在特定环境下，解决某类重复出现问题的一套成功或有效的解决方案。
2. **软件设计模式**：在软件工程中，为重复出现的软件问题提供合理、有效的方案，描述类与对象之间的相互作用。
3. **基本要素**：
   - **模式名称**：简短的词汇（中英文），方便开发人员沟通。
   - **问题**：描述何时使用该模式及其背后的原因。
   - **解决方案**：描述模式的组成部分、职责及协作方式（通常通过类图和核心代码表示）。
   - **效果**：描述模式的优缺点及权衡。

---

### Q4：设计模式如何分类？
**答：**
1. **按用途划分**：
   - **创建型**：用于如何创建对象（如单例、工厂）。
   - **结构型**：用于描述类和对象的组合（如适配器、代理）。
   - **行为型**：用于描述类或对象的交互及职责分配（如观察者、策略）。
2. **按范围划分**：
   - **类模式**：处理类与子类的关系（继承），编译时确定（静态）。
   - **对象模式**：处理对象间的关系，运行时变化（动态）。除了工厂方法、类适配器、解释器、模板方法外，大多为对象模式。



---

### Q5：为什么要学习设计模式？
**答：**
1. **官方原因**：实现可维护性的复用，提高开发效率；提供通用设计词汇，降低沟通成本；兼顾重用性与维护性，增强系统灵活性。
2. **个人提升**：加深对面向对象思想的理解；能针对具体问题给出成熟方案；提高阅读开源框架和底层源码的能力。

---

### Q6：如何描述类与类之间的关系？
**答：**
1. **关联关系**：成员对象间的结构化关系。代码表现：**成员变量**。
2. **聚合关系**：整体与部分，成员可独立于整体存在。代码表现：**成员变量**。
3. **组合关系**：整体与部分，整体控制成员生命周期（同生共死）。代码表现：**成员变量**。
4. **依赖关系**：使用关系。代码表现：**方法参数**或**局部变量**。
5. **泛化关系**：即**继承**关系。
6. **实现关系**：类实现接口的操作。

### Q7：面向对象设计原则有哪些？
**答：**
1. **单一职责 (SRP)**：一个类只负责一个功能。
2. **开闭原则 (OCP)**：对扩展开放，对修改关闭（核心原则）。
3. **里氏替换 (LSP)**：所有引用基类的地方必须能透明地使用其子类。
4. **依赖倒置 (DIP)**：高层不依赖低层，两者都依赖抽象；针对接口编程。
5. **接口隔离 (ISP)**：不依赖不需要的接口，接口要精简。
6. **组合/聚合复用 (CRP)**：优先使用组合/聚合，而非继承。
7. **迪米特法则 (LoD)**：最少知识原则，减少对象间的不必要了解。
