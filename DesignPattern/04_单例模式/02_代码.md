## 1. 饿汉模式 (Eager Initialization)

**【功能特性】**：类加载时即创建实例，天生线程安全，执行效率高。
**【核心技术】**：静态成员变量类外初始化。

```cpp
// 饿汉模式：定义类的时候创建单例对象
#include<iostream>
using namespace std;

class TaskQueue
{
public:
    // 禁用拷贝构造与赋值运算符，确保实例唯一
    TaskQueue(const TaskQueue& t) = delete;
    TaskQueue& operator = (const TaskQueue& t) = delete;

    // 静态成员函数：外部获取唯一实例的官方接口
    static TaskQueue* getInstance()
    {
        return m_taskQ;
    }

    void print()
    {
        cout << "我是单例对象的一个成员函数..." << endl;
    }

private:
    // 私有化构造函数，防止外部 new
    TaskQueue() = default;

    // 静态指针成员变量
    static TaskQueue* m_taskQ;
};

// 在类外部进行初始化（程序启动时执行）
TaskQueue* TaskQueue::m_taskQ = new TaskQueue;

int main()
{
    TaskQueue* taskQ = TaskQueue::getInstance();
    taskQ->print();
    return 0;
}
```

## 2. 懒汉模式 - 双重检查锁定 (DCL + Atomic)

**【功能特性】**：延迟加载（用时才创建），通过双重检查降低锁竞争开销。
**【核心技术】**：`std::atomic` 原子变量防止指令重排，`std::mutex` 保证线程安全。

```cpp
// 懒汉模式：使用原子变量解决双重检查锁定的线程安全问题
#include<iostream>
#include<mutex>
#include<atomic>
using namespace std;

class TaskQueue
{
public:
    TaskQueue(const TaskQueue& t) = delete;
    TaskQueue& operator = (const TaskQueue& t) = delete;

    static TaskQueue* getInstance()
    {
        TaskQueue* task = m_taskQ.load();
        if(task == nullptr)
        {
            m_mutex.lock();
            task = m_taskQ.load();
            if(task == nullptr) // 二次检查
            {
                task = new TaskQueue;
                m_taskQ.store(task); 
            }
            m_mutex.unlock();
        }
        return task;
    }

    void print()
    {
        cout << "我是单例对象的一个成员函数..." << endl;
    }

private:
    TaskQueue() = default;
    static mutex m_mutex;
    static atomic<TaskQueue*> m_taskQ;
};

// 静态成员变量类外定义
atomic<TaskQueue*> TaskQueue::m_taskQ;
mutex TaskQueue::m_mutex;

int main()
{
    TaskQueue* taskQ = TaskQueue::getInstance();
    taskQ->print();
    return 0;
}
```

## 3. 懒汉模式 - 静态局部变量 (Meyers' Singleton)

**【功能特性】**：C++11 之后的标准做法，实现最简洁。
**【核心技术】**：利用 C++11 标准中“静态局部变量初始化是线程安全”的特性。

```cpp
// 懒汉模式：利用静态局部变量实现（C++11标准）
#include<iostream>
using namespace std;

class TaskQueue
{
public:
    TaskQueue(const TaskQueue& t) = delete;
    TaskQueue& operator = (const TaskQueue& t) = delete;

    static TaskQueue* getInstance()
    {
        // 静态局部变量，只在第一次执行时初始化，且编译器保证线程安全
        static TaskQueue task;
        return &task;
    }

    void print()
    {
        cout << "我是单例对象的一个成员函数..." << endl;
    }

private:
    TaskQueue() = default;
};

int main()
{
    TaskQueue* taskQ = TaskQueue::getInstance();
    taskQ->print();
    return 0;
}
```

## 🛠️ 核心逻辑问答 (Q&A 归纳)

### Q1：为什么 `getInstance()` 必须是静态 (static) 的？
* **真相**：构造函数私有了，外面没法 `new` 对象。
* **逻辑**：普通函数必须“有对象”才能调。静态函数属于“类”，**不需要对象**就能通过类名调用。它是进入这个封闭类的唯一“合法通道”。

### Q2：为什么变量 `m_taskQ` 也要写成静态 (static) 的？
* **逻辑**：
    1. **权限匹配**：静态函数（getInstance）只能抓到同样是 `static` 的变量。
    2. **唯一性**：`static` 变量在内存全局区**只有一个坑位**，保证了物理地址的唯一。
    3. **初始化时机**：只有静态变量才允许在类外（main之前）初始化（针对饿汉式）。

### Q3：为什么类内叫 `m_taskQ`，而 main 函数里叫 `taskQ`？
* **关系**：`m_taskQ` 是类内的**长期管家**（持有真实地址），`main` 里的 `taskQ` 是**临时领路人**。调用函数时，只是把地址从管家手里复制到了领路人手里。名字不重要，**地址是同一个**。

### Q4：无论外面写多少个变量，地址都一样吗？
* **真相**：**是的**。无论调用多少次 `getInstance()`，它返回的永远是那个预先创建好或第一次创建时的静态地址，确保操作的是同一个实体。
